Configuration de votre serveur comme hôte par défaut du routeur

Si votre serveur se connecte à Internet via un routeur câble, un routeur DSL ou un autre routeur réseau configuré comme un appareil NAT, vous pouvez configurer votre routeur afin qu’il envoie toutes les demandes de service entrantes à votre serveur. Utilisez le logiciel de configuration du routeur pour définir l’adresse IP privée du serveur comme l’hôte par défaut, également appelé le serveur DMZ ou l’hôte DMZ. L’adresse IP privée du serveur est affichée dans la sous-fenêtre Informations de Préférences du serveur. Pour obtenir des informations précises sur la configuration de l’hôte par défaut sur votre routeur, consultez la documentation de celui-ci.

Important :  si vous configurez votre serveur comme l’hôte par défaut, protégez-le en activant son coupe-feu lorsque vous avez terminé la configuration initiale de Mac OS X Server. Pour obtenir des instructions, consultez Modification des réglages de sécurité.

Au lieu de configurer votre serveur comme l’hôte par défaut du routeur, vous pouvez configurer le routeur afin qu’il protège votre réseau local tout en autorisant un accès depuis Internet à des services sélectionnés. Vous configurez alors votre routeur pour qu’il transfère certaines demandes de service à votre serveur. Cette procédure s’appelle la redirection de ports ou le mappage de ports. Pour obtenir des instructions, consultez le chapitre 2 « Préparation pour Mac OS X Server » dans le manuel Premiers contacts.
