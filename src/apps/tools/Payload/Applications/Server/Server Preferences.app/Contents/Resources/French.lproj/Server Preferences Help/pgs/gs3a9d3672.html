Configuration de Mac qui se connectent à votre réseau

Si un Mac exécutant Mac OS X 10.6 n’est pas encore connecté à un serveur et si ce Mac s’assure que l’utilisateur actuellement en session correspond à un compte utilisateur créé sur votre serveur, le Mac affiche une invitation proposant de configurer les connexions à votre serveur. L’utilisateur peut accepter l’invitation, la refuser ou retarder sa décision jusqu’à sa prochaine ouverture de session. Si le Mac découvre plusieurs serveurs auxquels l’utilisateur pourrait se connecter, l’invitation propose à l’utilisateur un choix de serveurs.

L’invitation apparaît uniquement si le Mac exécute Snow Leopard et s’il détecte un serveur exécutant Snow Leopard Server sur le réseau. Le compte local de l’utilisateur sur le Mac doit comporter le même nom ou le même nom abrégé qu’un compte utilisateur sur le serveur et au moins un service doit être activé sur le serveur. Si l’utilisateur a déjà ouvert une session sur le Mac lorsque vous ajoutez un compte pour cet utilisateur sur le serveur, l’utilisateur doit fermer la session et la rouvrir (ou redémarrer le Mac) pour que l’invitation s’affiche. Si le compte de l’utilisateur sur le serveur est importé depuis un serveur d’annuaire, l’utilisateur reçoit une invitation par courrier électronique et la fenêtre d’invitation décrite ci-dessus ne s’affiche pas. Si un utilisateur Snow Leopard ne voit pas d’invitation, il peut démarrer manuellement la procédure de configuration au moyen de la sous-fenêtre Comptes des Préférences Système. Pour obtenir des instructions, consultez Configuration d’un Mac à l’aide des préférences Comptes.

Si l’utilisateur accepte l’invitation, la sous-fenêtre Comptes des Préférences Système s’ouvre. Elle invite l’utilisateur à saisir le nom et le mot de passe du compte utilisateur sur votre serveur.

Préférences Comptes configure alors le Mac de l’utilisateur et affiche des informations sur la procédure effectuée. Pour obtenir des informations sur les modifications de comptes utilisateur et sur les services auxquels l’utilisateur peut accéder, consultez Configuration automatique des Mac des utilisateurs.
