ファイル共有サービスを管理する

「ファイル共有」パネルを使用して、ファイル共有サービスを有効または無効にしたり、「Groups」、「Public」、「ユーザ」の各共有フォルダへのアクセスを制御したりします。また、自分の共有フォルダ（共有ポイントとも呼ばれます）を追加したり、削除したりすることもできます。

「ファイル共有」パネルを構成するには：

  • 「入」／「切」スイッチをクリックして、ファイル共有サービスを有効または無効します。また、このコントロールは、ファイル共有が有効であるか無効であるかを示しています。

  • 「共有フォルダ」リストの各チェックボックスをクリックして、「Backups」フォルダ、「Groups」フォルダ、「Public」フォルダ、「ユーザ」フォルダ、および追加した各共有フォルダへのアクセスを有効または無効にします。

  • 追加（＋）ボタンをクリックして、新しい共有フォルダを追加します。

    共有フォルダを追加する

  • 削除（－）ボタンをクリックして、ファイル共有ユーザが共有フォルダを使用できないようにします。

    共有フォルダを取り除く

  • 「アクセス権を編集」をクリックして、共有フォルダへのゲストアクセスを有効にするか、無効にするかを選択し、ユーザの共有フォルダの読み出し／書き込みアクセス権を変更します。

    共有フォルダへのアクセスを制御する

「Backups」共有フォルダには、「Time Machine」環境設定でサーバをバックアップの保管場所として使用するように設定しているユーザの、バックアップデータが保管されます。「Backups」共有フォルダでは、「アクセス権を編集」ボタンや削除（－）ボタンは使用できません。

「Groups」共有フォルダには、「グループ」パネルで設定されたファイル共有フォルダを持つ各グループのフォルダが含まれます。削除（－）ボタンをクリックしても、ファイル共有ユーザが「Groups」フォルダを使用できなくなることはありません。

「Public」共有フォルダは、「サーバ環境設定」の「ユーザ」パネルに一覧されるアカウントを持つユーザが、ファイルやフォルダを格納する場所です。

「ユーザ」共有フォルダは、「システム環境設定」の「アカウント」パネルに一覧されるアカウントを持つユーザの、ホームフォルダを格納する場所です。
