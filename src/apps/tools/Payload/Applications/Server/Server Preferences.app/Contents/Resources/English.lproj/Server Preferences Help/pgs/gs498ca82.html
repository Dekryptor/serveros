Making Your Server the Router’s Default Host

If your server gets its Internet connection through a cable router, DSL router, or other network router configured as a NAT device, you can configure your router to send all incoming requests for service to your server. Use the router’s configuration software to set the server’s private IP address as the default host, also called the DMZ server or DMZ host. The server’s private IP address is shown in the Information pane of Server Preferences. For specific information about how to configure the default host on your router, see its documentation.

Important:  If you make your server the default host, protect it by turning on its firewall after you finish initial Mac OS X Server setup. For instructions, see Changing Security Settings.

Instead of making your server the router’s default host, you can configure the router to protect your local network while allowing access to selected services from the Internet. You configure your router to forward requests for individual services to your server. This process is called port forwarding or port mapping. For instructions, see Chapter 2, “Getting Ready for Mac OS X Server,” in the Getting Started manual.
