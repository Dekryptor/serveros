カスタム Web サイトを公開する

Wiki の設定のほかに、静的な HTML ファイルで構成されたカスタム Web サイトをお使いのサーバで公開することができます。自分で選んだ Web 開発ソフトウェアを使用して作成するか、またはほかの人に作成してもらったカスタム Web サイトのファイルをサーバにコピーします。 各カスタム Web サイトは、仮想ホストとも呼ばれます。

カスタム Web サイトを公開するには：

  1. 「サーバ環境設定」の「Web」パネルで「カスタムサイト」をクリックします。

  2. 新規の Web サイトを追加するには、追加（＋）ボタンをクリックして Web サイトの完全修飾 DNS 名を入力し、必要に応じてサーバ上で Web サイトのファイルが保管されているフォルダを選択します。

    「サーバ環境設定」では、入力した DNS 名が未使用であればその DNS 名の Web サイトが作成され、ご使用の DNS サービスでサーバの IP アドレスに対してその名前を解決することができます。

    • 入力した DNS 名がすでに標準 Web サービス（Wiki、Web カレンダー、ブログ、または Web メール）で使用されている場合は、その DNS 名を変更するか、非標準的なアクセスポート番号を指定することができます。標準のアクセスポート番号は 80 です。別のポート番号を指定する場合は、サーバによって提供される別のサービスと競合していないことを、あらかじめ確認してください。Mac OS X Server で使用されるポートのリストについては、「サーバ管理」（「/アプリケーション/サーバ/」にあります）を開き、「ヘルプ」メニューで「TCP および UDP ポートの参照」を検索してください。

    • DNS 名の右側にある状況インジケータが赤くなっている場合、「サーバ環境設定」ではお使いのサーバの IP アドレスに対して DNS 名を解決できません。この状態を解消するには、DNS 名を変更するか、「作成」をクリックして手動で IP アドレスを設定します。

    Web サイトのファイルのフォルダを選択しない場合は、「サーバ環境設定」によって、デフォルトの場所（/ライブラリ/WebServer/Sites/＜ドメイン名＞（ドメイン名の部分には、入力した DNS 名が入ります）にフォルダが作成されます。

  3. Web サイトで使用可能にしたいファイルを Web サイトフォルダにコピーし、不要なプレースホルダファイルを削除します。

    Web サイトのホームページの名前が index.html または index.php であることを確認します。

    「Web」パネル内の「カスタムサイト」パネルで、Web サイト名の下に Web サイトフォルダの場所が表示されます。

  4. Web サイトの横にあるチェックボックスが選択されており、その Web サービスが有効になっていることを確認します。

    Web サイトのチェックボックスの選択が解除されている場合、その Web サイトは使用できません。

    Web サービスが無効になっている場合は、すべてのカスタム Web サイトが使用できません。

Web サイトフォルダにファイルとフォルダを追加した後は、Web サイトの全部または一部へのアクセスを制限できます。操作方法については、「カスタム Web サイトへのアクセスを制限する」を参照してください。
