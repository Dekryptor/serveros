Definieren des Servers als Standard-Host des Routers

Wenn Ihr Server über einen Kabel- oder DSL-Router oder einen anderen als NAT-Gerät konfigurierten Netzwerk-Router auf das Internet zugreift, können Sie den Router so konfigurieren, dass alle eingehenden Dienstanforderungen an Ihren Server gesendet werden. Verwenden Sie die Konfigurationssoftware des Routers, um die private IP-Adresse des Servers als Standard-Host, auch "DMZ-Server" oder "DMZ-Host" genannt, festzulegen. Die private IP-Adresse des Servers wird in der Servereinstellung "Informationen" angezeigt. In der Dokumentation zu Ihrem Router finden Sie spezifische Informationen darüber, wie Sie den Standard-Host Ihres Routers konfigurieren müssen.

Wichtig:  Wenn Sie den Server als Standard-Host einrichten, schützen Sie ihn durch Aktivierung der Firewall, nachdem die Mac OS X Server-Erstkonfiguration abgeschlossen ist. Anleitungen hierzu finden Sie unter Ändern von Sicherheitseinstellungen.

Anstatt den Server als Standard-Host des Routers festzulegen, können Sie den Router so konfigurieren, dass er Ihr lokales Netzwerk schützt und gleichzeitig den Zugriff auf ausgewählte Dienste über das Internet zulässt. Sie konfigurieren den Router für die Weiterleitung von Anforderungen nach einzelnen Diensten an Ihren Server. Dieser Vorgang wird als Portweiterleitung oder Portzuordnung bezeichnet. Anleitungen hierzu finden Sie in Kapitel 2 "Vorbereitung für die Installation von Mac OS X Server" der Einführung.
