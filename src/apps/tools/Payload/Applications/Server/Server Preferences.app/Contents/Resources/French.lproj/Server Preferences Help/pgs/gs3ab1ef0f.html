À propos de la sécurité des coupe-feu

Le coupe-feu que vous pouvez contrôler dans la sous-fenêtre Sécurité de Préférences du serveur est le coupe-feu Mac OS X Server. Appelé coupe-feu IP, il accepte ou refuse le trafic entrant ou sortant en se basant sur les attributs du trafic, tels que le port de destination ou l’adresse IP d’origine. Pour obtenir des informations sur le coupe-feu IP, ouvrez Admin Serveur, puis utilisez le menu Aide et recherchez la rubrique de vue d’ensemble des services de coupe-feu.

Votre serveur utilise également le coupe-feu Mac OS X, qui fonctionne différemment. Ce coupe-feu est appelé coupe-feu d’application, car il accepte ou refuse les connexions entrantes en fonction de l’application, du service ou de tout autre module logiciel particulier qui tente d’accepter la connexion. Ce coupe-feu ne contrôle pas le trafic réseau sortant. Vous gérez le coupe-feu d’application au moyen des Préférences Système et non Préférences du serveur.

Si vous avez mis à niveau votre serveur à partir de Mac OS X Server 10.5, le coupe-feu d’application peut éventuellement être actif. Vous devez le désactiver dans la sous-fenêtre Sécurité des Préférences Système avant de pouvoir gérer le coupe-feu IP dans la sous-fenêtre Sécurité des Préférences du serveur.

Le service VPN et le coupe-feu de votre serveur peuvent tous deux autoriser l’accès à des services proposés à l’extérieur de votre réseau local. Toutefois, le service VPN nécessite une authentification pour un tel accès alors que cela n’est pas nécessaire pour un accès autorisé via le coupe-feu. Si le service VPN est activé, vous souhaitez peut-être ne pas exposer certains services à Internet via votre coupe-feu Par exemple, vous pouvez définir le coupe-feu pour exposer sur Internet vos services web uniquement, afin que le public puisse afficher votre site web. Les utilisateurs de votre serveur peuvent accéder aux autres services : partage de fichiers, carnet d’adresses, iCal, iChat et courrier électronique, au moyen d’une connexion VPN.
