サーバをディレクトリサーバに接続する

お使いのサーバに固有のユーザおよびグループがある場合、所属する組織の Open Directory サーバまたは Active Directory サーバからユーザアカウントを読み込むようにサーバを設定することもできます。ユーザアカウントの読み込みを可能にするには、サーバがディレクトリサーバに接続している必要があります。読み込まれたユーザアカウントがアクセスできるサービスは、サーバ上に作成したユーザアカウントと同じです。

接続しているディレクトリサーバから一部のユーザアカウントを読み込まない場合は、それらのアカウントをグループの外部メンバーにすることができます。 それらのアカウントにサーバのプライベート Wiki へのアクセス権を付与することもできます。詳細については、「ユーザを読み込む」、「グループの外部メンバーを追加する／取り除く」、および「Web サービスを設定する」を参照してください。

ディレクトリサーバに接続するには：

  1. 「システム環境設定」の「アカウント」パネルを開きます。

    「アカウント」環境設定を開く

  2. 「ログインオプション」、「編集」とクリックします。

    「編集」ボタンが表示されず「接続」ボタンは表示されている場合、既存のディレクトリサーバからユーザを読み込むよう設定する前に、サーバが固有のユーザおよびグループを持つように設定しておく必要があります。操作方法については、「ユーザおよびグループの管理を設定する」を参照してください。

  3. 追加（＋）ボタンをクリックして、ポップアップメニューからディレクトリサーバを選択するか、そのディレクトリサーバの DNS 名または IP アドレスを入力します。

  4. ダイアログが拡大して「クライアントコンピュータ ID」、「ユーザ名」、および「パスワード」の各フィールドが表示された場合は、ディレクトリサーバのユーザアカウントの名前とパスワードを入力します。

    • Open Directory サーバの場合は、標準のユーザアカウントの名前とパスワードを入力できます。ディレクトリの管理者アカウントを使用する必要はありません。ダイアログに名前とパスワードのフィールドを空白にしてかまわないというメッセージが表示された場合は、安全性は劣りますが認証なしで接続が可能です。

    • Active Directory サーバの場合は、Active Directory の管理者アカウント、またはワークステーションをドメインに追加する権限を持つ標準のユーザアカウントの、名前とパスワードを入力します。
