ユーザが自分のコンピュータでサーバを使用するよう設定できない場合

サーバの DNS 名を変更する場合、サーバの古い DNS 名を使ってサービスを利用していたユーザは、新しい DNS 名を使うために自分のコンピュータを再設定する必要があります：

  • Mac OS X v10.6 Snow Leopard を使用するユーザは、「システム環境設定」の「アカウント」パネルを開き、「ログインオプション」をクリックし、「編集」をクリックしてから、ディレクトリサーバのリストでサーバを選択し、削除（－）ボタンをクリックして、リストからサーバを削除する必要があります。次に、追加（＋）ボタンをクリックして、サーバをもう一度リストに追加する必要があります。詳細については、「アカウント環境設定を使用して Mac を設定する」を参照してください。

    Snow Leopard ユーザのコンピュータが別のディレクトリサーバと接続している場合は、まれに、そのコンピュータがサーバに接続できなくなることがあります。上記のすべてのディレクトリサーバを削除すると、ユーザはサーバへの接続を追加できるようになります。

  • 古いバージョンの Mac OS X を使用するユーザまたは Windows を実行しているユーザは、共有カレンダー照会、iChat アドレス、メールアドレス、Web ブックマーク、およびサーバの VPN アドレスを、手動でアップデートする必要があります。詳細については、「ユーザのコンピュータを手動で設定する」を参照してください。
