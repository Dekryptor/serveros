Modifications d’un compte utilisateur

Lorsque l’utilisateur termine l’une des procédures de configuration automatique, le compte local de l’utilisateur (sur son Mac) est lié au compte de l’utilisateur sur le serveur et ce compte local est étiqueté « Contrôlé » dans la sous-fenêtre Comptes des Préférences Système de l’utilisateur.

  • Si le compte de l’utilisateur sur le serveur est importé à partir d’un compte existant sur un serveur d’annuaire, le mot de passe du compte local est remplacé par le mot de passe du compte sur le serveur et ces mots de passe restent synchronisés.

  • Si le compte sur le serveur a été créé (et non importé) sur votre serveur, le mot de passe du compte local n’est pas modifié ni synchronisé.

Si les comptes d’un utilisateur ont été créés avec différents noms d’utilisateur, l’utilisateur peut modifier le nom complet du compte local à l’aide de la sous-fenêtre Comptes des Préférences Système.

Pour obtenir des informations sur les comptes utilisateur locaux, sur le serveur et importés, consultez À propos des comptes utilisateur.

Une fois la configuration automatique terminée, l’utilisateur est prêt à accéder aux services comme indiqué dans le tableau ci-dessous. Bien sûr, il peut accéder uniquement aux services activés. Vous pouvez contrôler séparément l’accès de chaque utilisateur aux services, comme décrit dans Contrôle de l’accès d’un utilisateur aux différents services.
