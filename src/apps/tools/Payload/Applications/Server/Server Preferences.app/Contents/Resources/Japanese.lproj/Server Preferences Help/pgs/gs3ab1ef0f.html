ファイアウォールセキュリティについて

「サーバ環境設定」の「セキュリティ」パネルで制御できるファイアウォールは、Mac OS X Server ファイアウォールです。「IP ファイアウォール」と呼ばれるこのファイアウォールは、送信先ポートや送信元 IP アドレスなど、トラフィックの属性に基づいて受信トラフィックおよび送信トラフィックを受け付けるか、または拒否します。IP ファイアウォールについては、「サーバ管理」を開いて「ヘルプ」メニューで「ファイアウォールサービスの概要」を検索してください。

お使いのサーバは Mac OS X ファイアウォールも備えています。このファイアウォールは動作が異なり、「アプリケーションファイアウォール」と呼ばれます。これは、このファイアウォールが、受信接続を受け付けようとする特定のアプリケーション、サービス、またはそのほかのソフトウェアモジュールに基づいてその接続を受け付けるか、または拒否するためです。このファイアウォールでは、送信ネットワークトラフィックは制御されません。アプリケーションファイアウォールは、「サーバ環境設定」ではなく「システム環境設定」で管理します。

サーバを Mac OS X Server v10.5 Leopard からアップグレードした場合は、アプリケーションファイアウォールが有効になることがあります。「サーバ環境設定」にある「セキュリティ」パネルで IP ファイアウォールを管理するには、「システム環境設定」の「セキュリティ」パネルでアプリケーションファイアウォールを無効にする必要があります。

お使いのサーバのファイアウォールと VPN サービスは、どちらもローカルネットワークの外部からのサービスへのアクセスを許可できます。異なる点は、VPN サービスにはアクセスの認証が必要ですが、ファイアウォールを介して許可されたアクセスは認証が不要ということです。 VPN サービスが有効な場合は、お使いのファイアウォールを介して一部のサービスをインターネットに公開する必要がなくなることがあります。たとえば、Web サービスのみがインターネットに公開されるようにファイアウォールを設定して、自分の Web サイトを一般ユーザが閲覧できるようにすることが可能です。サーバのユーザは、VPN 接続を介してほかのサービス（ファイル共有、アドレスブック、iCal、iChat、メール）にアクセスできます。
