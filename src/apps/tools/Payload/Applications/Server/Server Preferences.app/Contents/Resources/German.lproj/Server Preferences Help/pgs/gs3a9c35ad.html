Benutzer können ihre Computer nicht für die Nutzung des Servers konfigurieren

Wenn Sie den DNS-Namen Ihres Servers ändern, müssen Benutzer, die über den bisherigen DNS-Namen auf Dienste zugegriffen haben, ihre Computer für die Verwendung des neuen DNS-Namens konfigurieren:

  • Benutzer mit Mac OS X 10.6 Snow Leopard müssen den Server aus der Liste der Verzeichnisserver entfernen. Dazu öffnen sie die Systemeinstellung "Benutzer", klicken auf "Anmeldeoptionen" und dann auf "Bearbeiten", wählen Ihren Server in der Liste aus und klicken abschließend auf die Taste "Löschen" (–). Danach müssen sie den Server wieder zur Liste hinzufügen, indem sie auf die Taste "Hinzufügen" (+) klicken. Weitere Informationen finden Sie unter Konfigurieren eines Mac-Computers mit der Systemeinstellung "Benutzer".

    In seltenen Fällen kann ein Snow Leopard-Benutzer u. U. keine Verbindung zu Ihrem Server herstellen, wenn der Computer des Benutzers mit einem weiteren Verzeichnisserver verbunden ist. Nach dem Entfernen aller Verzeichnisserver mit den oben beschriebenen Schritten sollten die Benutzer eine Verbindung zu Ihrem Server herstellen können.

  • Benutzer, die mit einer älteren Mac OS X-Version oder mit Windows arbeiten, müssen Abonnements freigegebener Kalender, iChat-Adressen, E-Mail-Adressen, Web-Lesezeichen und die VPN-Adresse des Servers manuell aktualisieren. Weitere Informationen hierzu finden Sie unter Manuelles Konfigurieren von Benutzercomputern.
