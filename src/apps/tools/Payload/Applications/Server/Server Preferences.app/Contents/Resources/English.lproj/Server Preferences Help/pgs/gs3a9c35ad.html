If Users Can’t Configure Their Computers to Use the Server

If you change your server’s DNS name, users who were getting services using your server’s old DNS name must reconfigure their computers to use the new DNS name:

  • Users with Mac OS X v10.6 Snow Leopard need to remove your server from the list of directory servers by opening the Accounts pane in System Preferences, clicking Login Options, clicking Edit, selecting your server in the list, and clicking the Delete (–) button. Then they need to add your server back to the list by clicking the Add (+) button. For more information, see Setting Up a Mac by Using Accounts Preferences.

    In rare cases, a Snow Leopard user may be unable to make a connection to your server if the user’s computer has a connection to an additional directory server. After removing all directory servers as described above, users should be able to add a connection to your server.

  • Users who have an earlier Mac OS X version, or who are running Windows, will have to manually update shared calendar subscriptions, iChat addresses, email addresses, web bookmarks, and the server’s VPN address. For more information, see Setting Up Users’ Computers Manually.
