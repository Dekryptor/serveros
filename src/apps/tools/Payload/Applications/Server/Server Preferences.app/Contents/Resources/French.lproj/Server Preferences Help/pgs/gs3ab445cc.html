Modification des réglages de sécurité

Utilisez la sous-fenêtre Sécurité de Préférences du serveur pour protéger votre serveur des utilisateurs sur d’autres réseaux ou sur Internet.

Pour protéger votre réseau local (sous-réseau IP), vous pouvez configurer un coupe-feu ou gérer une borne d’accès AirPort Extreme (802.11n) ou un Time Capsule. Quelle que soit la méthode choisie, vous pouvez spécifier séparément les services qui acceptent des connexions entrantes provenant d’ordinateurs situés à l’extérieur du réseau local (sous-réseau IP) de votre serveur. Si vous utilisez un coupe-feu (et non une gestion AirPort), vous pouvez autoriser les connexions entrantes à tous les services depuis l’extérieur du réseau local de votre serveur.

Si la sous-fenêtre Sécurité indique que vous utilisez un coupe-feu d’application, vous pouvez gérer celui-ci dans les Préférences Système ou vous pouvez l’y désactiver et gérer le coupe-feu du serveur dans Préférences du serveur. Pour en savoir plus, consultez À propos de la sécurité des coupe-feu et Si votre serveur utilise le coupe-feu d’application.

Si votre serveur se connecte à Internet via un autre routeur de réseau qu’un Time Capsule ou une borne d’accès AirPort Extreme, vous pouvez utiliser le logiciel de configuration de ce routeur pour protéger votre serveur et votre réseau local. Dans ce cas, vous pouvez désactiver le coupe-feu dans la sous-fenêtre Sécurité. Pour obtenir des informations sur la configuration de votre routeur, consultez le chapitre 2 « Préparation pour Mac OS X Server », du manuel Premiers contacts.

Pour contrôler l’exposition des services à Internet :

  1. Si vous disposez d’une borne d’accès AirPort Extreme ou d’un Time Capsule et si le bouton « Activer la gestion AirPort » s’affiche dans la sous-fenêtre Sécurité, cliquez sur ce bouton afin d’utiliser cette borne d’accès ou ce Time Capsule pour contrôler la sécurité.

    Si vous devez vous authentifier, saisissez le mot de passe de la borne d’accès ou du Time Capsule (et non le mot de passe du réseau sans fil).

    Si vous avez modifié les options NAT de votre borne d’accès AirPort Extreme ou de votre Time Capsule afin d’utiliser votre serveur comme « hôte par défaut », vous devez utiliser le coupe-feu du serveur et non la gestion AirPort pour contrôler la sécurité. Dans ce cas, ne cliquez pas sur « Activer la gestion AirPort » dans la sous-fenêtre Sécurité. Dans la configuration par défaut d’une borne d’accès AirPort Extreme ou d’un Time Capsule, l’option d’hôte par défaut est désactivée.

    Si le bouton « Activer la sécurité du coupe-feu » s’affiche dans la sous-fenêtre Sécurité, ne cliquez pas sur ce bouton si vous utilisez les options NAT par défaut de votre borne d’accès AirPort Extreme ou de votre Time Capsule. Cliquez sur ce bouton uniquement si vous avez modifié l’option d’hôte par défaut dans la sous-fenêtre NAT de l’application Utilitaire AirPort.

  2. Dans la sous-fenêtre Sécurité de Préférences du serveur, cliquez sur le bouton Marche/Arrêt pour activer le contrôle de la sécurité.

  3. Pour autoriser un service à accepter des connexions entrantes provenant de tous les réseaux, y compris d’Internet, cliquez sur le bouton Ajouter (+) et sélectionnez le service dans le menu local.

    Si le service que vous souhaitez ajouter n’est pas répertorié dans le menu local, cliquez sur Autre, puis saisissez le nom du service et son port. Pour obtenir la liste des noms et des ports des services, ouvrez Admin Serveur, puis utilisez le menu Aide et recherchez « Préférence du port TCP et UDP ».

  4. Si vous souhaitez mettre fin à l’acceptation des connexions entrantes provenant de tous les réseaux, y compris d’Internet, par un service répertorié, sélectionnez le service concerné et cliquez sur le bouton Supprimer (–).

    Lorsque le contrôle de la sécurité est activé, les services qui ne sont pas répertoriés dans la sous-fenêtre Sécurité peuvent accepter des connexions entrantes uniquement si elles proviennent du réseau local du serveur.

Si vous modifiez le contenu de la sous-fenêtre Sécurité alors que vous utilisez la gestion AirPort, Préférences du serveur vous demande si vous souhaitez appliquer vos modifications en redémarrant votre borne d’accès ou votre Time Capsule. Ce redémarrage interrompt l’accès à Internet et le service DHCP pour tous les ordinateurs de votre réseau local pendant une minute au plus.

Si vous activez le coupe-feu et si votre serveur se connecte à Internet via un routeur de réseau, vous devez configurer votre routeur afin qu’il envoie à votre serveur toutes les demandes de service entrantes. Pour obtenir des instructions, consultez Configuration de votre serveur comme hôte par défaut du routeur.

Pour autoriser toutes les demandes de service entrantes avec un coupe-feu :

Dans la sous-fenêtre Sécurité, cliquez sur le commutateur On/off pour désactiver le coupe-feu.

Lorsque le coupe-feu est désactivé, le coupe-feu de votre serveur autorise les connexions entrantes à tous les services. Toutefois, si votre serveur se connecte à Internet via un routeur câble, un routeur DSL ou un autre routeur, vous devez également configurer ce routeur pour qu’il envoie les demandes de service entrantes à votre serveur. Pour obtenir des informations sur la configuration de votre routeur, consultez le chapitre 2 « Préparation pour Mac OS X Server », du manuel Premiers contacts.

Si vous utilisez la gestion AirPort (et non un coupe-feu), la désactiver bloque toutes les demandes de service entrantes.
