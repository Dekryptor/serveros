ホーム
セキュリティの設定を変更する

「サーバ環境設定」の「セキュリティ」パネルを使って、ほかのネットワークまたはインターネット上のユーザからサーバを保護します。

ファイアウォールを設定してローカルネットワーク（IP サブネット）を保護できます。あるいは、AirMac Extreme ベースステーション（802.11n）または Time Capsule の管理によってローカルネットワークを保護できます。どちらの場合も、サーバのローカルネットワーク（IP サブネット）の外側にあるコンピュータからの接続を受け付けるサービスを、個別に指定することができます。（AirMac マネージメントではなく）ファイアウォールセキュリティを使用する場合は、サーバのローカルネットワークの外から来るすべてのサービスへの受信接続を許可することもできます。

アプリケーションファイアウォールを使用していることが「セキュリティ」パネルに表示された場合は、「システム環境設定」でそれを管理するか、アプリケーションファイアウォールを無効にして「サーバ環境設定」でサーバのファイアウォールを管理することができます。詳細については、「ファイアウォールセキュリティについて」および「サーバでアプリケーションファイアウォールが使用されている場合」を参照してください。

サーバのインターネット接続が AirMac Extreme ベースステーションまたは Time Capsule 以外のネットワークルーターを経由している場合は、ルーターの設定ソフトウェアを使って、サーバおよびローカルネットワークを保護することができます。これを行った場合は、「セキュリティ」パネルでファイアウォールセキュリティを無効にすることができます。お使いのルーターの設定については、「お使いになる前に」マニュアルの第 2 章「Mac OS X Server の準備をする」を参照してください。

どのサービスをインターネットに公開するかを制御するには：

  1. AirMac Extreme ベースステーションまたは Time Capsule を使用しており、「セキュリティ」パネルに「代わりに AirMac マネージメントを使用」ボタンが表示されている場合は、このボタンをクリックし、AirMac Extreme ベースステーションまたは Time Capsule を使ってセキュリティを制御します。

    認証を受けるよう要求された場合は、（ワイヤレスネットワークのパスワードではなく）ベースステーションまたは Time Capsule のパスワードを入力します。

    サーバを「デフォルトのホスト」として使用するために AirMac Extreme ベースステーションまたは Time Capsule で NAT オプションを変更した場合は、（AirMac マネージメントではなく）サーバのファイアウォールを使ってセキュリティを制御する必要があります。この場合、「セキュリティ」パネルの「代わりに AirMac マネージメントを使用」はクリックしないでください。デフォルトでは、AirMac Extreme ベースステーションまたは Time Capsule の「デフォルトのホスト」オプションは無効になっています。

    「セキュリティ」パネルに「代わりにファイアウォールセキュリティを使用」ボタンが表示されていて、AirMac Extreme ベースステーションまたは Time Capsule でデフォルトの NAT オプションを使用している場合は、このボタンをクリックしないでください。このボタンは、「AirMac ユーティリティ」アプリケーションの「NAT」パネルにある「デフォルトのホスト」オプションを変更した場合のみ、クリックしてください。

  2. 「サーバ環境設定」の「セキュリティ」パネルで、「入」／「切」スイッチをクリックしてセキュリティ制御を有効にします。

  3. インターネットを含めたすべてのネットワークからの受信接続をサービスが受け付けるようにするには、追加（＋）ボタンをクリックして、ポップアップメニューでそのサービスを選択します。

    追加したいサービスがポップアップメニューに一覧表示されない場合は、「その他」を選択して、サービス名とポートを入力します。サービス名とポートのリストについては、「サーバ管理」を開いて「ヘルプ」メニューで「TCP および UDP ポートの参照」を検索してください。

  4. リストにあるサービスがインターネットを含めたすべてのネットワークからの受信接続を受け付けるのを停止するには、そのサービスを選択して削除（－）ボタンをクリックします。

    セキュリティ制御が有効になっているとき、「セキュリティ」パネルに一覧されていないサービスは、サーバのローカルネットワークから来る受信接続のみを受け付けることができます。

AirMac マネージメントを使用していて「セキュリティ」パネルで変更を行った場合は、ベースステーションまたは Time Capsule を再起動してその変更内容を適用するかどうかを確認するメッセージが、「サーバ環境設定」に表示されます。再起動すると、ローカルネットワーク上のすべてのコンピュータのインターネットアクセスおよび DHCP サービスが、最大で 1 分間停止します。

ファイアウォールセキュリティを有効にした場合で、サーバのインターネット接続がネットワークルーターを経由している場合は、受信したすべてのサービス要求をサーバに送信するようルーターを設定する必要があります。操作方法については、「サーバをルーターのデフォルトのホストにする」を参照してください。

ファイアウォールセキュリティで、受信したすべてのサービス要求を許可するには：

「セキュリティ」パネルで、「入」／「切」スイッチをクリックしてファイアウォールセキュリティを無効にします。

ファイアウォールセキュリティが無効になっているとき、サーバのファイアウォールではすべてのサービスへの受信接続が許可されます。ただし、サーバのインターネット接続がケーブル・ルーター、DSL ルーター、またはそのほかのルーターを経由している場合は、受信したサービス要求をサーバに送信するようルーターを設定することも必要です。お使いのルーターの設定については、「お使いになる前に」マニュアルの第 2 章「Mac OS X Server の準備をする」を参照してください。

（ファイアウォールセキュリティではなく）AirMac マネージメントを使用している場合、これを無効にすると、すべてのサービスへの受信接続がブロックされます。
