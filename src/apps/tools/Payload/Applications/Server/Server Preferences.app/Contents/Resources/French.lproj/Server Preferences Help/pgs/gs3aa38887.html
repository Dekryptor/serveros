Configuration des services Web

Dans Préférences du serveur, vous pouvez utiliser la sous-fenêtre Web pour modifier la page d’accueil du site web de votre serveur et pour activer ou désactiver séparément des services web standard.

Pour configurer des services web :

  1. Dans la sous-fenêtre Web de Préférences du serveur, cliquez sur Services web.

  2. Utilisez le menu local pour choisir la page d’accueil du site web de votre serveur.

    • Sélectionnez « Page d’accueil du serveur » pour utiliser le site web par défaut de Mac OS X Server ou un site web HTML personnalisé que vous avez créé (ou que quelqu’un a créé pour vous) à l’aide d’un logiciel de développement de sites web.

    • Choisissez un wiki pour qu’il devienne le site web principal de votre serveur.

      Si le menu local est estompé, le service Wikis est désactivé.

      Si le menu local ne contient pas de wikis, vous devez en créer un ou charger quelqu’un disposant d’un compte utilisateur sur le serveur de le faire. 9.

    Si le bouton Créer le wiki apparaît à la place du menu local, le site utilisé pour les wikis a été supprimé au moyen d’Admin Serveur. Cliquez sur le bouton pour que les wikis deviennent disponibles.

    Si le bouton « Révéler le site personnalisé » apparaît à la place des réglages de page d’accueil et de services web, un site web personnalisé a été créé qui utilise le même nom DNS et le même numéro de port d’accès que des services web standard : wikis, calendriers web, blogs et webmail. En général, les services web utilisent le nom DNS du serveur (affiché dans la sous-fenêtre Informations de Préférences du serveur) et ils utilisent le port 80. Pour rendre les services web standard disponibles, cliquez sur le bouton et supprimez le site web personnalisé.

  3. Sélectionnez les services que vous souhaitez utiliser et désélectionnez les services superflus.

    • Wikis : 
      Permet à tous les utilisateurs du serveur d’employer un navigateur web pour afficher leur portail de wiki Ma page, créer des wikis et utiliser les wikis auxquels ils sont autorisés à accéder. Les propriétaires des wikis définissent des restrictions d’accès dans la section Réglages de leurs wikis.
    • Calendrier : 
      Ajoute un calendrier à chaque portail Ma page des utilisateurs et autorise les créateurs de wikis à inclure un calendrier web dans chaque wiki.
    • Blogs : 
      Permet à tous les utilisateurs du serveur de créer des blogs au moyen d’un navigateur web.
    • Webmail : 
      Permet à tous les utilisateurs du serveur d’accéder à leur courrier électronique au moyen d’un navigateur web.

    Si un service est activé, le fait de cliquer sur la flèche gauche située à côté ouvre le site web correspondant.

  4. Si les services web sont désactivés, cliquez sur l’interrupteur On/Off pour les activer.

    Si les services web sont désactivés, les wikis, les calendriers web, les blogs et le webmail ne sont pas disponibles.

Les utilisateurs peuvent accéder à la page d’accueil de votre serveur à l’adresse http://monserveur.exemple.com (en remplaçant le paramètre fictif en italique par le nom DNS de votre serveur). Ils peuvent alors cliquer sur des liens pour afficher Ma page, les wikis, les blogs, les calendriers et le webmail. Si la page d’accueil de votre serveur est un wiki privé, les visiteurs doivent ouvrir une session en utilisant le nom et le mot de passe d’un utilisateur autorisé à afficher le wiki.
