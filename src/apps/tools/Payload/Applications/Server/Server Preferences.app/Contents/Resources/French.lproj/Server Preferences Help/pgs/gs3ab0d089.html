Utilisation d’un certificat SSL

Un certificat SSL permet au serveur de s’identifier électroniquement et de communiquer en toute sécurité avec les ordinateurs des utilisateurs sur le réseau local et sur Internet. Il permet également de sécuriser davantage le carnet d’adresses, iCal, iChat, le courrier électronique et les services web. Ces services peuvent utiliser le certificat pour chiffrer et déchiffrer en toute sécurité les données qu’ils envoient aux applications et vice-versa sur les ordinateurs des utilisateurs.

Pour utiliser un certificat SSL

  1. Dans la sous-fenêtre Informations de Préférence du serveur, cliquez sur le bouton Modifier situé à droite de Certificat SSL.

  2. Sélectionnez « Utiliser un certificat SSL » et choisissez un certificat disponible dans la partie supérieure du menu local.

    Si le menu local ne contient aucun certificat, vous pouvez créer un certificat auto-signé. Pour obtenir des instructions, consultez Création d’un certificat auto-signé.

    Si vous souhaitez utiliser un certificat SSL préalablement généré, vous pouvez l’importer. Pour obtenir des instructions, consultez Importation d’un certificat.

Vous pouvez utiliser soit le certificat auto-signé créé pour votre serveur lors de sa configuration, soit un certificat auto-signé que vous avez créé. Cependant les applications des utilisateurs ne s’y fient pas automatiquement et affichent des messages demandant si l’utilisateur fait confiance à votre certificat. Utiliser un certificat signé permet aux utilisateurs d’échapper à l’incertitude et la contrariété d’accepter manuellement votre certificat. De plus, une attaque « man-in-the-middle » est possible avec un certificat auto-signé, mais pas avec un certificat signé. Cela signifie que les utilisateurs peuvent se fier aux services auxquels ils accèdent. Consultez Obtention d’un certificat signé et Remplacement d’un certificat auto-signé.
