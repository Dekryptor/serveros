お使いのネットワークに参加する Mac を設定する

Mac OS X v10.6 Snow Leopard を実行する、サーバに未接続の Mac で、ログイン中のユーザがサーバ上で作成されたユーザアカウントと一致することが確認された場合は、その Mac に、サーバとの接続の設定を促す案内メッセージが表示されます。ユーザはその案内を承諾したり、断ったり、次回ログインするまでアクションを保留にしたりすることができます。ユーザが接続できるサーバが複数存在することが Mac で検出された場合は、案内メールによってサーバを選択するよう促されます。

案内メッセージは、Snow Leopard を搭載した Mac がネットワーク上に Snow Leopard Server を実行するサーバを検出した場合にのみ表示されます。Mac 上のユーザのローカルアカウントは、サーバ上のユーザアカウントと同一の名前およびユーザ名を使用している必要があり、サーバでは少なくとも 1 つのサービスが有効になっている必要があります。サーバでユーザのアカウントを追加したときにそのユーザが Mac にログインしていた場合、そのユーザが案内メッセージを確認するには、いったんログアウトしてもう一度ログインする（または Mac を再起動する）必要があります。サーバ上のユーザのアカウントがディレクトリサーバから読み込まれた場合、そのユーザはメール案内を受け取るので、この案内メッセージは表示されません。Snow Leopard ユーザが案内メッセージを確認できない場合、そのユーザは「システム環境設定」の「アカウント」パネルを使って、設定手順を手動で開始することができます。 操作方法については、「アカウント環境設定を使用して Mac を設定する」を参照してください。

ユーザが案内を承諾した場合は、「システム環境設定」の「アカウント」パネルが開きます。サーバのユーザアカウントの名前とパスワードを要求されます。

次に、「アカウント」環境設定でユーザの Mac の設定が行われ、設定内容に関する情報が表示されます。ユーザアカウントの変更、およびユーザがアクセスできるサービスについては、「ユーザの Mac を自動的に設定する」を参照してください。
