Web サービスについて

Web サービスでは、Wiki、Web カレンダー、ブログ、メールへの Web アクセス、およびカスタム Web サイトを提供できます。

Web サービスが有効な場合は、サーバ上のユーザアカウントを持つユーザ全員が、「マイページ」という便利な Wiki ポータルを利用して、Wiki およびブログを閲覧、作成したり、Web カレンダーを使用したり、Wiki のアップデートを追跡したり、Web メールにアクセスしたりできます。

すべてのユーザが、簡単に Wiki を作成し、閲覧者を制御し、自分の Wiki コンテンツを表示、検索することができます。付属のテンプレートを使用したり、独自のテンプレートを作成したりして、マークアップコードや特殊な構文の知識がなくても無理なくコンテンツの追加、削除、編集、および書式設定を実行できます。 2、3 回クリックするか、ドラッグとドロップを行うだけで、ファイルやイメージの添付、Podcast の公開、キーワードの割り当て、ほかの Wiki ページまたは Web サイトへのリンクを実行できます。また、Wiki の変更履歴全体を表示したり、前のページに戻ったりすることもできます。

Web カレンダーでは、Web ブラウザを使って約束、会議、そのほかのイベントを管理できます。ユーザは、イベントの案内を送受信することができます。イベントに誘うときは、相手の都合をタイムラインで確認することができます。各ユーザの「マイページ」ポータルには個人用 Web カレンダーが含まれており、各 Wiki には Wiki へのアクセス権を持つユーザならだれでも使用できるカレンダーを含めることもできます。ユーザは、個人用 Web カレンダー、およびアクセス権のある Wiki のグループカレンダーに、複数のカレンダーを作成することができます。Snow Leopard 環境のユーザは、iCal アプリケーションを使って、Web カレンダーに表示されるイベントを作成、閲覧、および変更することができます。

ブログを使用すると、技術者以外のユーザでも、プロジェクト、自分が作業中のファイル、ピクチャ、Podcast の最新情報をほかの社員に提供できます。ユーザは、一連のプロフェッショナルな内蔵テンプレートを使って、ドラッグ・アンド・ドロップだけで簡単に自分のブログを公開できます。

Web メールを使用すると、ユーザはインターネットに接続している場所ならどこでも Web ブラウザを使用してメールを送受信できます。ユーザのコンピュータ上で Mac OS X の Mail またはほかのメールアプリケーションを使用しているかのように、すべてのメールにアクセスできます。

Web サービスでは、Web サイト開発ソフトウェアで（自分または自分以外の人が）作成したカスタム Web サイトを公開することもできます。各 Web サイト（または Web サイトの一部分）へのアクセスを特定のグループに限定することができます。 各 Web サイトの IP アドレス、アクセスポート、および Web サイトのファイルを保存するサーバ上のフォルダを指定することもできます。 カスタム Web サイトは仮想ホストとも呼ばれます。

インターネット上の Web サービスへのアクセスを許可したい場合で、ケーブル・ルーター、DSL ルーター、またはそのほかのネットワークルーターを持っている場合は、お使いのルーターで Web サービス用にポートフォワード（ポートマッピング）を設定する必要があります。80 以外のアクセスポート番号を使用するカスタム Web サイトを追加する場合は、それらのポートにポートフォワードを設定する必要があります。ポートフォワードの詳細については、「お使いになる前に」マニュアルを参照してください。

ローカルネットワークの外部の Web サービスへのアクセスを許可したい場合で、お使いのローカルネットワークに独立したファイアウォールデバイスがある場合は、ファイアウォールの管理者に問い合わせて、Web サービスで使用するポートのファイアウォールを解放してもらってください。80 以外のアクセスポート番号を使用するカスタム Web サイトを追加する場合は、それらのポートでファイアウォールを解放する必要があります。ポートのリストについては、「お使いになる前に」マニュアルの付録を参照してください。
