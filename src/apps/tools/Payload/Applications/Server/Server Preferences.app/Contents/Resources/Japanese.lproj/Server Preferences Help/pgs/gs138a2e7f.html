サーバへの接続案内がユーザに届かない場合

Mac OS X v10.6 Snow Leopard を実行する Mac がネットワークに参加すると、サーバのサービスを使用するための案内が表示されます。案内が表示されない場合は、いくつかの理由が考えられます。

以下の場合、サーバへの接続案内が自動的には表示されません：

  • ユーザの Mac で、10.6 Snow Leopard よりも古いバージョンの Mac OS X が使用されています。このような Mac の設定については、「ユーザのコンピュータを手動で設定する」を参照してください。

  • ユーザが前回のサーバの接続案内を辞退しました。

  • サーバでユーザのアカウントを作成している間に、そのユーザが自分の Mac にログインしました。ユーザはいったんログアウトして、もう一度ログインする必要があります。

  • ユーザが自分の Mac に通常のアカウントでログインしていません。

  • ユーザのアカウントが、サーバ側とユーザの Mac 側とで一致していません。一致させるには、2 つのアカウントに同一の名前またはユーザ名を付ける必要があります。たとえば、サーバ上のアカウント名とユーザ名がそれぞれ「Gita Kumar」と「gkumar」で、ユーザの Mac 上では「Gita」と「gita」になっている場合、アカウントは一致しません。

    これを修正するには、サーバ上のユーザアカウントの名前を、ユーザの Mac のアカウント名に合わせて変更します。ユーザは、名前を一致させないで手動で接続することもできます。操作方法については、「アカウント環境設定を使用して Mac を設定する」を参照してください。

  • すでにユーザの Mac でディレクトリサービスの接続が確立しています。ユーザはサーバへの接続を手動で追加できます。操作方法については、「アカウント環境設定を使用して Mac を設定する」を参照してください。

  • サーバ上のユーザのアカウントが（サーバで作成されるのではなく）ディレクトリサーバから読み込まれています。ユーザは、お使いのサーバのサービスを利用するためのボタンが含まれた案内メールを受信します。詳細については、「新規に読み込まれたユーザの Mac を設定する」を参照してください。

  • サーバで「サーバ環境設定」のサービスが 1 つも有効になっていません。詳細については、「サービスの状況を確認する」を参照してください。

  • ユーザの Mac、またはサーバのネットワーク接続に問題があります。
