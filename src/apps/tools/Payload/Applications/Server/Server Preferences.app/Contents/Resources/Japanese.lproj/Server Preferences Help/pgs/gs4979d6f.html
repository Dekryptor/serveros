System Administrator（root）アカウントを保護する

設定中に入力した管理者パスワードは、サーバの System Administrator ユーザアカウント（ユーザ名は root）にも使用されます。System Administrator（root）アカウントは、システム内のファイル（サーバ管理者アカウントまたはそのほかのいずれのユーザアカウントでも使用できないシステムファイルを含む）を移動または削除する際にも使用されます。root ユーザ権限がなくてもサーバを管理することはできます。

重要： root ユーザのパスワードの保護は非常に重要であるため、必ずほかのアカウントのパスワードと重複しないようにしてください。サーバの設定が完了したら、root ユーザアカウントのパスワードを変更してください。

root ユーザのパスワードを変更するには、「システム環境設定」の「アカウント」パネルを開き、「ログインオプション」をクリックし、「編集」ボタンをクリックしてから、「ディレクトリユーティリティを開く」をクリックし、次に「編集」＞「ルートパスワードを変更」と選択します。「ログインオプション」に「編集」ボタンが表示されず、「接続」ボタンは表示されている場合は、「接続」ボタンをクリックし、「ディレクトリユーティリティを開く」をクリックしてから、「編集」＞「ルートパスワードを変更」と選択します。

root ユーザのパスワードは、「ターミナル」（「/アプリケーション/ユーティリティ/」にあります）を開いて「sudo passwd root」と入力することによっても変更できます。
