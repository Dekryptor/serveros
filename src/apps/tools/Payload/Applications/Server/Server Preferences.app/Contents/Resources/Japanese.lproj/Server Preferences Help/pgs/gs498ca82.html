サーバをルーターのデフォルトのホストにする

ただし、サーバのインターネット接続が、ケーブル・ルーター、DSL ルーター、または NAT 装置として設定されたそのほかのネットワークルーターを経由している場合は、受信したすべてのサービス要求をサーバに送信するように、お使いのルーターを設定することができます。ルーター設定用ソフトウェアを使って、サーバのプライベート IP アドレスをデフォルトのホスト（DMZ サーバ、DMZ ホストとも呼ばれます）として設定してください。サーバのプライベートアドレスは、「サーバ環境設定」の「情報」パネルに表示されています。お使いのルーターでデフォルトのホストを設定する方法について詳しくは、ルーターのマニュアルを参照してください。

重要： サーバをデフォルトのホストにする場合は、Mac OS X Server の初期設定完了後にサーバのファイアウォールを有効にして、サーバを保護してください。 操作方法については、「セキュリティの設定を変更する」を参照してください。

サーバをルーターのデフォルトのホストにする代わりに、ローカルネットワークを保護しながらもインターネット上の特定のサービスにはユーザがアクセスできるように、ルーターを設定することも可能です。個々のサービス要求をサーバに転送するようにルーターを設定します。このプロセスは、ポートフォワードまたはポートマッピングと呼ばれます。操作方法については、「お使いになる前に」マニュアルの第 2 章「Mac OS X Server の準備をする」を参照してください。
