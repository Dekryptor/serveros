Publication de sites web personnalisés

Outre la configuration de wikis, votre serveur peut également publier des sites web personnalisés constitués de fichiers HTML statiques. Pour créer un site web personnalisé, utilisez le logiciel de développement de votre choix ou faites-le créer par un tiers, puis copiez les fichiers du site web sur votre serveur. Chaque site web personnalisé est également appelé un hôte virtuel.

Pour publier un site web personnalisé :

  1. Dans la sous-fenêtre Web de Préférences du serveur, cliquez sur Sites personnalisés.

  2. Pour ajouter un nouveau site web, cliquez sur le bouton Ajouter (+), puis saisissez le nom DNS complet du site web et, si vous le souhaitez, choisissez le dossier où les fichiers du site web sont stockés sur le serveur.

    Préférences du serveur crée un site web pour le nom DNS que vous avez saisi, s’il n’est pas déjà utilisé et si votre service DNS peut le résoudre en l’adresse IP de votre serveur.

    • Si vous tapez un nom DNS déjà utilisé par les services web standard (wikis, calendrier web, blogs ou webmail) vous pouvez modifier le nom DNS ou préciser un numéro de port d’accès non standard. Le port d’accès standard pour les sites web est 80. Avant d’indiquer un port différent, vérifiez qu’il n’est pas en conflit avec un autre service disponible sur votre serveur. Pour obtenir la liste des ports utilisés par Mac OS X Server, ouvrez Admin Serveur (dans /Applications/Server/) et utilisez le menu Aide pour rechercher « Référence du port UDP et TCP ».

    • Si l’indicateur d’état situé à droite du nom DNS s’affiche en rouge, Préférences du serveur ne peut pas résoudre ce nom DNS en l’adresse IP de votre serveur. Pour y remédier, vous pouvez modifier le nom DNS ou cliquer sur Créer quand même et définir l’adresse IP manuellement.

    Si vous ne choisissez pas de dossier pour les fichiers du site web, Préférences du serveur en crée un pour vous à l’emplacement par défaut : /Bibliothèque/WebServer/Sites/nom_du_domaine (où nom_du_domaine est le nom DNS que vous avez saisi).

  3. Copiez dans ce dossier les fichiers qui doivent être disponibles pour le site web et supprimez les fichiers fictifs dont vous n’avez pas besoin.

    Assurez-vous que la page d’accueil de votre site web porte le nom index.html ou index.php.

    L’emplacement du dossier du site web s’affiche sous le nom du site dans la sous-fenêtre Sites personnalisés de la sous-fenêtre Web.

  4. Vérifiez que la case située en regard du site web est cochée et que les services web sont activés.

    Si la case d’un site web n’est pas cochée, ce site n’est pas disponible.

    Si les services web sont désactivés, aucun site web personnalisé n’est disponible.

Après avoir ajouté des fichiers et des dossiers au dossier du site web, vous pouvez restreindre l’accès à une partie ou à l’intégralité de ce site. Pour obtenir des instructions, consultez Restriction de l’accès à un site web personnalisé.
