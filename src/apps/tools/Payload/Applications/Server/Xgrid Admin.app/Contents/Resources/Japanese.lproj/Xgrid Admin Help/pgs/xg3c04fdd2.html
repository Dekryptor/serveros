Xgrid のコンポーネント

Xgrid の 3 層アーキテクチャによって、複雑なタスクでも簡単に分散できます。ユーザクライアント、グリッドコントローラ、および計算エージェントが連携して、ノードの統合、ジョブの送信、および結果の取得のプロセスが効率的に行われます。

計算グリッドの主要なコンポーネントは、次の機能を実行します：

  • エージェントは、CPU ごとに一度に 1 つのタスクを実行します。そのため、マルチプロセッサのコンピュータでは、同時に複数のタスクを実行できます。

  • コントローラは、タスクをキューに保存し、それらのタスクをエージェントに分散して、タスクの再割り当てを処理します。

  • クライアントは、ジョブを複数のタスクの形で Xgrid コントローラに送信します。（Mac OS X v10.4 以降または Mac OS X Server v10.4 以降が動作するコンピュータであれば、どのようなコンピュータでもクライアントにすることができます。）

原則的には、エージェント、コントローラ、およびクライアントは同一のサーバ上で実行できますが、専用のコントローラノードを設置する方が効率的なことがよくあります。
