Xgrid-Komponenten

Die dreistufige Xgrid-Architektur vereinfacht die Verteilung komplexer Aufgaben. Die Zusammenarbeit der beteiligten Benutzer-Clients, Grid-Controller und Agenten vereinfacht den Prozess, Knoten zusammenzustellen, Aufträge zu übermitteln und Ergebnisse abzurufen.

Die wichtigsten Komponenten eines Computing-Grids haben folgende Funktionen:

  • Ein Agent führt pro CPU jeweils eine Aufgabe aus. Ein Computer mit mehreren Prozessoren kann also mehrere Aufgaben gleichzeitig ausführen.

  • Ein Controller stellt Aufgaben in eine Warteliste, verteilt diese Aufgaben an Agenten und weist Aufgaben neu zu.

  • Ein Client sendet Aufträge in Form mehrerer Aufgaben an den Xgrid-Controller. (Jeder Computer mit Mac OS X 10.4 (oder neuer) oder Mac OS X Server 10.4 (oder neuer) kann als Client fungieren.)

Grundsätzlich können Agent, Controller und Client auf dem gleichen Server ausgeführt werden. Häufig ist es jedoch effizienter, dedizierte Controller-Knoten zu verwenden.
