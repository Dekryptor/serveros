Verteilte Grids

Wenn ein System Rechenleistung zur Verfügung stellen kann, wird ein verteiltes Grid gebildet.

Mithilfe des Xgrid-Agenten kann ein Benutzer eine beliebige IP-Adresse oder einen beliebigen Hostnamen für den Controller angeben. Durch Angabe eines Grids kann ein Benutzer die CPU-Leistung seines Computers diesem Grid zur Verfügung stellen. Es spielt dabei keine Rolle, wo sich der Controller befindet.

Der Administrator des Controllers besitzt keine direkte Kontrolle oder Informationen über das Agentensystem, kann jedoch dessen CPU-Leistung trotzdem nutzen.

Verteilte Grids haben sehr hohe Ausfallraten für Aufträge, sind für den Grid-Administrator jedoch sehr einfach zu verwalten. Bei äußerst umfangreichen Aufträgen haben hohe Ausfallraten bei Aufgaben u. U. keinen wesentlichen Einfluss auf die Leistung des Grids, wenn die Aufgaben schnell einem anderen verfügbaren Agenten zugewiesen werden können.

Die Netzwerkleistung spielt möglicherweise ebenfalls eine Rolle, da Daten über das Internet und nicht über ein lokales Netzwerk an Agenten in einem Grid gesendet werden. Die Kosten für solche verteilten Grids sind sehr gering.
