Choix des réglages de compression

QuickTime Broadcaster offre une gamme complète de codecs audio et vidéo pour la compression et la diffusion en temps réel. Cette rubrique vous permet de choisir les codecs qui vous conviennent le mieux.

Il est généralement préférable de définir les options audio avant les réglages vidéo. Dans QuickTime Broadcaster, vous pouvez choisir parmi tous les compresseurs fournis avec QuickTime. Ceci vous permet de sélectionner le meilleur codec pour le type de contenu diffusé et pour les connexions de vos spectateurs.

Pour les débutants, QuickTime Broadcaster inclut des réglages par défaut (préréglages) optimisés pour les types de contenu et de connexion habituels. Par exemple, le réglage par défaut pour la parole est MPEG-4 audio (AAC) à 8 kHz mono. Bien que le codec MPEG-4 audio soit généralement considéré comme un codec musical, il fonctionne bien pour la parole également.

Vous pouvez aussi créer vos propres réglages dans la sous-fenêtre Audio de QuickTime Broadcaster. (Si le bouton Audio n’apparaît pas, cliquez sur Afficher les détails.) Vous pouvez enregistrer les réglages créés pour les réutiliser et les partager et que d’autres personnes les utilisent avec leurs propres diffusions.

Pour la plupart des types de diffusion, choisissez les réglages vidéo après avoir déterminé la compression audio. La qualité vidéo est généralement moins importante que la qualité audio. En effet, des données audio saccadées ou brouillées sont très difficiles à comprendre et agaceront rapidement votre public.

Les réglages de compression vidéo d’une diffusion dépendent du type d’animation diffusée et de la largeur de bande dont disposent vos destinataires. QuickTime Broadcaster fournit des réglages par défaut (préréglages) pour les situations les plus courantes.

Par exemple, une vidéo prévue pour les clients d’un réseau local utilise le codec H.264 avec une taille d’image de 480 x 360 à 30 images par seconde (IPS), ce qui limite le débit des données à 1372 kilobits par seconde (Kbps). La vidéo pour les clients disposant de connexions commutées utilise le codec H.264 avec une taille d’image de 160 x 120 à 6 images par seconde, ce qui limite le débit des données à 30 Kbps.
