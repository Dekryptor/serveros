Sélection d’une méthode de diffusion

La méthode de diffusion à utiliser dépend de la taille du public attendu :

  • Lorsque vous diffusez un événement en direct vers un seul spectateur, vous pouvez utiliser QuickTime Broadcaster sous Mac OS X ; vous n’avez pas besoin du logiciel de serveur. L’envoi d’une diffusion à un seul destinataire est appelée monodiffusion. La monodiffusion est la méthode de distribution la plus commune pour les diffusions générales sur Internet.

    Le destinataire d’une monodiffusion doit disposer d’un ordinateur sur lequel QuickTime Player, QuickTime Streaming Server (QTSS) ou Darwin Streaming Server est installé. Si le destinataire dispose de logiciel de serveur, le serveur peut renvoyer votre diffusion vers plusieurs spectateurs.

  • Pour un public de plusieurs centaines d’utilisateurs connectés, un seul ordinateur Mac peut utiliser à la fois QuickTime Broadcaster et QTSS. QuickTime Broadcaster envoie le flux vers QTSS qui à son tour livre le flux à un groupe de clients.

    Le nombre de clients pouvant se connecter à la diffusion est fonction de la largeur de bande passante et de la capacité du processeur. Une configuration normale Mac Pro prend en charge plus de 200 connexions DSL ou 400 connexions commutées. Cette configuration convient parfaitement pour les cours en ligne ainsi que pour toute autre situation où le public est peu nombreux.

  • Si votre diffusion est prévue pour un grand nombre de personnes, utilisez deux ordinateurs, l’un avec QuickTime Broadcaster, pour diffuser le flux, et l’autre avec QTSS, pour renvoyer le flux à de multiples spectateurs.

  • Si vous êtes sur un réseau habilité pour la multidiffusion, vous pourrez atteindre un public nombreux en multidiffusant vos données. La multidiffusion est une méthode de diffusion via un réseau local (et non via Internet) qui n’utilise qu’un seul flux, partagé par tous les spectateurs de la diffusion. Chaque spectateur se règle sur la diffusion comme une radio se règle sur une diffusion FM.
