Konfigurieren des QuickTime Broadcaster

In diesem Beispiel wird der QuickTime Broadcaster auf einem Notebook installiert. Schließen Sie eine FireWire-Kamera an das Notebook an, schalten Sie die Kamera ein und überprüfen Sie, ob sie vom QuickTime Broadcaster erkannt wird.

  1. Öffnen Sie den QuickTime Broadcaster und klicken Sie auf "Details einblenden".

  2. Klicken Sie auf "Video" und wählen Sie eine Videoquelle aus dem Einblendmenü "Quelle" aus.

    Wird die Kamera erkannt, wird sie im Menü "Quelle" angezeigt. Auch die Videoeinstellungen sollten aktiviert (also nicht grau angezeigt) sein.

    Wenn die Kamera im Menü "Quelle" nicht angezeigt, beenden Sie den QuickTime Broadcaster und stellen Sie sicher, dass das Eingabegerät angeschlossen und eingeschaltet ist. Öffnen Sie anschließend den QuickTime Broadcaster erneut. Wird die Kamera weiterhin nicht angezeigt, informieren Sie sich auf der QuickTime-Website oder der Website "AppleCare Service & Support" über die Kompatibilität und weitere mögliche Probleme.

    Beim Senden an einen QTSS auf einem separaten Computer (wie in diesem Beispiel) sollte die Übertragungsmethode "Unicast (automatisch mit Ankündigung)" verwendet werden. Diese ist einfach zu konfigurieren.

  3. Klicken Sie im QuickTime Broadcaster auf "Details einblenden". Klicken Sie dann auf "Netzwerk" und wählen Sie "Unicast (automatisch mit Ankündigung)" aus dem Einblendmenü "Übertragung" aus.

  4. Geben Sie im Bereich "Netzwerk" die IP-Adresse oder den Hostnamen des empfangenden Computers, den Namen der Sendedatei, den Benutzernamen und das Kennwort für den Broadcaster-Benutzer sowie die Pufferverzögerung (oder akzeptieren Sie den Standardwert) ein.

    Hinweis:  Die Pufferverzögerung definiert die Anzahl Sekunden, die QuickTime die Sendung vor der Wiedergabe puffert. Bei Breitbandverbindungen füllt QuickTime-Player 6 (oder neuer) den Puffer schneller als in Echtzeit, was eine unverzügliche Wiedergabe ermöglicht.

  5. Klicken Sie auf "Senden" und starten Sie den QTSS (falls dieser noch nicht aktiv ist). Wählen Sie dazu den Computer im Programm "Server-Admin" aus. Klicken Sie auf "QuickTime-Streaming" und dann auf "Dienst starten".

    Zum Feststellen, ob der Live-Datenstrom von einem Client aus sichtbar ist, öffnen Sie QuickTime-Player auf einem Clientcomputer und wählen Sie "Ablage" > "URL in neuem Fenster öffnen".

  6. Geben Sie die im Bereich "Ort" im Fenster des QuickTime Broadcaster angezeigte RTSP-URL (Real-Time Streaming Protocol) ein (z. B. rtsp://myserver.com/mystream.sdp).

    Hinweis:  Bei Eingabe der URL-Adresse muss die Groß-/Kleinschreibung beachtet werden. Außerdem muss die Adresse exakt in der im QuickTime Broadcaster-Fenster angezeigten Schreibweise eingegeben werden.

    Der Live-Datenstrom sollte jetzt in QuickTime-Player wiedergegeben werden.

  7. Testen Sie die Konfiguration.

    Dies sollte unter möglichst praxisnahen Bedingungen erfolgen. Testen Sie in jedem Fall Folgendes:

    • Konfigurieren Sie Ihre Geräte, so wie sie für die Live-Präsentation verwendet werden - nach Möglichkeit am selben Ort.

    • Bitten Sie einen Kollegen, die Rolle des Moderators zu übernehmen bzw. beteiligen Sie den Moderator am Test. Machen Sie eine Kostümprobe daraus.

    • Überprüfen Sie das Videobild und den Ton.

    • Überprüfen Sie, ob Clients im lokalen Netzwerk und im Internet den Live-Datenstrom aufrufen können.

    • Passen Sie die Position von Kamera, Mikrofon und Beleuchtung nach Bedarf an und beheben Sie mögliche weitere Probleme.
