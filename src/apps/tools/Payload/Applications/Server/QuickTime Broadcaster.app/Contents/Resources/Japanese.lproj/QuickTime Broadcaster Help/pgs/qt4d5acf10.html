QuickTime Broadcaster をセットアップする

この例では、「QuickTime Broadcaster」はノート型コンピュータにインストールされています。FireWire カメラをノート型コンピュータに接続し、カメラの電源を入れてから、「QuickTime Broadcaster」が認識するかどうかを確認します。

  1. 「QuickTime Broadcaster」を開いて、「詳細を表示」をクリックします。

  2. 「ビデオ」をクリックし、「入力源」ポップアップメニューからビデオソースを選択します。

    カメラが認識される場合は、「入力源」メニューに表示されます。ビデオ設定も使用可能な状態（薄く表示されていない）になっているはずです。

    「入力源」メニューにカメラが表示されない場合は「QuickTime Broadcaster」を終了し、カメラが接続されていて電源が入っているかどうか確認してから「QuickTime Broadcaster」をもう一度開いてください。それでもカメラが表示されない場合は、QuickTime の Web サイトまたは AppleCare サービス＆サポートの Web サイトで、互換性およびその他の問題に関する情報を調べてください。

    この例のように、別のコンピュータの「QTSS」にブロードキャストするときは、自動ユニキャスト（アナウンス）転送方式をお勧めします。この方式はセットアップが簡単です。

  3. 「QuickTime Broadcaster」の「詳細を表示」をクリックし、「ネットワーク」をクリックしてから、「転送」ポップアップメニューから「自動ユニキャスト（アナウンス）」を選択します。

  4. 「ネットワーク」パネルで、受信側コンピュータの IP アドレスかホスト名、ブロードキャストファイルの名前、Broadcaster ユーザのユーザ名とパスワード、およびバッファ遅延（あるいはデフォルトを使用）を入力します。

    注記： バッファ遅延には、QuickTime がブロードキャストを再生する前に、どのくらいバッファに蓄積するかを秒単位で設定します。ブロードバンド接続の場合は、QuickTime Player 6 以降では実際に再生される前にバッファに蓄積されるので、インスタントオンで視聴することができます。

  5. 「QTSS」がまだ動作していない場合には、「ブロードキャスト」をクリックして「QTSS」を起動します。「サーバ管理」でコンピュータを選択し、「QuickTime ストリーミング」をクリックしてから、「サービスを開始」をクリックしてください。

    クライアントからライブストリームを視聴できるかどうかを確認するために、クライアントコンピュータで「QuickTime Player」を開き、「ファイル」＞「新規 Player で URL を開く」と選択します。

  6. 「QuickTime Broadcaster」ウインドウの「場所」セクションに表示される RTSP（Real-Time Streaming Protocol）の URL を入力します（たとえば、rtsp://myserver.com/mystream.sdp）。

    注記： URL は大文字と小文字を区別する必要があります。「QuickTime Broadcaster」ウインドウの表示と同じように入力してください。

    「QuickTime Player」でライブストリームの再生が開始されるはずです。

  7. セットアップをテストします。

    このテストはできるだけ実際と同じように行ってください。以下の点は必ずテストしてください：

    • ライブプレゼンテーションで使用できるように装置をセットアップします。できれば、実際の場所で行います。

    • プレゼンタの代役を同僚に依頼します。できれば、プレゼンタが参加して本番並みのテストを行うことをお勧めします。

    • ビデオの映像とサウンドを確認します。

    • ローカルネットワーク上およびインターネット上のクライアントがライブストリームを受信できることを確認します。

    • 必要に応じてカメラやマイクの位置および照明を調整し、何か問題があれば解決します。
