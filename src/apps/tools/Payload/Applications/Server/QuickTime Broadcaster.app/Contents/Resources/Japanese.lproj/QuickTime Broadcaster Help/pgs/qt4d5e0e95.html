ブロードキャスト方法を選択する

選択するブロードキャスト方法は、予想される視聴者の規模によって決まります：

  • 1 人の視聴者だけにライブイベントをブロードキャストする場合は、Mac OS X 上の「QuickTime Broadcaster」を使用できるため、サーバソフトウェアは必要ありません。 ブロードキャストを 1 人の受信者に送信することは、ユニキャストと呼ばれます。ユニキャストは、一般的なインターネットでのブロードキャスティングで最もよく行われている配信方法です。

    ユニキャストの受信者のコンピュータには、「QuickTime Player」、「QuickTime Streaming Server（QTSS）」、または「Darwin Streaming Server」がインストールされている必要があります。受信者がサーバソフトウェアを持っている場合、そのサーバはブロードキャストを複数の視聴者に提供できます。

  • 視聴者が接続されている数百人のユーザである場合は、1 台の Mac コンピュータで「QuickTime Broadcaster」と「QTSS」の両方を実行することができます。「QuickTime Broadcaster」がストリームを QTSS に送信し、QTSS がストリームをクライアントのグループに配信します。

    ブロードキャストに接続できるクライアントの数は、使用できる帯域幅とプロセッサの能力によって決まります。代表的な Mac Pro 構成は、200 以上の DSL 接続または 400 以上のダイヤルアップ接続に対応しています。この構成は、視聴者が限定されているオンラインでの授業、レクチャー、その他のイベントに適しています。

  • ブロードキャストの視聴者がこれよりも多い場合は 2 台のコンピュータを使用し、1 台はストリームをブロードキャストする「QuickTime Broadcaster」用、もう 1 台は多数の視聴者にストリームを提供する「QTSS」用とします。

  • マルチキャスト対応のネットワークを使用している場合は、ブロードキャストをマルチキャスティングすることで多数の視聴者に配信できます。マルチキャスティングは、ブロードキャストのすべての視聴者が共有する単一のストリームを使って、（インターネットではなく）ローカルネットワークを介してブロードキャストする方法です。それぞれの視聴者は、ラジオを FM 放送に合わせるのと同じようにブロードキャストに合わせます。
