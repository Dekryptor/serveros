Warum wird ein Kennwort abgefragt?

Wenn Sie mithilfe der Methode "Unicast (automatisch mit Ankündigung)" an einen QTSS oder DSS übertragen, müssen Sie einen Benutzernamen und ein Kennwort bereitstellen, damit ein Benutzer-Account und eine Sendedatei auf dem Streaming-Server erstellt werden können. Weitere Informationen über das Erstellen eines eigenen Benutzer-Accounts finden Sie in der QTSS- oder DSS-Hilfe.

Der Benutzername und das Kennwort, die Sie für das automatische Unicasting benötigen, sind nicht identisch mit den entsprechenden Angaben für das Webadministrationsprogramm des Streaming-Servers.
