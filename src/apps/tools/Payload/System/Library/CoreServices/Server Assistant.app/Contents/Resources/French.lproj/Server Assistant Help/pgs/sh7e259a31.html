Compte d’administrateur

Utilisez la sous-fenêtre Compte d’administrateur pour configurer un compte d’utilisateur doté de privilèges d’administration du serveur.

Réglages du compte d’administrateur
Nom

Nom complet à utiliser pour le compte d’administrateur principal (Administrateur du serveur, par exemple). Ce nom peut comporter jusqu’à 255 caractères (ou 85 caractères japonais). Il peut contenir des espaces. En revanche, il ne peut pas être identique à un nom d’utilisateur prédéfini, tel que « Administrateur système ».

Nom abrégé

Abréviation du nom complet. Généralement, un nom abrégé est constitué de huit caractères maximum, même s’il peut comporter jusqu’à 255 caractères romains. Seules les lettres a à z, les lettres A à Z, les chiffres 0 à 9, le point (.), le trait de soulignement (_) et le trait d’union (-) sont autorisés. Vous ne pouvez pas utiliser le nom abrégé d’un utilisateur prédéfini, tel que « root ». Une fois le compte créé, il n’est plus possible de modifier le nom abrégé.

Mot de passe

Le mot de passe doit être confidentiel et difficile à deviner. Toute personne ayant connaissance d’un nom et d’un mot de passe d’administrateur peut modifier les réglages du serveur et gérer les comptes des utilisateurs.

Indice du mot de passe

Il peut s’agir de tout élément susceptible de vous aider à vous souvenir de votre mot de passe (à l’exception du mot de passe lui-même).
Autoriser les administrateurs à se connecter à distance via SSH

Cette option permet aux administrateurs du serveur d’utiliser le protocole réseau SSH (Secure Shell) pour se connecter au serveur depuis un autre emplacement.
Autoriser les administrateurs à gérer ce serveur à distance

Cette option permet aux administrateurs d’observer et de contrôler le serveur via le réseau en utilisant le moniteur, le clavier et la souris d’un ordinateur Mac avec partage d’écran Mac OS X ou avec le logiciel Apple Remote Desktop (vendu séparément).

Autres comptes d’administrateur

Le mot de passe que vous indiquez dans ce champ est également employé pour le compte Administrateur système, dont le nom abrégé est « root ». Si votre serveur est destiné à avoir ses propres utilisateurs et groupes, ce mot de passe sera également utilisé pour un compte d’administrateur d’annuaire, dont le nom complet est « Administrateur d’annuaire » et le nom abrégé « diradmin ». Vous pourrez indiquer un nom complet et un nom abrégé différents pour l’administrateur d’annuaire si vous choisissez l’option Configurer manuellement dans la sous-fenêtre Utilisateurs et groupes.

Le compte Administrateur système dispose d’un accès total au système, y compris aux fichiers système. Il est donc primordial de protéger le mot de passe de cet utilisateur root afin qu’aucun autre compte ne possède le même mot de passe. Les administrateurs du serveur n’ont pas besoin des privilèges de l’utilisateur root.

Important :  après avoir configuré le serveur, il est recommandé de changer le mot de passe du compte créé ici et du compte de l’administrateur d’annuaire, ou bien de changer le mot de passe du compte Administrateur système (root).

Pour changer le mot de passe ou d’autres attributs du compte d’administrateur créé ici, utilisez la sous-fenêtre Comptes de l’application Préférences Système après vous être connecté au serveur.

Pour changer le mot de passe ou d’autres attributs du compte Administrateur d’annuaire, utilisez l’application « Gestionnaire de groupe de travail » après avoir configuré le serveur.

Pour changer le mot de passe du compte Administrateur système (root), utilisez la rubrique Options de la sous-fenêtre Comptes dans l’application Préférences Système afin d’ouvrir l’application Utilitaire d’annuaire après avoir configuré le serveur.

Sécurisation du serveur

Pour des raisons de sécurité, il est recommandé de créer un compte d’utilisateur standard (non un compte d’administrateur) et de l’utiliser de façon routinière pour vous connecter. Faites appel à un compte d’administrateur dans les applications Préférences du serveur, Admin Serveur, Gestionnaire de groupe de travail ou tout autre outil d’administration lorsque vous avez besoin de gérer des utilisateurs, des groupes ou des services.

Vous pouvez créer des comptes d’utilisateur standard à partir des Préférences du serveur ou du Gestionnaire de groupe de travail.
