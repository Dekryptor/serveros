Welcome

You can use Server Assistant to install and set up Mac OS X Server software on another computer, as described in the following table. You can use Server Assistant from a server that is already set up, or from a Mac OS X computer with Mac OS X Server administration software installed.

After setting up a server, you manage users and groups and customize the settings of individual services using the Server Preferences application or advanced server administration tools, such as Server Admin and Workgroup Manager. For more information about setting up and administering Mac OS X Server, see the Getting Started manual included with Mac OS X Server.

Use Server Assistant to:
Install Mac OS X Server remotely

Install server software on a computer over the network. You can:

  • Install on a disk that doesn’t already have Mac OS X Server or Mac OS X installed

  • Upgrade Mac OS X Server version 10.4.11 or Mac OS X Server v10.5

The remote computer must be started up using the Mac OS X Server Install Disc or equivalent, with the Installer displaying its language picker.

Set up a remote server

Set up a server over the network. The remote server must be:

  • A new server or a computer with Mac OS X Server v10.6 newly installed

  • Started up with Server Assistant displaying its Welcome pane

You can set up multiple servers at the same time. They must have the same hardware configuration and use the same setup data except for Mac OS X Server serial numbers, network connection services, and network names. If you want to set up AirPort management on a server, set up the server separately or turn on AirPort management in Server Preferences after setup.

Create a profile for automatic setup

Go through the setup process, selecting setup options and entering setup data, and then instead of using the setup information to configure a server right now, save the setup information as an auto setup profile on a removable drive or disc. Later you can use the saved auto setup profile to automate the setup of one or more servers.

You can use auto setup profiles to set up multiple servers at once or to set up test servers after reinstalling server software.

You can create a generic auto setup profile for setting up multiple servers, or create an individual auto setup profile for a particular server. When setting up a generic auto setup profile:

  • You’ll specify one administrator account name and password that all servers will use. You can change a server’s administrator password after setup by using the Accounts pane of System Preferences.

  • You can specify a network connection service for each hardware address (also called Ethernet ID or MAC address) or a generic network connection service for an interface name (BSD port name), such as en0 or en1, and have your DHCP server configured to assign each server a fixed (static) IP address according to its hardware address. This feature is called static mapping or DHCP reservations.

  • You can leave the network names blank, and have your DNS service configured to assign each server a name according to its IP address.

You can save the auto setup profile in encrypted form for security.
