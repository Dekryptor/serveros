Directory Services

Use the Directory Services pane to make your server an Open Directory master so it can provide network accounts for users, groups, and computers. Your server provides the Open Directory service in addition to any existing directory server that you connected your server to in the previous pane. If you choose not to make your server an Open Directory master now, you can do it with Server Admin after finishing server setup.

Settings for an Open Directory master
Name

The full name you want used for the directory administrator account. The default name is Directory Administrator.
Short name

This is an abbreviation of the full name. The default short name is diradmin.
UID

The user ID number, which is used for folder and file permissions. The default UID is 1000.
Password

Automatically set to the same password as the server administrator account you set up earlier.
Restrict individual user and group access to services

Selecting this option restricts access to address book, file sharing, iCal calendar, iChat instant messaging, mail, and VPN remote access services. User accounts added in Server Preferences are granted access to all these services. You can change user and group access to services with Server Preferences Server Admin.

If you don’t create an Open Directory master now, and you didn’t connect to a directory server in the previous pane, Server Assistant doesn’t set up any services for you. After Server Assistant finishes, you can use Server Admin or command-line tools to set up the services you want this server to provide.
