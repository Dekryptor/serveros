Configuration d’un service de connexion réseau à l’aide du protocole DHCP avec une adresse manuelle

Si vous souhaitez indiquer une adresse IP statique et laisser le serveur DHCP attribuer d’autres réglages de connexion réseau, choisissez « Utilisation de DHCP avec une adresse manuelle » dans le menu local Configurer IPv4 de la sous-fenêtre Réseau.

Pour configurer un service de connexion réseau à l’aide du protocole DHCP avec une adresse manuelle :

Après avoir sélectionné un service de connexion réseau et choisi « Utilisation de DHCP avec une adresse manuelle » dans le menu local Configurer, entrez les détails de configuration à partir des données suivantes :

  • Adresse IP :  
    Adresse numérique qui identifie de façon unique le serveur sur le réseau. Elle est constituée de quatre nombres séparés par des points (par exemple, 192.0.2.123). Si vous possédez une borne d’accès AirPort ou tout autre routeur Internet partageant une connexion Internet sur votre réseau local, l’adresse IP du serveur doit se trouver sur le même sous-réseau IP que l’adresse IP de la borne ou du routeur en question (par exemple, 10.0.7.2 ou 192.168.99.2).
  • Serveur DNS :  
    Adresse IP d’un serveur DNS (par exemple, 192.0.2.12) ou liste d’adresses IP de plusieurs serveurs DNS. En général, le serveur DHCP fournit automatiquement les adresses du serveur DNS. Un serveur DNS convertit les noms DNS tels que serveur.exemple.com en adresses IP, et inversement.
  • Domaines de recherche :  
    Cette valeur peut être fournie automatiquement par votre service DHCP. Vous pouvez également taper la dernière partie de noms à ajouter automatiquement aux noms DNS partiels saisis lors de l’utilisation du serveur. Par exemple, si vous indiquez campus.univ.edu en tant que domaine de recherche, vous pouvez saisir serveur1 dans la zone de dialogue Connexion au serveur du Finder, afin de vous connecter à serveur1.campus.univ.edu.
