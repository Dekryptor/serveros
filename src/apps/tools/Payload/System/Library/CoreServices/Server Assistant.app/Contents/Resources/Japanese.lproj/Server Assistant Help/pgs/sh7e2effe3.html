ホーム
ユーザとグループ（Xsan）

Xsan プライマリ・メタデータ・コントローラの「ユーザとグループ」パネルで選択するオプションによって、ユーザおよびグループアカウントの管理方法を指定します。ユーザおよびグループアカウントは、SAN コンピュータのユーザの認証とアクセスを許可するサービスの決定に使用します。ユーザおよびグループアカウントを使用して、ユーザとグループにアクセス権を割り当てることによって、SAN 上のファイルとフォルダへのアクセスを制御します。

ユーザとグループを管理するためのオプション
Xsan Admin を使ってユーザとグループを管理

ユーザおよびグループアカウントに対するそれ自体のディレクトリサービスを持つように、プライマリ・メタデータ・コントローラを Open Directory のマスターとして設定します。設定後に、「Xsan Admin」を使用してユーザとグループを作成および管理します。「Xsan Admin」により、Open Directory サービスに接続するように自動的に SAN コンピュータが構成されます。

このオプションは、まだディレクトリサーバがなく、SAN ユーザが 20 人を超えない予定の場合に推奨されます。 これらのユーザは、サーバ上のネットワーク・ホームフォルダではなく、自分のコンピュータ上にローカル・ホームフォルダを持つことになります。

重要： このオプションを選択する場合は、必ず「Xsan Admin」を使用してユーザとグループを作成してください。「サーバ環境設定」は使用しないでください。「Xsan Admin」では新しいユーザアカウントのホームフォルダが作成されますが、「サーバ環境設定」では作成されません。
重要： このオプションを選択できるのは、この設定アシスタントを使ってプライマリ・メタデータ・コントローラ上の Mac OS X Server を設定している間のみです。 その後でこのオプションを構成することはできません。
Open Directory サーバから提供される既存のユーザとグループを使用

プライマリ・メタデータ・コントローラを、次のパネルで指定する DNS 名または IP アドレスを持つ既存の Open Directory サーバに接続できます。「Xsan Admin」により、Open Directory サーバに接続するように自動的に SAN コンピュータが構成されます。ユーザとグループは、Open Directory サーバの管理者によって管理されます。

後でディレクトリサーバに接続

既存の Open Directory サーバへの接続を設定後まで延期したり、設定後に Active Directory などの別のタイプのディレクトリサーバに接続することができます。Snow Leopard で SAN コンピュータ上のディレクトリサービス接続を構成するには、「システム環境設定」の「アカウント」パネルの「ログインオプション」を使用します。Leopard で SAN コンピュータ上のディレクトリサービス接続を構成するには、「ディレクトリユーティリティ」アプリケーションを使用します。

これらのオプションが表示されない場合は、Xsan プライマリ・メタデータ・コントローラの設定は行われていません。「ユーザとグループ」を参照してください。
