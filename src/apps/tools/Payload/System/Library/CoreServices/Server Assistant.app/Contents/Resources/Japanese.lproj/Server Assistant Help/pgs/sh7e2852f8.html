ネットワーク名

「ネットワーク名」パネルでは、ネットワーク上のほかのコンピュータでサーバを識別させる名前を設定します。

ネットワーク名の設定
プライマリ DNS 名

これは、ネットワーク上の DNS サーバから判断できる場合は、自動的に設定されます。これは、DNS サービスでのサーバに対する完全な一意の名前です（たとえば、myserver.example.com）。この名前が実際はサーバに登録されている場合は、変更しないでください。ただし、間違っている場合は、「編集」ボタンをクリックして変更することができます。このパネルで DNS 名を変更する場合は、DNS サーバが新しい名前を使用するように更新されることを確認してください。

ネットワークに DNS サーバがない場合は、myserver.private のような名前を使用します。
コンピュータ名

ネットワークアドレスではなくコンピュータ名で識別されるネットワーク・ファイル・サーバまたはその他のネットワークリソースをブラウズするユーザに対して、サーバを識別します。 63 文字以下のローマ字の名前を指定します。名前に空白を含めることはできますが、「=」、「:」、または「@」は使用しないでください。（ユーザのコンピュータが SMB ファイル共有プロトコルを使用してサーバに接続している場合、Mac OS X Server によって、コンピュータ名が SMB で有効なフォームに自動的に変換されます。）

ローカルホスト名

コンピュータは、「.local」で終わるローカルホスト名も持っています（myserver.local など）。サーバのローカルサブネット上のコンピュータは、グローバル DNS システムを使用して名前を IP アドレスに解決することなく、そのプライマリ DNS 名ではなくサーバのローカルホスト名を使用して、サーバにアクセスすることができます。コンピュータのローカルホスト名は、そのローカルネットワーク（IP サブネット）上でのみ使用できます。

ローカルホスト名は、デフォルトではコンピュータ名から付けられます。変更したい場合は、設定の完了後に「サーバ管理」または「システム環境設定」の「共有」パネルを使用します。

自動設定プロファイルのネットワーク名

後でサーバを自動的に設定するためにプロファイルを作成し、それを使って複数のサーバを設定したい場合は、「プライマリ DNS 名」と「コンピュータ名」を空白のままにすることができます。プロファイルを使用してサーバを設定する場合は、サーバの IP アドレスを使用して DNS 名が検索され、この DNS 名からコンピュータ名が付けられます。
