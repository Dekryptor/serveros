Finding an Interface Name or Hardware Address

If you’re creating an auto server setup profile and want to add a network connection service in the Network pane but don’t know an interface name (BSD port) or hardware address (MAC address) for a server that’s connected to your network, you can use command-line tools to get the information. You need to know the remote server’s IP address or DNS name and its hardware serial number. Use ssh to log in to the remote server from the command line, and then use networksetup to get network port information.

To get network interface information for a remote server:

  1. Make sure the remote server has started up and is connected to the network.

    If Mac OS X Server isn’t installed yet, you can start up the remote server using the Mac OS X Server Install Disc.

  2. Open Terminal on your Mac, and then log in to the remote server by typing this command and pressing Return:

    ssh root@serveraddress

    Replace serveraddress with the IP address or DNS name of the remote server.

  3. Enter yes if you’re asked if you’re sure you want to continue connecting even though the authenticity of the remote server can’t be established.

    You’re asked this the first time you log in to a remote server.

  4. When prompted for the password, enter the first eight characters of the remote server’s serial number.

  5. Type this command and press Return:

    networksetup -listallhardwareports

    The output of this command identifies an interface name as “Device” and a hardware address as “Ethernet Address.”
