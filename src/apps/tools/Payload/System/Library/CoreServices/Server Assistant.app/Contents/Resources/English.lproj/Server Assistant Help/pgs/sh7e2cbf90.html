Configuring an LOM Port Network Connection Service

Xserve systems with Intel processors include a separate lights-out management (LOM) processor that lets you communicate with the server even when it is not powered on. The LOM processor has its own network settings for the server’s built-in Ethernet ports.

To configure an LOM Port network connection service:

  1. Select an LOM Port service in the list on the left.

  2. If requested, enter the name and password for the LOM administrator.

    The LOM administrator account is not the same as the Mac OS X Server administrator account, but you can use the same name and password if you want to.

  3. Click Configure, choose a configuration method, and enter the configuration details using the information below.

    • IP Address:  
      A numeric address that uniquely identifies the server on the network. It’s written as four numbers separated by periods (for example, 192.0.2.123).

      Give the LOM interface its own IP address. Don’t use the same address that you assigned to built-in Ethernet 1 or Ethernet 2, even though it uses the same physical connector (port 1 or 2) on the back of the Xserve.

    • Subnet Mask:  
      A group of four numbers that identifies the network part of the server’s IP address; the remainder of the IP address uniquely identifies the server on that network
    • Router:  
      The IP address of the router for the server’s local network (for example, 192.168.99.1 or 10.0.7.1). The router is a device that forwards data packets from the local network toward their destinations on other networks or the Internet. If you don’t have a router in your network, enter the server’s IP address instead.

You can change the LOM administrator name or password by clicking Admin Account.

After setting up the server, you can change the LOM Port network settings and add or change LOM administrator accounts using Server Monitor. For instructions, open Server Monitor (in /Applications/Server/) and then use the Help menu. You can also change these settings from the command line using the IPMITool in Terminal. For information, see the IPMITool man topic.
