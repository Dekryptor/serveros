Netzwerknamen

Geben Sie im Fenster "Netzwerknamen" die Namen ein, die Ihren Server gegenüber anderen Computern im Netzwerk identifizieren.

Einstellungen für Netzwerknamen
Primärer DNS-Name

Dieser Wert wird automatisch festgelegt, wenn er vom DNS-Server in Ihrem Netzwerk abgerufen werden kann. Es handelt sich um den vollständigen, eindeutigen Namen für den Server beim DNS-Dienst, z. B. "myserver.example.com". Sie sollten diesen Namen nicht ändern, wenn er so bei Ihrem Server registriert ist. Ist er aber falsch, können Sie auf die Taste "Bearbeiten" klicken, um ihn zu ändern. Wenn Sie den DNS-Namen in diesem Fenster ändern, stellen Sie sicher, dass Ihr DNS-Server zur Verwendung des neuen Namens aktualisiert wird.

Wenn in Ihrem Netzwerk kein DNS-Server vorhanden ist, verwenden Sie einen Namen wie "myserver.private".
Computername

Identifiziert den Server gegenüber Benutzern, die nach Netzwerk-Dateiservern oder anderen Netzwerkressourcen suchen, für die der Computername statt der Netzwerkadresse angegeben wurde. Geben Sie einen Namen mit maximal 63 lateinische Zeichen an. Der Name kann Leerzeichen enthalten, vermeiden Sie jedoch die Zeichen "=", ":" oder "@". (Wenn der Computer eines Benutzers mit dem Netzwerkprotokoll SMB die Verbindung zum Server herstellt, konvertiert Mac OS X Server den Computernamen automatisch in ein für SMB gültiges Format.)

Lokaler Hostname

Der Computer hat auch einen lokalen Hostnamen, der auf ".local" endet (wie myserver.local). Computer im lokalen Teilnetzwerk des Servers können den lokalen Hostnamen des Servers statt seines primären DNS-Namens verwenden, um den Server ohne Verwendung eines globalen DNS-Systems zum Auflösen von Namen in IP-Adressen zu erreichen. Der lokale Hostname eines Computers kann nur in seinem lokalen Netzwerk (IP-Teilnetzwerk) verwendet werden.

Der lokale Hostname wird standardmäßig vom Computernamen abgeleitet. Wenn Sie ihn ändern wollen, verwenden Sie nach Beendigung der Konfiguration das Programm "Server-Admin" oder die Systemeinstellung "Freigaben".

Netzwerknamen für ein Profil für die automatische Konfiguration

Wenn Sie ein Profil für die spätere automatische Serverkonfiguration erstellen und es zum Konfigurieren mehrerer Server verwenden wollen, können Sie die Felder "Primärer DNS-Name" und "Computername" frei lassen. Wenn Sie das Profil zum Konfigurieren eines Servers verwenden, sucht der Serverassistent den DNS-Namen anhand der IP-Adresse des Servers und leitet den Computernamen vom DNS-Namen ab.
