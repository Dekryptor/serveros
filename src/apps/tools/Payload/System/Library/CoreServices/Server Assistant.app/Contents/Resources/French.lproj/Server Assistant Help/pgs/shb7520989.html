Recherche d’un nom d’interface ou d’une adresse matérielle

Lorsque vous créez un profil de configuration serveur automatique et souhaitez ajouter un service de connexion réseau dans la sous-fenêtre Réseau mais que vous ne disposez ni du nom d’interface (port BSD) ni de l’adresse matérielle (adresse MAC) correspondant à un serveur connecté à votre réseau, les outils de ligne de commande vous permettent d’obtenir ces informations. Vous devez connaître l’adresse IP ou le nom DNS du serveur distant ainsi que son numéro de série matériel. Utilisez la commande ssh pour vous connecter au serveur distant depuis la ligne de commande, puis la commande networksetup pour obtenir des informations sur le port réseau.

Pour obtenir des informations sur l’interface réseau d’un serveur distant :

  1. Vérifiez que le serveur distant a démarré et qu’il est connecté au réseau.

    Si Mac OS X Server n’est pas encore installé, vous pouvez démarrer le serveur distant à l’aide du disque d’installation Mac OS X Server.

  2. Ouvrez le Terminal sur votre Mac, puis connectez-vous au serveur distant en saisissant la commande suivante et en appuyant ensuite sur la touche Retour :

    ssh root@serveraddress

    Remplacez serveraddress par l’adresse IP ou le nom DNS du serveur distant.

  3. Saisissez oui si l’on vous demande si vous êtes sûr de vouloir poursuivre la connexion même si l’authenticité du serveur distant ne peut être établie.

    Cette question vous est posée lorsque vous vous connectez à un serveur distant pour la première fois.

  4. Lorsque vous êtes invité à fournir un mot de passe, saisissez les huit premiers caractères du numéro de série du serveur distant.

  5. Saisissez la commande suivante, puis appuyez sur la touche Retour :

    networksetup -listallhardwareports

    La sortie de cette commande identifie le nom d’une interface comme « périphérique » et une adresse matérielle comme « adresse Ethernet ».
