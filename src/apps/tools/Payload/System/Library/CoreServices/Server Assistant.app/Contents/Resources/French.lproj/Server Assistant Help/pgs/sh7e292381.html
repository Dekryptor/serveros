Configuration d’un service de connexion réseau à l’aide du protocole BootP

Si vous souhaitez qu’un serveur BootP attribue une adresse IP et d’autres réglages de connexion réseau, choisissez Utilisation de BootP dans le menu local Configurer IPv4. Avec BootP, la même adresse IP est toujours affectée à une interface réseau spécifique. Elle est principalement utilisée pour les ordinateurs qui démarrent à partir d’une image NetBoot.

Si vous configurez un serveur avec Xsan, vous ne pouvez pas choisir l’option Via BootP car cette technologie n’est pas adaptée à un réseau Xsan.

Pour configurer un service de connexion réseau à l’aide du protocole BootP :

Après avoir sélectionné un service de connexion réseau et choisi l’option Via BootP dans le menu local Configurer, indiquez les détails de configuration BootP à partir des données suivantes :

  • Serveur DNS :  
    Adresse IP d’un serveur DNS (par exemple, 192.0.2.12) ou liste d’adresses IP de plusieurs serveurs DNS. Un serveur DNS convertit les noms DNS tels que serveur.exemple.com en adresses IP, et inversement.
  • Domaines de recherche :  
    Fournit des noms à ajouter automatiquement aux noms DNS non complets. Par exemple, si vous indiquez campus.univ.edu en tant que domaine de recherche, vous pouvez saisir serveur1 dans la zone de dialogue Connexion au serveur du Finder, afin de vous connecter à serveur1.campus.univ.edu.
