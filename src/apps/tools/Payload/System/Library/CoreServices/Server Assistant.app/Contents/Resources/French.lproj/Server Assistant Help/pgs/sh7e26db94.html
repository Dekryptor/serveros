Réseau

Utilisez la sous-fenêtre Réseau pour définir les réglages de connexion spécifiques aux interfaces réseau de votre serveur. Le groupe de réglages de chaque interface forme un service de connexion réseau. Une interface réseau est également qualifiée de port, il s’agit en fait de la partie de l’interface sur laquelle est branché un câble ou une prise.

Votre serveur a besoin d’un service de connexion pour le réseau local auquel se connectent les ordinateurs des utilisateurs. Il s’agit généralement d’un réseau Ethernet, mais un réseau AirPort est également possible. Si votre fournisseur d’accès à Internet exige une connexion via PPPoE, vous devez configurer une connexion de service réseau. Si votre serveur se connecte à plusieurs réseaux, vous devez configurer un service de connexion réseau pour chacun d’eux.

Pour configurer un service de connexion réseau :

  • Sélectionnez un service de connexion réseau dans la liste, puis définissez les réglages appropriés. Pour en savoir plus sur les réglages adaptés aux différents types de services de connexion réseau, consultez les rubriques suivantes :

    Configuration d’un service de connexion réseau Ethernet
    Configuration d’un service de connexion réseau AirPort
    Configuration d’un service de connexion réseau FireWire
    Configuration d’un service de connexion réseau PPPoE
    Configuration d’un service de connexion réseau de port LOM
    Désactivation d’un service de connexion réseau

  • Pour ajouter un service de connexion réseau, cliquez sur le bouton Ajouter (+). Consultez la rubrique Ajout d’un service de connexion réseau pour obtenir des instructions.

  • Pour supprimer un service de connexion réseau, sélectionnez-le et cliquez sur le bouton Supprimer (–).

  • S’il est disponible, le bouton Action (image d’un engrenage) vous permet d’afficher uniquement vos services de connexion réseau favoris ou bien d’afficher l’ensemble de ces services.

  • Pour connaître l’adresse matérielle et le nom d’interface d’un service de connexion réseau, il vous suffit de passer le pointeur de la souris dessus dans la liste.

  • Pour modifier l’ordre de priorité des services de connexion réseau, déplacez-les dans la liste. Lorsque vous tentez de communiquer via un réseau, le serveur fait appel aux services de connexion dans l’ordre, en fonction de cette liste.

    Le premier service de connexion réseau actif constitue la connexion réseau principale du serveur. Il est utilisé par le serveur pour toutes les communications extérieures au réseau local.

Initialement, la sous-fenêtre Réseau mentionne un service de connexion réseau pour chaque interface Ethernet du serveur. Les autres interfaces réseau disponibles ne sont pas répertoriées car le protocole Ethernet est préconisé pour les connexions réseau du serveur. Pour afficher toutes les interfaces réseau, utilisez le bouton Action (image d’un engrenage). Si le serveur ne comporte aucune interface Ethernet, la sous-fenêtre Réseau dresse la liste de toutes les interfaces réseau disponibles. Par ailleurs, la sous-fenêtre Réseau ne mentionne aucun service si vous créez un profil en vue d’une configuration automatique ultérieure du serveur.

La plupart des ordinateurs Macintosh sont équipés d’un port Ethernet intégré. D’autres interfaces peuvent y être intégrées ou ajoutées, telles qu’une interface AirPort pour une connexion réseau sans fil, un port FireWire compatible avec une connexion réseau ou un second port Ethernet.

Les systèmes Xserve à processeur Intel disposent également d’une interface spécifique au processeur LOM (Lights-Out Management). Celle-ci possède sa propre adresse IP et vous permet de vous connecter au serveur par le biais de l’application Contrôle de serveur, même lorsque le serveur est éteint.

Services de connexion réseau pour une passerelle Internet

Si la sous-fenêtre Passerelle Internet s’est affichée avant la sous-fenêtre Réseau et que vous avez choisi de définir le serveur en tant que passerelle Internet, vous ne pouvez pas modifier les réglages de la connexion Ethernet vers le réseau local. L’Assistant du serveur configure automatiquement ces réglages.

Services de connexion réseau pour plusieurs serveurs

Si vous configurez plusieurs serveurs à la fois, l’Assistant du serveur affiche la sous-fenêtre Réseau pour chaque serveur successif. Le texte affiché dans la partie supérieure de la sous-fenêtre vous indique le nom DNS et l’adresse IP du serveur en cours de configuration.

Services de connexion réseau pour un contrôleur de métadonnées Xsan

Si la sous-fenêtre Xsan s’est affichée auparavant et que vous avez choisi de définir le serveur en tant que contrôleur de métadonnées Xsan :

  • Assurez-vous que le service de connexion associé au réseau de métadonnées privé est inactif si vous prévoyez que ce dernier soit configuré automatiquement lors de la configuration de Xsan. Consultez la rubrique Désactivation d’un service de connexion réseau pour obtenir des instructions.

  • Assurez-vous que tous les autres services de connexion réseau aient chacun une adresse IP fixe (statique). Par exemple, définissez l’option Configurer IPv4 sur « Manuellement » ou sur « Utilisation de DHCP avec une adresse manuelle ».

    Les options de configuration réseau BootP, PPPoE et DHCP ne sont pas disponibles car elles ne sont pas adaptées à un réseau Xsan.

Services de connexion réseau pour un profil de configuration automatique

Si vous créez un profil en vue de la configuration automatique ultérieure du serveur, les services de connexion réseau que vous configurez dans cette sous-fenêtre seront utilisés de la façon suivante : lorsque vous configurez automatiquement un serveur, l’Assistant du serveur tente de faire correspondre un service de connexion réseau du profil enregistré avec les interfaces réseau du serveur, en comparant leurs adresses matérielles (également appelées adresses MAC ou identifiants Ethernet). Si l’opération échoue, il tente de faire correspondre le nom d’interface (ou nom de port BSD) d’un service de connexion réseau enregistré, tel que en0, en1 ou fw0, avec les interfaces réseau du serveur. Si l’opération échoue à nouveau, le service de connexion réseau enregistré n’est pas utilisé.

Vous pouvez configurer des services de connexion réseau génériques correspondant aux interfaces réseau sur un grand nombre de serveurs. Configurez deux services Ethernet et un service AirPort, puis définissez l’option Configurer IPv4 de chacun d’eux sur Via DHCP. Avant de lancer la configuration automatique d’un serveur, veillez à ce que votre service DHCP soit configuré de façon à attribuer une adresse IP fixe (statique) à chaque interface réseau de chaque serveur configuré automatiquement. Si un serveur donné ne possède pas deux interfaces Ethernet ou une interface AirPort, l’Assistant du serveur ne fait pas appel au service de connexion réseau enregistré correspondant.
