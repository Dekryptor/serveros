Netzwerk

Geben Sie im Fenster "Netzwerk" Verbindungseinstellungen für die Netzwerkschnittstellen Ihres Servers an. Die Einstellungsgruppe für jede Schnittstelle stellt einen Netzwerkverbindungsdienst dar. Eine Netzwerkschnittstelle wird auch Anschluss oder Port genannt. Dabei handelt es sich um den physischen Anschluss, über den ein Stecker oder ein Kabel anschlossen wird.

Ihr Server benötigt einen Netzwerkverbindungsdienst für das lokale Netzwerk, zu dem die Computer der Benutzer eine Verbindung herstellen. Dies ist gewöhnlich ein Ethernetnetzwerk, kann aber auch ein AirPort-Netzwerk sein. Wenn Ihr Internetanbieter eine Verbindung über PPPoE erfordert, müssen Sie einen PPPoE-Netzwerkdienstverbindung konfigurieren. Wenn Ihr Server eine Verbindung zu mehreren Netzwerken herstellt, müssen Sie einen Netzwerkverbindungsdienst für jedes Netzwerk konfigurieren, zu dem Ihr Server eine Verbindung herstellt.

Gehen Sie wie folgt vor, um einen Netzwerkverbindungsdienst zu konfigurieren:

  • Wählen Sie einen Netzwerkverbindungsdienst in der Liste aus und geben Sie dann Einstellungen an. Weitere Informationen zu den Einstellungen für die verschiedenen Arten von Netzwerkverbindungsdiensten finden Sie in den folgenden Themen:

    Konfigurieren eines Ethernet-Netzwerkverbindungsdiensts
    Konfigurieren eines AirPort-Netzwerkverbindungsdiensts
    Konfigurieren eines FireWire-Netzwerkverbindungsdiensts
    Konfigurieren eines PPPoE-Netzwerkverbindungsdiensts
    Konfigurieren eines LOM Port-Netzwerkverbindungsdiensts
    Deaktivieren eines Netzwerkverbindungsdiensts

  • Fügen Sie einen Netzwerkverbindungsdienst durch Klicken auf die Taste "Hinzufügen" (+) hinzu. Entsprechende Anweisungen finden Sie unter Hinzufügen eines Netzwerkverbindungsdiensts.

  • Löschen Sie einen Netzwerkverbindungsdienst, indem Sie ihn auswählen und auf die Taste "Löschen" (-) klicken.

  • Zeigen Sie nur bevorzugte oder alle Netzwerkverbindungsdienste an. Verwenden Sie hierfür die Aktionstaste (Zahnrad), falls sie vorhanden ist.

  • Zeigen Sie die Hardwareadresse und den Schnittstellennamen für einen Netzwerkverbindungsdienst an, indem Sie den Zeiger auf einen Netzwerkverbindungsdienst in der Liste bewegen.

  • Ändern Sie die Priorität von Netzwerkverbindungsdiensten, indem Sie sie in der Liste bewegen. Beim Versuch, in einem Netzwerk zu kommunizieren, verwendet der Server die Netzwerkverbindungsdienste in der aufgeführten Reihenfolge.

    Der erste aktive Netzwerkverbindungsdienst ist die primäre Netzwerkverbindung des Servers. Der Server verwendet diesen Dienst für die gesamte Kommunikation außerhalb des lokalen Netzwerks.

Im Fenster "Netzwerk" wird anfangs ein Netzwerkverbindungsdienst für jede der Ethernetschnittstellen des Servers aufgeführt. Weitere verfügbare Netzwerkschnittstellen werden nicht aufgeführt, weil Ethernet für Servernetzwerkverbindungen bevorzugt wird. Möchten Sie alle Netzwerkschnittstellen anzeigen, verwenden Sie die Aktionstaste (Zahnrad). Wenn der Server keine Ethernetschnittstellen besitzt, werden im Fenster "Netzwerk" alle verfügbaren Netzwerkschnittstellen aufgeführt. Im Fenster "Netzwerk" werden anfangs keine Dienste aufgeführt, wenn Sie ein Profil für die spätere automatische Serverkonfiguration erstellen.

Die meisten Macintosh-Computer haben einen integrierten Ethernetanschluss. Sie können zusätzliche integrierte oder nachgerüstete Schnittstellen, etwa eine AirPort-Schnittstelle für eine drahtlose Netzwerkverbindung, einen FireWire-Anschluss, der für eine Netzwerkverbindung verwendet werden kann, oder einen zweiten Ethernetanschluss, aufweisen.

Intel-basierte Xserve-Systeme haben auch eine Schnittstelle für LOM (Lights-Out Management). Diese Schnittstelle hat eine eigene IP-Adresse und ermöglicht die Verbindung zum Server mithilfe des Programms "Servermonitor", auch wenn der Server heruntergefahren wurde.

Netzwerkverbindungsdienste für ein Internet-Gateway

Wenn der Fenster "Internet-Gateway" vor dem Fenster "Netzwerk" angezeigt wurde und Sie gewählt haben, dass dieser Server als Internet-Gateway verwendet wird, können Sie keine Einstellungen für die Ethernetverbindung zu Ihrem lokalen Netzwerk ändern. Der Serverassistent konfiguriert diese Einstellungen automatisch.

Netzwerkverbindungsdienste für mehrere Server

Wenn Sie mehrere Server gleichzeitig konfigurieren, zeigt der Serverassistent das Fenster "Netzwerk" für jeden Server nacheinander an. Der Text oben im Fenster informiert Sie über den DNS-Namen und die IP-Adresse des Servers, den Sie derzeit konfigurieren.

Netzwerkverbindungsdienste für einen Xsan-Metadaten-Controller

Gehen Sie wie folgt vor, wenn das Fenster "Xsan" zuvor bereits angezeigt wurde und Sie gewählt haben, dass Sie den Server als Xsan-Metadaten-Controller verwenden wollen:

  • Stellen Sie sicher, dass der Netzwerkverbindungsdienst für das private Metadaten-Netzwerk inaktiv ist, wenn Sie das private Metadaten-Netzwerk automatisch bei der Xsan-Konfiguration konfigurieren lassen wollen. Entsprechende Anweisungen finden Sie unter Deaktivieren eines Netzwerkverbindungsdiensts.

  • Stellen Sie sicher, dass alle anderen Netzwerkverbindungsdienste jeweils eine feste (statische) IP-Adresse haben. Legen Sie zum Beispiel die Option "IPv4 konfigurieren" auf "Manuell" oder "DHCP mit manueller Adresse" fest.

    Die Netzwerkkonfigurationsoptionen für BootP, PPPoE und DHCP sind nicht verfügbar, weil sie für ein Xsan-Netzwerk nicht zutreffen.

Netzwerkverbindungsdienste für ein Profil für die automatische Konfiguration

Wenn Sie ein Profil für die spätere automatische Serverkonfiguration erstellen, werden die in diesem Fenster konfigurierten Netzwerkverbindungsdienste wie folgt verwendet: Beim automatischen Konfigurieren eines Servers versucht der Serverassistent, einen Netzwerkverbindungsdienst im gesicherten Profil den Netzwerkschnittstellen des Servers zuzuordnen, indem die Hardwareadressen (auch MAC-Adressen oder Ethernet-IDs genannt) verglichen werden. Wenn das fehlschlägt, wird versucht, den Schnittstellennamen (auch BSD-Portname genannt) eines gesicherten Netzwerkverbindungsdiensts, etwa en0, en1 oder fw0, den Netzwerkschnittstellen des Servers zuzuordnen. Wenn das fehlschlägt, wird der gesicherte Netzwerkverbindungsdienst nicht verwendet.

Sie können allgemeine Netzwerkverbindungsdienste konfigurieren, denen die Netzwerkschnittstellen auf einer Vielzahl von Servern zugeordnet werden. Konfigurieren Sie zwei Ethernetdienste und einen AirPort-Dienst und legen Sie jeweils die Option "IPv4 konfigurieren" auf "DHCP" fest. Stellen Sie vor dem Starten der automatischen Serverkonfiguration sicher, dass Ihr DHCP-Dienst dafür konfiguriert ist, jeder Netzwerkschnittstelle eines jeden Servers, der automatisch konfiguriert wird, eine feste (statische) IP-Adresse zuzuweisen. Wenn ein bestimmter Server nicht über zwei Ethernetschnittstellen oder eine AirPort-Schnittstelle verfügt, verwendet der Serverassistent den entsprechenden gesicherten Netzwerkverbindungsdienst nicht.
