DHCP を使用してネットワーク接続サービスを構成する

DHCP サーバですべてのネットワーク接続設定（サーバの IP アドレスを含む）を割り当てたい場合は、「ネットワーク」パネルの「IPv4 の構成」ポップアップメニューから「DHCP サーバを参照」を選択します。

注記： DHCP サーバは、お使いのサーバに静的 IP アドレス（常に同じ IP アドレス）を割り当てるように構成されている必要があり、DHCP サービスが実行中である必要があります。

Xsan を使用するサーバを設定する場合、Xsan ネットワークには適していないため、「DHCP サーバを参照」を選択することはできません。

DHCP を使用してネットワーク接続サービスを構成するには：

ネットワーク接続サービスを選択し、「構成」ポップアップメニューから「DHCP サーバを参照」を選択した後、以下の情報を参考にして DHCP 構成の詳細を入力します：

  • DNS サーバ： 
    DNS サーバの IP アドレス（たとえば、192.0.2.12）、または DNS サーバの IP アドレスのリスト。通常は、DHCP サーバによって DNS サーバに自動的にアドレスが提供されます。DNS サーバは、server.example.com のような DNS 名を IP アドレスに変換したり、その逆を行います。
  • 検索ドメイン： 
    DHCP サービスによって自動的に提供される場合があります。1 つまたは複数の名前の最後の部分で、サーバを使用しているときに入力する部分的な DNS 名に自動的に付加されるようにしたい部分を入力することもできます。たとえば、検索ドメインとして campus.univ.edu を指定する場合は、「Finder」の「サーバへ接続」ダイアログで「server1」と入力して server1.campus.univ.edu に接続することができます。
  • DHCP クライアント ID： 
    ISP またはネットワークの DHCP サーバ管理者からクライアント ID コードを受け取った場合は、それを入力します。クライアント ID はサーバを識別します（サーバに複数のネットワークポートがある場合は、ネットワークポートを識別します）。「サーバアシスタント」でリモートサーバを設定するときは、DHCP クライアント ID を指定しないでください。代わりに、設定後に、サーバ上の「システム環境設定」の「ネットワーク」パネルで DHCP クライアント ID を入力してください。
