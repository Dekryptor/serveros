ネットワーク接続サービスを手動で構成する

ネットワークに DHCP サーバがない場合、またはほかの何らかの理由ですべてのネットワーク接続設定を手入力で指定する必要がある場合は、「IPv4 の構成」ポップアップメニューから「手入力」を選択します。

ネットワーク接続サービスを手動で構成するには：

ネットワーク接続サービスを選択し、「構成」ポップアップメニューから「手入力」を選択した後、以下の情報を参考にして構成の詳細を入力します：

  • IP アドレス： 
    ネットワーク上でサーバを一意に識別する数字アドレス。ピリオドで区切られた 4 つの数字として表されます（たとえば、192.0.2.123）。ローカルネットワーク上でインターネット接続を共有している AirMac ベースステーションまたはその他のインターネットルーターがある場合、サーバの IP アドレスは、ベースステーションまたはルーターの IP アドレス(10.0.7.2 や 192.168.99.2 など）と同じ IP サブネット上にある必要があります。
  • サブネットマスク： 
    サーバの IP アドレスのネットワーク部分を識別する、4 つの番号のグループ。IP アドレスの残りの部分は、ネットワーク上のサーバを一意に識別します。サーバにプライベート IP アドレスがある場合は、サブネットマスクとして 255.255.255.0 を使用します。サーバにパブリック IP アドレス（192.168、10.0 ～ 10.255、または 172.16 ～ 172.31 で始まらないアドレス）がある場合は、サーバの IP アドレスと同じソースからサブネットマスクを取得します。
  • ルーター： 
    サーバのローカルネットワークのルーターの IP アドレス（たとえば、192.168.99.1 または 10.0.7.1）。AirMac ベースステーションまたはほかのインターネットルーターがある場合は、そのアドレスを使用します。それ以外の場合は、サーバの IP アドレスと同じソースからルーターアドレスを取得します。ネットワークにルーターがない場合は、サーバの IP アドレスをルーターアドレスとして使用します。ルーターは、データパケットを、ローカルネットワークからほかのネットワークまたはインターネット上の宛先に転送する装置です。
  • DNS サーバ： 
    DNS サーバの IP アドレス（たとえば、192.0.2.12）、または DNS サーバの IP アドレスのリスト。DNS サーバは、server.example.com のような DNS 名を IP アドレスに変換したり、その逆を行います。
  • 検索ドメイン： 
    完全修飾されていない DNS 名に自動的に付加する名前を指定します。たとえば、検索ドメインとして campus.univ.edu を指定する場合は、「Finder」の「サーバへ接続」ダイアログで「server1」と入力して server1.campus.univ.edu に接続することができます。
