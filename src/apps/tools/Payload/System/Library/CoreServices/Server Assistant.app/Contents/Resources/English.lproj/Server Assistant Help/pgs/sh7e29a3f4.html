Mail Options

Use the Mail Options pane to specify a mail relay server and an introduction to the welcome message sent to new users. The Mail Options pane appears if you selected mail service in the Services pane earlier.

Mail options
Relay outgoing mail through __

Some ISPs or organizations require routing all outgoing mail through a relay server. If this applies to your network, you need to specify the relay server’s DNS name. For example, you must use a relay server if the DNS service for your ISP or organization can’t resolve the public IP address of your server (or your AirPort Base Station or other router) to your server’s DNS name. This is called a reverse lookup.

You can also configure mail relay, including the authentication required by some relay servers, in the Mail pane of the Server Preferences application after setup.

Important:  Use only a relay server specified by your ISP or organization. Trying to relay mail through another server without permission may make your server appear to be a junk mail sender.
Enable SMTP relay authentication

Select this option if your ISP or organization requires your server to authenticate before sending mail, and then enter the user name and password from your ISP or organization.

Send a welcome email to new users

Select this option if you want the server to send an email telling new users about its services. The standard message specifies the server’s DNS name and explains the services that the server provides.

You can enter a custom introduction to the standard message. For example:

Hi, I’m the administrator for our server. If you need help getting services from it, please don’t hesitate to send me an email or call me at 310-555-4357. —Bill
