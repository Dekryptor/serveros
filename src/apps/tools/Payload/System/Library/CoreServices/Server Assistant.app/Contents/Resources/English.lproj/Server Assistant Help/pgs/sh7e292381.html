Configuring a Network Connection Service Using BootP

If you want a BootP server to assign an IP address and other network connection settings, choose Using BootP from the Configure IPv4 pop-up menu. With BootP, the same IP address is always assigned to a particular network interface. It’s used primarily for computers that start up from a NetBoot image.

If you’re setting up a server with Xsan, you can’t choose Using BootP because it’s inappropriate for an Xsan network.

To configure a network connection service using BootP:

After selecting a network connection service and choosing Using BootP from the Configure pop-up menu, enter the BootP configuration details using the following information:

  • DNS Server:  
    The IP address of a DNS server (for example, 192.0.2.12) or a list of IP addresses of DNS servers. A DNS server translates DNS names like server.example.com to IP addresses and vice versa.
  • Search Domains:  
    Specifies names to automatically append to DNS names that aren’t fully qualified. For example, if you specify campus.univ.edu as a search domain, you can type server1 in the Finder’s “Connect to Server” dialog to connect to server1.campus.univ.edu.
