Internet-Gateway

Konfigurieren Sie im Fenster "Gateway-Systemassistent" Ihren Server für die gemeinsame Nutzung der Internetverbindung mit anderen Computern im lokalen Netzwerk, wodurch Ihr lokales Netzwerk mit dem Internet verbunden wird.

Dieses Fenster wird nur angezeigt, wenn für den Server zwei Ethernetanschlüsse wie folgt konfiguriert sind:

  • Ein Ethernetanschluss muss eine öffentliche IP-Adresse im Internet (keine private IP-Adresse wie 10.0.0.1 oder 192.168.1.1) haben. Dieser Anschluss wird mit Ihrem DSL-Modem, Kabelmodem oder einer anderen Internetquelle verbunden. Gewöhnlich verwenden Sie hierfür den ersten integrierten Ethernetanschluss des Servers.

  • Ein weiterer Ethernetanschluss muss mit einem funktionsfähigen Netzwerk-Switch oder -Hub in Ihrem lokalen privaten Netzwerk verbunden werden. Dieser Ethernetanschluss darf nicht automatisch konfiguriert werden. Er muss eine manuelle IP-Adresse oder eine selbst zugeordnete IP-Adresse haben, die mit 169.254 beginnt. Andere mit diesem lokalen Netzwerk verbundene Computer nutzen die Internetverbindung des Servers gemeinsam.

Gehen Sie wie folgt vor, um Ihren Server als Internet-Gateway zu konfigurieren:

Wählen Sie "Diesen Server als Internet-Gateway konfigurieren".

Der Serverassistent ermittelt automatisch, welcher Ethernetanschluss die öffentliche und welcher die lokale Schnittstelle ist.

Wenn mehr als ein Ethernetanschluss für die öffentliche oder lokale Schnittstelle verwendet werden kann, wählen Sie den gewünschten Anschluss aus dem Einblendmenü aus.

Beim Konfigurieren Ihres Servers als Gateway geschieht Folgendes:

  • Dem mit dem lokalen Netzwerk verbundenen Ethernetanschluss wird die private IP-Adresse 192.168.1.1 zugewiesen.

  • Der DHCP-Dienst wird aktiviert und so konfiguriert, dass er die IP-Adressen von 192.168.1.100 bis 192.168.1.199 Computern im lokalen Netzwerk zur Verfügung stellt. Der DHCP-Dienst weist diese Adressen Computern zu, deren Ethernetanschlüsse mit der Option "DHCP" konfiguriert wurden.

    Sie können Benutzern auch Adressen von 192.168.1.2 bis 192.168.1.99 geben, mit denen sie ihre Ethernetanschlüsse bei Verwendung der Option "DHCP mit manueller Adresse" konfigurieren können. Die Adressen von 192.168.1.200 bis 192.168.1.220 sind für den VPN-Dienst Ihres Servers reserviert.

  • Der NAT-Dienst wird konfiguriert, um die Internetverbindung des Servers mit Computern im lokalen Netzwerk gemeinsam zu nutzen.

  • Die Firewall des Servers wird konfiguriert, um eingehende Verbindungen zu blockieren, die von Computern im Internet stammen. Die Firewall ermöglicht ausgehende Verbindungen von Computern im lokalen Netzwerk. Sie ermöglicht auch eingehende Verbindungen, die auf die ausgehenden Verbindungen von Computern antworten. Nach der Konfiguration können Sie im Fenster "Sicherheit" des Programms "Servereinstellungen" eingehende Anforderungen für bestimmte Dienste in der Firewall zulassen.

  • Der DNS-Dienst für das lokale Netzwerk wird so konfiguriert, dass DNS-Namensabfragen zwischengespeichert werden, um die Leistung für lokale Netzwerkcomputer zu verbessern.
