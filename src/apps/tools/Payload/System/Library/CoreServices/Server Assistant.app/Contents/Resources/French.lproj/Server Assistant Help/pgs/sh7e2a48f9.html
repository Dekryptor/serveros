Passerelle Internet

Utilisez la sous-fenêtre de configuration de la passerelle pour définir le serveur de manière qu’il partage sa connexion Internet avec d’autres ordinateurs du réseau local, reliant ainsi ce dernier à Internet.

La sous-fenêtre Passerelle apparaît uniquement si le serveur dispose de deux ports Ethernet configurés de la façon suivante :

  • Un port Ethernet doit disposer d’une adresse IP publique sur Internet (et non pas d’une adresse IP privée telle que 10.0.0.1 ou 192.168.1.1). Ce port est connecté au modem DSL, au modem câble ou à une autre source Internet. En règle générale, vous utilisez pour cela le premier port Ethernet intégré du serveur.

  • Un second port Ethernet doit être connecté à un commutateur réseau ou un concentrateur fonctionnels sur le réseau privé local. Ce port Ethernet doit être déconfiguré, disposer d’une adresse IP manuelle ou d’une adresse IP auto-assignée commençant par 169.254. Les autres ordinateurs connectés à ce réseau local partagent la connexion Internet du serveur.

Pour configurer le serveur en tant que passerelle Internet :

Sélectionnez « Configurer ce serveur en tant que passerelle Internet ».

L’Assistant du serveur détermine automatiquement le port Ethernet qui constitue l’interface publique et celui qui correspond à l’interface locale.

Si plusieurs ports Ethernet peuvent faire office d’interface publique ou d’interface locale, choisissez celui que vous voulez utiliser dans le menu local.

La configuration du serveur en tant que passerelle a les conséquences suivantes :

  • Attribue l’adresse IP privée 192.168.1.1 au port Ethernet connecté au réseau local.

  • Active le service DHCP et le configure afin de fournir les adresses IP 192.168.1.100 à 192.168.1.199 aux ordinateurs du réseau local. Le service DHCP attribue ces adresses aux ordinateurs dont les ports Ethernet sont configurés avec l’option Via DHCP.

    Vous pouvez aussi affecter les adresses 192.168.1.2 à 192.168.1.99 aux utilisateurs, afin de configurer leurs ports Ethernet avec l’option « Utilisation de DHCP avec une adresse manuelle ». Les adresses 192.168.1.200 à 192.168.1.220 sont réservées au service VPN du serveur.

  • Configure le service NAT pour partager la connexion Internet du serveur avec d’autres ordinateurs du réseau local.

  • Configure le coupe-feu du serveur afin qu’il bloque les connexions entrantes provenant d’ordinateurs connectés à Internet. Le coupe-feu autorise les connexions sortantes émanant des ordinateurs du réseau local. Il autorise aussi les connexions entrantes qui répondent aux connexions sortantes des ordinateurs locaux. Une fois la configuration terminée, vous pouvez utiliser la sous-fenêtre Sécurité des Préférences du serveur pour autoriser les requêtes entrantes via le coupe-feu pour des services spécifiques.

  • Configure le service DNS pour le réseau local et le définit sur des recherches de nom DNS cache afin d’améliorer les performances des ordinateurs du réseau local.
