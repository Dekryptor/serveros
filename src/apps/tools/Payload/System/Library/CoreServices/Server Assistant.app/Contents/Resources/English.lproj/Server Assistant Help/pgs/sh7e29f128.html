Configuring a Network Connection Service Using DHCP

If you want a DHCP server to assign all network connection settings, including the server’s IP address, choose Using DHCP from the Configure IPv4 pop-up menu in the Network pane.

Note:  The DHCP server should be configured to assign your server a static IP address (the same IP address all the time), and DHCP service should be running.

If you’re setting up a server with Xsan, you can’t choose Using DHCP because it’s inappropriate for an Xsan network.

To configure a network connection service using DHCP:

After selecting a network connection service and choosing Using DHCP from the Configure pop-up menu, enter the DHCP configuration details using the following information:

  • DNS Server:  
    The IP address of a DNS server (for example, 192.0.2.12) or a list of IP addresses of DNS servers. Usually, the DHCP server supplies the DNS server addresses automatically. A DNS server translates DNS names like server.example.com to IP addresses and vice versa.
  • Search Domains:  
    May be supplied automatically by your DHCP service. You can also enter the last part of one or more names that you want automatically appended to partial DNS names you enter while using the server. For example, if you specify campus.univ.edu as a search domain, you can type server1 in the Finder’s “Connect to Server” dialog to connect to server1.campus.univ.edu.
  • DHCP Client ID:  
    Enter the client identifier code if you received one from your ISP or network DHCP server administrator. The client ID identifies the server (or a network port, if the server has more than one network port). Don’t specify a DHCP client ID when using Server Assistant to set up up a remote server. Instead, after setup, enter a DHCP client ID in the Network pane of System Preferences on the server.
