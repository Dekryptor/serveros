Users and Groups

In the Users and Groups pane, the option you select specifies how you’ll manage the user and group accounts that the server uses to authenticate users and determine which services they’re allowed to access.

Options for managing users and groups
Create Users and Groups

Sets up an independent server for a small organization without an existing directory server. The server provides its own directory service as an Open Directory master, so the server has its own user and group accounts. In the next pane, you can select basic services that you want set up for you.

Import Users and Groups

Sets up a server for a workgroup in an organization with an existing directory server. The server can import users from that directory server. It can import users individually and can automatically import members of groups. You connect (bind) the server to the directory server in the next pane. The server also provides its own directory service as an Open Directory master, and thus can have its own user and group accounts. In a later pane, you can select basic services that you want set up for you.

Configure Manually

Gives the experienced system administrator complete control of directory service configuration to accommodate a wide variety of business needs.

After this pane, you can enter information for connecting (binding) the server to a directory server or creating an Open Directory master to make this a directory server. If you choose either of those options, you can also select basic services that you want set up for you. If you don’t connect to a directory server or create an Open Directory master, you configure directory services and all other services yourself after setup.

Note:  If the server isn’t connected to a network, select Configure Manually. The other two options will configure the server’s directory services, which require a network connection.

If don’t see these options in the Users and Groups pane, you’re setting up Mac OS X Server for an Xsan primary metadata controller. See Users and Groups (Xsan).
