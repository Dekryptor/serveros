Konfigurieren eines Netzwerkverbindungsdiensts mit BootP

Wenn Sie wollen, dass ein BootP-Server eine IP-Adresse und weitere Netzwerkverbindungseinstellungen zuweist, wählen Sie "BootP" aus dem Einblendmenü "IPv4 konfigurieren" aus. Bei BootP wird einer bestimmten Netzwerkschnittstelle die gleiche IP-Adresse zugewiesen. Diese Methode wird in erster Linie bei Computern verwendet, die über ein NetBoot-Image gestartet werden.

Wenn Sie einen Server mit Xsan konfigurieren, können Sie "BootP" nicht wählen, weil es für ein Xsan-Netzwerk ungeeignet ist.

Gehen Sie wie folgt vor, um einen Netzwerkverbindungsdienst mit BootP zu konfigurieren:

Geben Sie nach Auswählen eines Netzwerkverbindungsdiensts und der Option "BootP" aus dem Einblendmenü "Konfigurieren" die BootP-Konfigurationsdetails ein. Verwenden Sie hierzu die folgenden Informationen:

  • DNS-Server  
    Die IP-Adresse eines DNS-Servers (z. B. 192.0.2.12) oder eine Liste mit IP-Adressen von DNS-Servern. Ein DNS-Server übersetzt DNS-Namen wie server.example.com in IP-Adressen und umgekehrt.
  • Such-Domains  
    Gibt Namen an, die automatisch an DNS-Namen angehängt werden, die nicht vollständig qualifiziert sind. Wenn Sie z. B. "campus.univ.edu" als Such-Domain angeben, können Sie "server1" in das Finder-Dialogfenster "Mit Server verbinden" eingeben, um die Verbindung zu server1.campus.univ.edu herzustellen.
