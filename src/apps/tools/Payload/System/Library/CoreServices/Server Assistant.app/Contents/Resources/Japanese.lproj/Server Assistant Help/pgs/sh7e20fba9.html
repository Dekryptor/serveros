次の表で説明しているように、「サーバアシスタント」を使用して、Mac OS X Server ソフトウェアを別のコンピュータにインストールして設定することができます。「サーバアシスタント」は、すでに設定されているサーバから使用できます。または、Mac OS X Server 管理用ソフトウェアがインストールされている Mac OS X コンピュータから使用できます。

サーバを設定した後、「サーバ環境設定」アプリケーション、または「サーバ管理」や「ワークグループマネージャ」などの高度なサーバ管理ツールを使用して、ユーザとグループを管理したり、個々のサービスの設定をカスタマイズしたりします。Mac OS X Server の設定と管理について詳しくは、Mac OS X Server に付属のマニュアル「お使いになる前に」を参照してください。

「サーバアシスタント」を使って実行する操作：
Mac OS X Server をリモートでインストール

ネットワークを介してサーバソフトウェアをコンピュータにインストールします。次のことができます：

  • Mac OS X Server または Mac OS X がまだインストールされていないディスク上にインストール

  • Mac OS X Server バージョン 10.4.11 または Mac OS X Server v10.5 の最新アップデートをアップグレード

リモートコンピュータは、Mac OS X Server Install Disc、またはその言語ピッカーを表示するインストーラがある同等のものを使って起動する必要があります。

リモートサーバを設定する

ネットワークを介してサーバを設定します。リモートサーバは以下のものである必要があります：

  • 新しいサーバ、または Mac OS X Server v10.6 を新しくインストールしたコンピュータ

  • 「サーバアシスタント」で「ようこそ」パネルを表示して起動したもの

複数のサーバを同時に設定することができます。これらのサーバは、同じハードウェア構成を持ち、Mac OS X Server のシリアル番号、ネットワーク接続サービス、およびネットワーク名以外は同じ設定データを使用する必要があります。サーバ上に AirMac マネージメントを設定したい場合は、サーバを個別に設定するか、設定後に「サーバ環境設定」で AirMac マネージメントを有効にします。

自動設定用のプロファイルを作成

設定プロセスを実行して設定オプションの選択と設定データの入力を行います。その後、その設定情報を使用してすぐにサーバを構成するのではなく、設定情報をリムーバブルドライブまたはディスク上に自動設定プロファイルとして保存します。後で、保存した自動設定プロファイルを使用して、1 台または複数のサーバの設定を自動化することができます。

自動設定プロファイルを使用して、複数のサーバを一度に設定したり、サーバソフトウェアの再インストール後にテストサーバを設定したりできます。

複数のサーバを設定するための汎用の自動設定プロファイルを作成することも、特定のサーバ用の個別の自動設定プロファイルを作成することもできます。汎用の自動設定プロファイルを設定する場合：

  • すべてのサーバで使用される 1 つの管理者アカウント名およびパスワードを指定します。サーバの管理者パスワードは、「システム環境設定」の「アカウント」パネルを使って設定後に変更することができます。

  • 各ハードウェア・アドレス（Ethernet ID または MAC アドレスとも呼ばれます）のネットワーク接続サービス、または en0 や en1 のようなインターフェイス名（BSD ポート名）のネットワーク接続サービスを指定して、ハードウェア・アドレスに従って各サーバに固定（静的）IP アドレスを割り当てるように DHCP サーバを構成することができます。この機能は、「静的マッピング」または「DHCP の予約」と呼ばれています。

  • ネットワーク名を空白にしたままで、DNS サービスを構成して、各サーバに IP アドレスに従って名前を割り当てることができます。

セキュリティのため、自動設定プロファイルは暗号化された形式で保存できます。
