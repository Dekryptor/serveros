管理者アカウント

「管理者アカウント」パネルでは、サーバ管理権限を持つユーザアカウントを設定します。

管理者アカウント設定
名前

主要な管理者アカウント用に使用するフルネーム（「Server Administrator」など）。この名前には最大で 255 文字（日本語文字では 85 文字まで）を入力できます。空白を含めることができます。定義済みのユーザ名（「System Administrator」など）と同じ名前にすることはできません。

ユーザ名

これは、フルネームの省略形です。ユーザ名は、通常は 8 文字以下ですが、255 文字以内の 1 バイト文字を使用できます。a 〜 z、A 〜 Z、0 〜 9、.（ピリオド）、_（アンダースコア）、または -（ハイフン）のみを使用してください。 root のような定義済みのユーザ名は使用できません。アカウントを作成した後でユーザ名を変更することはできません。

パスワード

パスワードは推測困難なものにし、他の人に知られないようにしてください。管理者名とパスワードを知っている人はだれでも、サーバ設定を変更したりユーザアカウントを管理することができます。

パスワードのヒント

パスワードを思い出すために役立つヒント（パスワードそのものを除く）にすることができます。
管理者が SSH を使用してリモートログインすることを許可

サーバ管理者に、SSH（Secure Shell）ネットワークプロトコルを使用して別の場所からサーバにログインすることを許可します。
管理者がこのサーバをリモート管理することを許可

サーバ管理者が、Mac OS X の画面共有または Apple Remote Desktop ソフトウェア（別途購入できます）と Mac のディスプレイ、キーボード、マウスを使って、ネットワーク経由でサーバを監視および制御することを許可します。

その他の管理者アカウント

ここで入力するパスワードは、System Administrator アカウント（ユーザ名が root であるアカウント）にも使用されます。サーバにそれ自体のユーザとグループがある場合、このパスワードはディレクトリ管理者アカウントにも使用されます。このアカウントの名前は Directory Administrator で、ユーザ名は diradmin です。後で「ユーザとグループ」パネルで「手動で設定」オプションを選択した場合は、別のディレクトリ管理者名とユーザ名を入力することができます。

System Administrator アカウントは、システムファイルも含め、システムへのフルアクセス権を持ちます。この root ユーザパスワードの保護は非常に重要なため、別のアカウントのパスワードと同じものにはしないでください。サーバ管理者には root ユーザ権限は必要ありません。

重要： サーバの設定後に、ここで作成するアカウントのパスワードとディレクトリ管理者アカウントを変更するか、System Administrator（root）アカウントのパスワードを変更する必要があります。

ここで作成する管理者アカウントのパスワードまたはその他の属性を変更するには、サーバにログインした後で「システム環境設定」の「アカウント」パネルを使用します。

Directory Administrator アカウントのパスワードまたはその他の属性を変更するには、サーバの設定後に「ワークグループマネージャ」アプリケーションを使用します。

System Administrator（root）アカウントのパスワードを変更するには、サーバの設定後に、「システム環境設定」の「アカウント」パネルの「ログインオプション」セクションで「ディレクトリユーティリティ」アプリケーションを開きます。

コンピュータのセキュリティを保つ

セキュリティのため、管理者アカウントではなく標準のユーザアカウントを作成し、常にそれを使用してログインしてください。管理者アカウントは、「サーバ環境設定」、「サーバ管理」、「ワークグループマネージャ」などの管理用アプリケーションやツールで、ユーザ、グループ、またはサービスを管理する必要がある場合に使用します。

標準のユーザアカウントは、「サーバ環境設定」または「ワークグループマネージャ」を使用して作成できます。
