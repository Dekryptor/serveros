Users and Groups (Xsan)

In the Users and Groups pane for an Xsan primary metadata controller, the option you select specifies how you manage user and group accounts, which are used to authenticate users of SAN computers and determine which services they’re allowed to access. You use the user and group accounts to control access to files and folders on the SAN by assigning access privileges to users and groups.

Options for managing users and groups
Manage users and groups with Xsan Admin

Sets up the primary metadata controller as an Open Directory master, so it has its own directory service for user and group accounts. You create and manage users and groups using Xsan Admin after setup. Xsan Admin automatically configures SAN computers to connect to the Open Directory service.

This option is recommended if you don’t already have a directory server and you don’t expect to have more than 20 SAN users. These users will have local home folders on their computers, not network home folders on the server.

Important:  If you select this option, be sure to use Xsan Admin—do not use Server Preferences—to create users and groups. Xsan Admin creates a home folder for a new user account, but Server Preferences doesn’t.
Important:  You can select this option only while using this setup assistant to set up Mac OS X Server on the primary metadata controller. You can’t configure this option afterward.
Use existing users and groups from an Open Directory server

Allows you to connect the primary metadata controller to an existing Open Directory server whose DNS name or IP address you specify in the next pane. Xsan Admin automatically configures SAN computers to connect to the Open Directory server. Users and groups are managed by an administrator of the Open Directory server.

Connect to a directory server later

Allows you to postpone connecting to an existing Open Directory server until after setup, or to connect to another type of directory server, such as Active Directory, after setup. To configure directory service connections on SAN computers with Snow Leopard, use the Login Options section of the Accounts pane of System Preferences. To configure directory service connections on SAN computers with Leopard, use the Directory Utility application.

If you don’t see these options, you’re not setting up an Xsan primary metadata controller. See Users and Groups.
