Konfigurieren eines Netzwerkverbindungsdiensts mit DHCP und manueller Adresse

Wenn Sie eine statische IP-Adresse eingeben und die übrigen Netzwerkverbindungseinstellungen durch den DHCP-Server zuweisen lassen wollen, wählen Sie "DHCP mit manueller Adresse" aus dem Einblendmenü "IPv4 konfigurieren" im Fenster "Netzwerk" aus.

Gehen Sie wie folgt vor, um einen Netzwerkverbindungsdienst mit DHCP und manueller Adresse zu konfigurieren:

Geben Sie nach Auswählen eines Netzwerkverbindungsdiensts und der Option "DHCP mit manueller Adresse” aus dem Einblendmenü "Konfigurieren" die Konfigurationsdetails ein. Verwenden Sie hierzu die folgenden Informationen:

  • IP-Adresse:  
    Eine numerische Adresse, die den Server im Netzwerk eindeutig identifiziert. Sie wird in Form von vier durch Punkte getrennten Zahlen angegeben (z. B. 192.0.2.123). Wenn Sie eine AirPort-Basisstation oder einen anderen Internet-Router mit einer freigegebenen Internetverbindung in Ihrem lokalen Netzwerk haben, muss die IP-Adresse des Servers demselben IP-Teilnetzwerk angehören wie die IP-Adresse der Basisstation oder des Routers, etwa 10.0.7.2 oder 192.168.99.2.
  • DNS-Server:  
    Die IP-Adresse eines DNS-Servers (z. B. 192.0.2.12) oder eine Liste mit IP-Adressen von DNS-Servern. Gewöhnlich stellt der DHCP-Server die DNS-Serveradressen automatisch bereit. Ein DNS-Server übersetzt DNS-Namen wie server.example.com in IP-Adressen und umgekehrt.
  • Such-Domains:  
    Können automatisch von Ihrem DHCP-Dienst bereitgestellt werden. Sie können auch den letzten Teil von einem oder mehr Namen eingeben, der automatisch an unvollständige DNS-Namen angehängt werden soll, die Sie bei Verwendung des Servers eingeben. Wenn Sie z. B. "campus.univ.edu" als Such-Domain angeben, können Sie "server1" in das Finder-Dialogfenster "Mit Server verbinden" eingeben, um die Verbindung zu server1.campus.univ.edu herzustellen.
