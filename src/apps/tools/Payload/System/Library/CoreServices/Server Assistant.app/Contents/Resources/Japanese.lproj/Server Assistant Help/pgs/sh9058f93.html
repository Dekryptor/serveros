ディレクトリサービス

「ディレクトリサービス」パネルでは、ユーザ、グループ、コンピュータにネットワークアカウントを提供できるように、サーバを Open Directory のマスターにします。サーバは、前のパネルでサーバを接続した既存のディレクトリサーバに加え、Open Directory サービスを提供します。この時点ではサーバを Open Directory のマスターにしないと選択する場合、サーバ設定の終了後に「サーバ管理」を使って実行できます。

Open Directory のマスターの設定
名前

ディレクトリ管理者アカウント用に使用したいフルネーム。デフォルトのユーザ名は Directory Administrator です。
ユーザ名

これは、フルネームの省略形です。デフォルトのユーザ名は diradmin です。
UID

ユーザ ID 番号。フォルダとファイルのアクセス権に使用します。デフォルトの UID は 1000 です。
パスワード

自動的に、以前に設定したサーバ管理者アカウントと同じパスワードに設定されます。
個々のユーザとグループのサービスへのアクセスを制限

このオプションを選択すると、アドレスブック、ファイル共有、iCal カレンダー、iChat インスタントメッセージ、メール、および VPN リモート・アクセス・サービスへのアクセスが制限されます。「サーバ環境設定」で追加したユーザアカウントには、これらのすべてのサービスへのアクセス権が付与されます。サービスへのユーザとグループのアクセス権は、「サーバ環境設定」と「サーバ管理」を使って変更することができます。

Open Directory のマスターをこの時点で作成せず、前のパネルでディレクトリサーバに接続しなかった場合、「サーバアシスタント」ではサービスは設定されません。「サーバアシスタント」を終了した後、「サーバ管理」またはコマンドラインツールを使用して、このサーバで提供したいサービスを設定することができます。
