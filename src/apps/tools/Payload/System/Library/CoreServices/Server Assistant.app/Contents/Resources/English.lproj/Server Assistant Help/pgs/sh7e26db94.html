Network

Use the Network pane to specify connection settings for your server’s network interfaces. The group of settings for each interface is a network connection service. A network interface is also called a port, which is the part of the interface that a plug or cable connects to.

Your server needs a network connection service for the local network that users’ computers connect to. This is usually an Ethernet network, but may be an AirPort network. If your Internet service provider (ISP) requires connecting via PPPoE, you need to configure a PPPoE network service connection. If your server connects to more than one network, you need to configure a network connection service for each network your server connects to.

To configure a network service connection:

  • Select a network connection service in the list, and then specify settings. For information about the settings for the different kinds of network connection services, see the following topics:

    Configuring an Ethernet Network Connection Service
    Configuring an AirPort Network Connection Service
    Configuring a FireWire Network Connection Service
    Configuring a PPPoE Network Connection Service
    Configuring an LOM Port Network Connection Service
    Making a Network Connection Service Inactive

  • Add a network connection service by clicking the Add (+) button. For instructions, see Adding a Network Connection Service.

  • Remove a network connection service by selecting it and clicking the Delete (–) button.

  • Show only preferred network connection services, or show all network connection services, by using the Action (gear) button if it’s present.

  • See the hardware address and interface name for a network connection service by hovering the pointer over the network connection service in the list.

  • Change the priority of network connection services by dragging them in the list. When trying to communicate over a network, the server uses the network connection services in the listed order.

    The first active network connection service is the server’s primary network connection. The server uses this service for all communications outside the local network.

The Network pane initially lists a network connection service for each of the server’s Ethernet interfaces. Other available network interfaces aren’t listed, because Ethernet is preferred for server network connections. To see all network interfaces, use the Action (gear) button. If the server has no Ethernet interfaces, the Network pane lists all available network interfaces. The Network pane doesn’t initially list any services if you’re creating a profile for later automatic server setup.

Most Macintosh computers have a built-in Ethernet port. They may have additional interfaces built in or added on, such as an AirPort interface for a wireless network connection, a FireWire port that can be used for a network connection, or a second Ethernet port.

Intel-based Xserve systems also have an interface for the lights-out management (LOM) processor. This interface has its own IP address and lets you connect to the server using the Server Monitor application even when the server is shut down.

Network connection services for an Internet gateway

If the Internet Gateway pane appeared prior to the Network pane, and you chose to make this server an Internet gateway, you can’t change settings for the Ethernet connection to your local network. Server Assistant configures those settings automatically.

Network connection services for multiple servers

If you’re setting up multiple servers at the same time, Server Assistant displays the Network pane for each one in turn. The text near the top of the pane tells you the DNS name and IP address of the server you’re currently configuring.

Network connection services for an Xsan metadata controller

If the Xsan pane appeared earlier and you chose to make the server an Xsan metadata controller:

  • Make sure the network connection service for the private metadata network is inactive if you plan to have the private metadata network configured automatically during Xsan setup. For instructions, see Making a Network Connection Service Inactive.

  • Make sure all other network connection services will each have a fixed (static) IP address. For example, set the Configure IPv4 option to “Manually” or “Using DHCP with manual address.”

    The BootP, PPPoE, and DHCP network configuration options aren’t available because they aren’t appropriate for an Xsan network.

Network connection services for an auto setup profile

If you’re creating a profile for later automatic server setup, the network connection services you configure in this pane will be used as follows: While setting up a server automatically, Server Assistant tries to match a network connection service in the saved profile to the server’s network interfaces by comparing hardware addresses (also called MAC addresses or Ethernet IDs). If that fails, it tries to match the interface name (also called the BSD port name) of a saved network connection service, such as en0, en1, or fw0, to the server’s network interfaces. If that fails, it doesn’t use the saved network connection service.

You can set up generic network connection services that will match the network interfaces on a variety of servers. Set up two Ethernet services and one AirPort service, and set the Configure IPv4 option of each to Using DHCP. Before starting automatic server setup, make sure your DHCP service is configured to assign a fixed (static) IP address to each network interface of each server being set up automatically. If a particular server doesn’t have two Ethernet interfaces or an AirPort interface, Server Assistant doesn’t use the corresponding saved network connection service.
