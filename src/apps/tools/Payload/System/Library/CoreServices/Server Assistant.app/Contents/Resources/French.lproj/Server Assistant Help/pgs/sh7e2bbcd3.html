Configuration manuelle d’un service de connexion réseau

Si le réseau ne dispose d’aucun serveur DHCP ou si vous avez besoin de spécifier tous les réglages de connexion réseau manuellement pour un autre motif, choisissez Manuellement dans le menu local Configurer IPv4.

Pour configurer un service de connexion réseau manuellement :

Après avoir choisi un service de connexion réseau et sélectionné l’option Manuellement dans le menu local Configurer, saisissez les informations de configuration à l’aide des données suivantes :

  • Adresse IP :  
    Une adresse numérique qui identifie de façon unique le serveur sur le réseau. Elle se présente sous la forme de quatre nombres séparés par des points (par exemple, 192.0.2.123). Si vous disposez d’une borne d’accès AirPort ou d’un autre routeur Internet partageant une connexion Internet sur le réseau local, l’adresse IP du serveur doit figurer sur le même sous-réseau IP que celle de la borne d’accès ou du routeur, par exemple 10.0.7.2 ou 192.168.99.2.
  • Masque de sous-réseau :  
    Un groupe de quatre nombres qui identifie la partie réseau de l’adresse IP du serveur ; le reste de l’adresse IP identifie de façon unique le serveur sur ce réseau. Si le serveur dispose d’une adresse IP privée, utilisez 255.255.255.0 comme masque de sous-réseau. Si l’adresse IP du serveur est publique (si elle ne commence pas par 192.168, 10.0–10.255 ou 172.16–172.31), obtenez le masque de sous-réseau à partir de la même source que l’adresse IP du serveur.
  • Routeur :  
    L’adresse IP du routeur pour le réseau local du serveur (par exemple, 192.168.99.1 ou 10.0.7.1). Si vous disposez d’une borne d’accès AirPort ou d’un autre routeur Internet, utilisez son adresse. Sinon, obtenez l’adresse du routeur auprès de la même source que l’adresse IP du serveur. Si le réseau ne dispose pas de routeur, utilisez l’adresse IP du serveur en tant qu’adresse du routeur. Le routeur est un périphérique qui réexpédie les paquets de données du réseau local vers leur destination sur d’autres réseaux ou sur Internet.
  • Serveur DNS :  
    L’adresse IP d’un serveur DNS (par exemple, 192.0.2.12) ou une liste d’adresses IP de serveurs DNS. Un serveur DNS traduit les noms DNS tels que serveur.exemple.com en adresses IP et vice versa.
  • Domaines de recherche :  
    Spécifie des noms à ajouter automatiquement aux noms DNS incomplets. Par exemple, si vous spécifiez le domaine de recherche campus.univ.edu, vous pouvez saisir serveur1 dans la zone de dialogue Connexion au serveur du Finder pour établir la connexion avec serveur1.campus.univ.edu.
