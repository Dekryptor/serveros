ディレクトリサーバに接続

「ディレクトリサーバに接続」パネルでは、組織内の既存のディレクトリサーバへの接続を手動で構成します。 ディレクトリサーバ接続をここでは構成しないと選択した場合は、サーバの設定を終了した後でディレクトリサーバに接続することができます。サーバの設定後に 1 つまたは複数のディレクトリサーバに接続するには、「システム環境設定」の「アカウント」パネルの「ログインオプション」セクションを使用します。

ディレクトリサーバ接続の設定
外部ディレクトリサーバに接続

サーバを既存のディレクトリシステムに接続（バインド）したい場合は、このオプションを使用します。

サーバ上の Xsan を設定する場合、このオプションは選択できません。この時点でディレクトリサーバに接続したくない場合は、「戻る」をクリックし、「ユーザとグループ」パネルで「後でディレクトリサーバに接続」を選択します。

サーバアドレス

ポップアップメニューからディレクトリサーバを選択するか、ディレクトリサーバの DNS 名を入力します。Open Directory サーバに接続する場合は、その IP アドレスを入力することができます。

サーバのタイプ

Open Directory または Active Directory を選択できます。

この時点でサーバを設定し、後で使用するための自動設定プロファイルは作成しない場合、「サーバのタイプ」は自動的に決定されます。

別のタイプのディレクトリサーバに接続する必要がある場合は、「外部ディレクトリサーバに接続」オプションの選択を解除し、サーバ設定の終了後に接続を構成します。

サーバ上の Xsan を設定する場合、このパネルでのみ Open Directory サーバに接続できます。Xsan を含むサーバを別のタイプのディレクトリサーバに接続するには、「戻る」をクリックし、「ユーザとグループ」パネルで「後でディレクトリサーバに接続」を選択します。

ユーザアカウント名とパスワード

これらのフィールドは、ディレクトリサーバが認証を許可または必要としている場合に表示されます。

  • Open Directory の場合は、標準のユーザアカウントのユーザ名またはロングネームとパスワードを入力できるため、ディレクトリ管理者アカウントを使用する必要はありません。

  • Active Directory の場合は、Active Directory 管理者アカウント、または「ワークステーションをドメインに追加する」権限を持つ標準のユーザアカウントの名前とパスワードを入力できます。
