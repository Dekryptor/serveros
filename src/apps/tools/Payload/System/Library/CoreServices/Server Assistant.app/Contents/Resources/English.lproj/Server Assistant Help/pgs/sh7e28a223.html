Configuring a Network Connection Service Using DHCP with Manual Address

If you want to enter a static IP address and let the DHCP server assign other network connection settings, choose “Using DHCP with Manual Address” from the Configure IPv4 pop-up menu in the Network pane.

To configure a network connection service using DHCP with manual address:

After selecting a network connection service and choosing “Using DHCP with manual address” from the Configure pop-up menu, enter the configuration details using the following information:

  • IP Address:  
    A numeric address that uniquely identifies the server on the network. It’s written as four numbers separated by periods (for example, 192.0.2.123). If you have an AirPort Base Station or other Internet router that shares an Internet connection on your local network, the server’s IP address must be on the same IP subnet as the IP address of the base station or router, such as 10.0.7.2 or 192.168.99.2.
  • DNS Server:  
    The IP address of a DNS server (for example, 192.0.2.12) or a list of IP addresses of DNS servers. Usually, the DHCP server supplies the DNS server addresses automatically. A DNS server translates DNS names like server.example.com to IP addresses and vice versa.
  • Search Domains:  
    May be supplied automatically by your DHCP service. You can also enter the last part of one or more names that you want automatically appended to partial DNS names you enter while using the server. For example, if you specify campus.univ.edu as a search domain, you can type server1 in the Finder’s “Connect to Server” dialog to connect to server1.campus.univ.edu.
