Noms de réseau

Utilisez la sous-fenêtre Noms de réseau pour configurer les noms permettant d’identifier votre serveur auprès des autres ordinateurs du réseau.

Réglages relatifs aux noms de réseau
Nom DNS principal

Cette valeur est définie automatiquement si elle peut être déterminée à partir du serveur DNS de votre réseau. Il s’agit du nom complet unique du serveur dans le service DNS, tel que monserveur.exemple.com. Il est préférable de ne pas modifier ce nom s’il est enregistré pour le serveur, toutefois s’il est erroné vous pouvez cliquer sur le bouton Modifier pour le corriger. Si vous modifiez le nom DNS dans cette sous-fenêtre, veillez à ce que votre serveur DNS soit mis à jour de façon à utiliser le nouveau nom.

Si votre réseau ne comporte pas de serveur DNS, utilisez un nom du type monserveur.personnel.
Nom de l’ordinateur

Cette valeur identifie le serveur pour les utilisateurs qui recherchent des serveurs de fichiers réseau ou d’autres ressources réseau identifiées par le nom de l’ordinateur plutôt que par l’adresse du réseau. Indiquez un nom constitué de 63 caractères romains maximum. Les espaces sont autorisés, en revanche les signes égal, deux-points et arobase (=, :, @) sont à proscrire. (Si l’ordinateur d’un utilisateur se connecte au serveur à l’aide du protocole de partage de fichiers SMB, Mac OS X Server convertit automatiquement le nom de l’ordinateur en un formulaire compatible avec SMB.)

Nom d’hôte local

L’ordinateur possède également un nom d’hôte local, qui se termine par « .local » (monserveur.local, par exemple). Les ordinateurs présents sur le sous-réseau local du serveur peuvent faire appel à son nom d’hôte plutôt qu’à son nom DNS principal pour atteindre le serveur sans utiliser de système DNS global pour convertir les noms en adresses IP. Le nom d’hôte local d’un ordinateur ne peut être employé que sur son réseau local (sous-réseau IP).

Le nom d’hôte local est dérivé par défaut du nom de l’ordinateur. Si vous souhaitez le modifier, utilisez l’application Admin Serveur ou la sous-fenêtre Partage de l’application Préférences Système une fois la configuration terminée.

Noms de réseau pour un profil de configuration automatique

Si vous créez un profil en vue d’une configuration automatique ultérieure et souhaitez l’utiliser pour configurer plusieurs serveurs, vous pouvez laisser les champs « Nom DNS principal » et « Nom de l’ordinateur » vides. Lorsque vous utiliserez ce profil pour configurer un serveur, l’Assistant du serveur recherchera le nom DNS à partir de l’adresse IP du serveur et déterminera le nom de l’ordinateur à partir de ce nom DNS.
