Network Names

Use the Network Names pane to set up names that will identify your server to other computers on the network.

Settings for network names
Primary DNS Name

This is set automatically if it can be determined from the DNS server on your network. It’s the full, unique name for the server in the DNS service, for example myserver.example.com. You shouldn’t change this name if it’s actually registered to your server, but if it’s wrong you can click the Edit button to change it. If you change the DNS name in this pane, make sure your DNS server is updated to use the new name.

If your network doesn’t have a DNS server, use a name like myserver.private.
Computer Name

Identifies the server to users who are browsing for network file servers or other network resources identified by computer name rather than network address. Specify a name 63 Roman characters or fewer. The name can include spaces, but avoid using =, :, or @. (If a user’s computer connects to the server using the SMB file sharing protocol, Mac OS X Server automatically converts the computer name to a form that’s valid with SMB.)

Local Hostname

The computer also has a local hostname, which ends with “.local” (such as myserver.local). Computers on the server’s local subnet can use the server’s local hostname instead of its primary DNS name to reach the server without using a global DNS system to resolve names to IP addresses. A computer’s local hostname can be used only on its local network (IP subnet).

The local hostname is derived by default from the computer name. If you want to change it, use Server Admin or the Sharing pane of System Preferences after setup is complete.

Network names for an auto setup profile

If you’re creating a profile for later automatic server setup and want to use it to set up multiple servers, you can leave the Primary DNS Name and Computer Name blank. When you use the profile to set up a server, Server Assistant looks up the DNS name using the server’s IP address and derives the computer name from the DNS name.
