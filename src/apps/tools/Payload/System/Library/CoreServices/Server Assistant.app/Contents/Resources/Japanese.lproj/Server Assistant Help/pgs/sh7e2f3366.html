Xsan コントローラのタイプ

「Xsan コントローラのタイプ」パネルでは、このサーバを SAN のプライマリ・メタデータ・コントローラまたはセカンダリー・メタデータ・コントローラにするかを選択します。コントローラは、ボリュームのメタデータを管理したり、ファイルシステムのジャーナルを維持したり、ファイルへの同時アクセスを制御したりします。 メタデータには、ファイルの実際の保存場所や、使用可能なストレージのどの部分が新しいファイルに割り当てられるかなどの情報が含まれています。

SAN にはプライマリコントローラが含まれている必要があります。ボリュームの可用性を保証するには、SAN に 1 つ以上のスタンバイコントローラを含める必要があります。 プライマリコントローラに障害が発生した場合は、スタンバイコントローラがその役割を引き継ぎます。
