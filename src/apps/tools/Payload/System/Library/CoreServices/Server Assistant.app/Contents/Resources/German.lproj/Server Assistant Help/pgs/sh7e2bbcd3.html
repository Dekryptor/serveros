Manuelles Konfigurieren eines Netzwerkverbindungsdiensts

Wenn Ihr Netzwerk keinen DHCP-Server hat oder wenn Sie aus einem anderen Grund alle Netzwerkverbindungseinstellungen manuell angeben müssen, wählen Sie "Manuell" aus dem Einblendmenü "IPv4 konfigurieren" aus.

Gehen Sie wie folgt vor, um einen Netzwerkverbindungsdienst manuell zu konfigurieren:

Geben Sie nach Auswählen eines Netzwerkverbindungsdiensts und der Option "Manuell" aus dem Einblendmenü "Konfigurieren" die Konfigurationsdetails ein. Verwenden Sie hierzu die folgenden Informationen:

  • IP-Adresse:  
    Eine numerische Adresse, die den Server im Netzwerk eindeutig identifiziert. Sie wird in Form von vier durch Punkte getrennten Zahlen angegeben (z. B. 192.0.2.123). Wenn Sie eine AirPort-Basisstation oder einen anderen Internet-Router mit einer freigegebenen Internetverbindung in Ihrem lokalen Netzwerk haben, muss die IP-Adresse des Servers demselben IP-Teilnetzwerk angehören wie die IP-Adresse der Basisstation oder des Routers, etwa 10.0.7.2 oder 192.168.99.2.
  • Teilnetzmaske:  
    Eine Gruppe von vier Zahlen, die den Netzwerkteil der IP-Adresse des Servers angibt. Der Rest der IP-Adresse identifiziert den Server in diesem Netzwerk eindeutig. Hat der Server eine private IP-Adresse, verwenden Sie 255.255.255.0 als Teilnetzmaske. Hat der Server eine öffentliche IP-Adresse, die nicht mit 192.168, 10.0 - 10.255 oder 172.16 - 172.31 beginnt, beziehen Sie die Teilnetzmaske aus derselben Quelle wie die IP-Adresse Ihres Servers.
  • Router:  
    Die IP-Adresse des Routers für das lokale Netzwerk des Servers (z. B. 192.168.99.1 oder 10.0.7.1). Wenn Sie eine AirPort-Basisstation oder einen anderen Internet-Router haben, verwenden Sie deren bzw. dessen Adresse. Andernfalls erhalten Sie die Router-Adresse aus derselben Quelle wie die IP-Adresse Ihres Servers. Wenn Ihr Netzwerk keinen Router umfasst, verwenden Sie die IP-Adresse des Servers als Router-Adresse. Der Router ist ein Gerät, das Datenpakete aus dem lokalen Netzwerk an ihre Ziele in anderen Netzwerken oder im Internet weiterleitet.
  • DNS-Server:  
    Die IP-Adresse eines DNS-Servers (z. B. 192.0.2.12) oder eine Liste mit IP-Adressen von DNS-Servern. Ein DNS-Server übersetzt DNS-Namen wie server.example.com in IP-Adressen und umgekehrt.
  • Such-Domains:  
    Gibt Namen an, die automatisch an DNS-Namen angehängt werden, die nicht vollständig qualifiziert sind. Wenn Sie z. B. “campus.univ.edu” als Such-Domain angeben, können Sie "server1" in das Finder-Dialogfenster "Mit Server verbinden" eingeben, um die Verbindung zu server1.campus.univ.edu herzustellen.
