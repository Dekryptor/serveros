Agent

Xgrid-Agenten führen die Rechenanforderungen eines Auftrags aus. Unter Mac OS X Server ist der Agent standardmäßig deaktiviert. Wenn ein Agent aktiviert wurde und nach dem Starten des Computers aktiv wird, wird er bei einem Controller registriert. (Ein Agent kann nur mit jeweils einem Controller gleichzeitig verbunden sein.) Der Controller sendet Anweisungen und Daten an den Agenten wie für die Aufträge des Controllers erforderlich. Nachdem der Agent Anweisungen vom Controller erhalten hat, führt er die ihm zugewiesenen Aufgaben aus und sendet die Ergebnisse zurück an den Controller.

Standardmäßig stellen Agenten eine Verbindung mit dem ersten verfügbaren Controller im lokalen Netzwerk her. Sie können alternativ dazu die Verbindung mit einem bestimmten Controller festlegen.

Sie können außerdem angeben, ob ein Agent immer verfügbar sein soll oder nur dann, wenn der Computer nicht verwendet wird. Ein Computer wird als verfügbar eingestuft, wenn weder Maus- noch Tastatureingaben erfolgen und CPU- und Netzwerkaktivitäten ignoriert werden. Setzt ein Benutzer seine Arbeit am Computer fort, auf dem eine Grid-Aufgabe ausgeführt wird, setzt der Computer die Verarbeitung der Aufgabe fort, bis diese beendet ist.

Standardmäßig arbeitet der Agent von Mac OS X Server dediziert. Der Agent auf einem Mac OS X-Computer (nicht einem Server) ist standardmäßig so konfiguriert, dass er Aufgaben nur dann akzeptiert, wenn 15 Minuten lang keine Benutzereingabe am System erfolgt ist.
