Funktionsweise von Xgrid

Xgrid erstellt aus jedem Auftrag mehrere Aufgaben und verteilt diese an mehrere Knoten. Bei diesen Knoten kann es sich um Desktop-Computer handeln, die mit Mac OS X 10.3 (oder neuer) arbeiten, oder um Systeme mit Mac OS X Server 10.4 (oder neuer).

Häufig sind Desktop-Computer zu bestimmten Zeiten im Verlauf eines Tages, an den Abenden und an Wochenenden nicht in Gebrauch. Das Zusammenfügen dieser Systeme zu einem Computing-Grid ermöglicht es, Ressourcen von Desktop-Computern effizient zu nutzen. Dies wird auch als effiziente Ressourcennutzung bezeichnet.Diese Methode der Grid-Erstellung ermöglicht eine bedeutende Steigerung der verfügbaren Rechenleistung ohne Kauf zusätzlicher Hardware. Mit Xgrid wird die Konfiguration der Software zu einer sehr einfachen Aufgabe.

Damit ein Server als Controller verwendet werden kann, ist für Xgrid Mac OS X Server 10.4 (oder neuer) sowie mindestens 256 MB Arbeitsspeicher oder Mac OS X Server 10.5 (oder neuer) mit 1 GB Arbeitsspeicher erforderlich. Für die Verwendung als Agent in einem Grid ist für Xgrid Mac OS X 10.3 (oder neuer) sowie mindestens 128 MB Arbeitsspeicher (256 MB empfohlen) oder Mac OS X 10.4 (oder neuer) mit 512 MB Arbeitsspeicher erforderlich. Alle Xgrid-Komponenten müssen über eine Netzwerkverbindung verfügen. Auch hier gilt der Grundsatz: Je mehr Arbeitsspeicher, desto besser die Leistung eines Systems, besonders bei rechenintensiven Programmen.

Unter einem Grid versteht man eine Gruppe von Computern, die gemeinsam an der Lösung eines Problems arbeiten. Die Systeme in einem Grid können lose miteinander verbunden, geografisch voneinander getrennt und bis zu einem bestimmen Maß verschiedenartig sein. Die Systeme in einem Cluster sind dagegen oft von gleichem Typ, in unmittelbarer Nachbarschaft angeordnet und nach strengen Regeln verwaltet.

Weit verteilte Grids wie SETI@Home ermöglichen es einzelnen Benutzern, ihre nicht benötigte Prozessorleistung für eine bestimmte Aufgabe bereitzustellen. In Büronetzwerken können komplexe Render- oder Simulationsaufgaben auf alle Systeme verteilt werden, die nachts nicht verwendet werden. Mit einer solchen Konfiguration kann auch ein dedizierter Computer-Cluster erweitert werden, der Xgrid-Clients jederzeit zur Verfügung steht.

Die von Xgrid unterstützten Rechenleistung ist nahezu unbegrenzt. Die Leistung des Grid wird u. a. durch die darin verwendeten Systeme, die ausgeführte Software und das Netzwerk bestimmt. Einzelne Programme beeinflussen die Leistung des Grids allerdings in hohem Maße.

Sie bestimmen selbst, ob ein Programm durch die Implementierung in einem Computing-Grid optimiert wird. Im besten Fall nimmt die Leistung des Programms linear mit der Größe des Grids zu. Im ungünstigsten Fall führt das Hinzufügen von Agenten zu einem Grid dazu, dass ein Auftrag noch langsamer ausgeführt wird, als dies mit weniger Agenten möglich wäre. (In einer solchen Situation werden Aufgaben so stark aufgeteilt, dass der durch die Verteilung der größeren Anzahl an Aufgaben entstehende Overhead den Leistungszuwachs übersteigt, der durch die Verwendung zusätzlicher Agenten erzielt wird.) Diese Aspekte sollten Sie berücksichtigen.

Bei vielen proprietären Projekten ist die Teilnahme an einem großen Computing-Grid möglich. Diese Projekte wie SETI@Home und FightAids@home dienen häufig bestimmten wissenschaftlichen Zwecken. Sie umfassen normalerweise eine einfach zu installierende Software, über die Freiwillige an diesem Projekt teilnehmen können, und werden oftmals als Bildschirmschoner oder Hintergrundprozess ausgeführt.

Sie müssen sich jedoch nicht Tausende oder gar Millionen wenig genutzter Computer vorstellen, um die Bedeutung eines Computing-Grids zu erkennen. Die von den Studenten einer Universität oder den Mitarbeitern eines Unternehmens verwendeten Computer werden im Regelfall weniger Stunden aktiv verwendet als sie über Nacht oder an Wochenenden ungenutzt bleiben. Diese Computer könnten die Verarbeitung in einem Grid auf produktive Weise unterstützen, ohne dass dabei ihr Nutzen für die Studenten oder Mitarbeiter beeinträchtigt wird.

Andere Grid-Projekte sind für große Computing-Grids ausgelegt, etwa Globus Alliance (eine von Universitäten und Forschern gegründete Gruppe). Diese Grids besitzen Werkzeuge für eine flexible Ressourcen-Verwaltung und intelligentere Implementierungsmethoden für das Grid. Anstelle einer Entwicklung fertig ausgearbeiteter Programme für ein bestimmtes Grid bieten solche Projekte umfassende Rahmenbedingungen für die Erstellung von Programmen.

Mithilfe von Xgrid können Benutzer an einem Computing-Grid ihrer Wahl teilnehmen. Gleichzeitig bietet die Technologie Grid-Entwicklern weiterhin die Flexibilität allgemeinerer Rahmenbedingungen für die Entwicklung ihrer Grid-Programme. Somit stellt Xgrid die wichtigsten Vorteile für beide Anforderungen bereit.

  • Einfache Grid-Konfiguration und -Implementierung

  • Unkompliziertes und trotzdem flexibles Übertragen von Aufträgen

  • Automatische Controller-Erkennung durch Agenten und Clients

  • Flexible, auf offenen Standards basierende Architektur

  • Unterstützung für das UNIX-Sicherheitsmodell, einschließlich Gesamt-Authentifizierung über Kerberos oder einer regulären Identifikationsüberprüfung durch Kennwörter

  • Auswahl zwischen einem Befehlszeilenfenster oder einem API-basierten Modell für die Grid-Interaktion
