Controller

Der Xgrid-Controller verwaltet die Kommunikation zwischen den Rechenressourcen eines Grids. Für den Controller ist Mac OS X Server 10.4 (oder neuer) erforderlich. Der Controller akzeptiert Netzwerkverbindungen von Clients und Agenten. Er empfängt Aufträge von Clients, unterteilt die Aufträge in Aufgaben, sendet die Aufgaben an die Agenten und übermittelt die Ergebnisse zurück an die Clients.

In einem Teilnetzwerk können zwar mehrere Xgrid-Controller ausgeführt werden, es ist jedoch nur ein Controller pro logischem Grid möglich.

An jeden Controller können beliebig viele Agenten angeschlossen sein, Apple hat in Tests allerdings maximal 128 Agenten pro Controller eingesetzt. Durch die Software wird die Anzahl der Agenten nicht eingeschränkt und Benutzer von Xgrid können auf eigenes Risiko pro Controller auch mehr als 128 Agenten verwenden. Die theoretische Obergrenze an Clients wird dabei durch die Anzahl der verfügbaren Anschlüsse am Controller-System bestimmt.
