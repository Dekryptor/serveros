Lokale Grids

Systeme, die in einem Unternehmen, in Computer-Räumen einer Universität oder anderen Administrationsumgebungen gemeinsam verwaltet werden, können häufig problemlos zu einen Grid verbunden werden, wodurch eine effiziente Nutzung der freien Ressourcen von Desktop-Computern ermöglicht wird. Diese Systeme befinden sich häufig in einem LAN (Local Area Network) und werden im Allgemeinen von einer Organisation verwaltet. Daher bieten sie eine gute Netzwerkleistung und umfangreiche Möglichkeiten für die Verwaltung.

Da diese Systeme in der Regel tagsüber auch als Clientcomputer verwendet werden, können Benutzer Grid-Aufgaben durch Bewegen der Maus, Zurücksetzen des Systems oder sogar durch versehentliches Trennen des Systems vom Netzwerk leicht unterbrechen. In einem solchen Fall kann eine Aufgabe eines Xgrid-Auftrags möglicherweise fehlschlagen. Der Xgrid-Controller weist die fehlgeschlagene Aufgabe einem anderen Agenten zu, sodass der Auftrag erfolgreich beendet wird.

In lokalen Grids wird die Leistung durch solche Situationen und durch die variierende Leistung aller Agenten im Grid eingeschränkt.
