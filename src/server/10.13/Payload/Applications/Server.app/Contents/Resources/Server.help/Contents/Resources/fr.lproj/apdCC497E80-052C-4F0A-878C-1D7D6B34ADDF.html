Gestion AirPort

La sous-fenêtre Gestion AirPort s’affiche si l’interface réseau principale de votre serveur est connectée à Internet par l’intermédiaire d’une borne d’accès AirPort Extreme ou d’une Time Capsule AirPort. De plus, l’option Partage de connexion d’une borne d’accès AirPort Extreme ou d’une Time Capsule AirPort doit être définie sur « Partager une adresse IP publique » (c’est-à-dire une connexion Internet).

Utilisez la sous-fenêtre Gestion AirPort pour autoriser macOS Server à gérer une borne d’accès AirPort Extreme (802.11n) ou une AirPort Time Capsule.

Pour autoriser la gestion AirPort, saisissez le mot de passe de l’appareil AirPort (et non le mot de passe du réseau Wi-Fi).

L’app Server configure le serveur DHCP de votre appareil AirPort pour réserver l’adresse IP de votre serveur et pour toujours attribuer cette adresse IP à votre serveur.

Vous pouvez utiliser l’app Server pour désigner les services publics accessibles depuis les ordinateurs, via Internet. macOS Server configure votre appareil AirPort pour exposer ces services publics sur Internet. Le processus d’exposition de services individuels sur Internet se nomme mappage de ports ou redirection de ports.

Si vous souhaitez exposer des services publics sur Internet par le biais d’un appareil AirPort ou un autre routeur, et si votre serveur fournit un service VPN aux utilisateurs ayant leur propre appareil AirPort ou un autre routeur configuré afin de partager une connexion Internet, l’adresse IP de votre serveur ne peut pas commencer par les trois mêmes nombres que les adresses IP des utilisateurs du VPN, par exemple 10.0.1 ou 192.168.1.

Pour gérer votre appareil AirPort une fois l’app Server installée, sélectionnez votre appareil en haut de la barre latérale de l’app, dans la section Matériel.
