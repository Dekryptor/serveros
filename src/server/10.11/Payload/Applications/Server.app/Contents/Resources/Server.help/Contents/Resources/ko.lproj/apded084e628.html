접근성 테스트

연결 테스트로 사용자의 서버가 인터넷에 접근 가능한 정도를 알 수 있습니다.

사용자 서버가 공인 IP 주소를 사용하고 라우터나 게이트웨이 없이 인터넷에 직접 연결되어 있는 경우 인터넷상의 클라이언트가 일부 활성 서비스를 사용할 수 없을 수 있습니다. 연결 테스트는 해당 클라이언트에게 사용 가능한 서비스를 보여줍니다.

연결 테스트 작동 방법

Server App은 안전하게 Apple에 연결하여 Apple에서 자동화된 서버에게 활성화된 서비스마다 사용자의 서버에 연결해보기를 요청합니다. Apple 서버가 로컬 네트워크 밖에 있기 때문에 외부 클라이언트가 사용자의 서버에 연결하도록 시뮬레이션할 수 있습니다. 결과가 Server App에 표시됩니다.

Apple 서버는 다음을 확인합니다.

  • 외부에서 접근 가능한 IP 주소.

  • 공용 호스트 이름.

  • 서버에서 활성화된 각 서비스에 대한 핸드셰이크. 연결 사용 가능한 시간 외에 아무 정보도 교환되지 않습니다.

  • 필요한 경우 적절하게 구성된 포트 포워딩.

Apple은 어떠한 개인 정보도 유지하지 않으며 Apple의 서버가 확인할 연결의 종류를 알도록 사용자의 구성에 대한 정보만 전송합니다.

연결 테스트를 사용해야 하는 이유

이 서비스는 외부 소스가 사용자의 네트워크 연결을 확인하도록 하는 안정적인 방법을 제공합니다. 모든 것이 올바르게 구성되어 있지만 연결을 테스트할 방법이 없으면 좌절감을 줍니다. 이 서비스는 구성의 실제 외부 접근성 제공 여부에 대한 즉각적인 반응을 제공합니다.
