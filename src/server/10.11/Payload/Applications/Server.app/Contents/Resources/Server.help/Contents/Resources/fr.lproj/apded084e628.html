Test de l’accessibilité

Le test de l’accessibilité permet de déterminer le degré d’accessibilité de votre serveur sur Internet.

À moins que votre serveur soit doté d’une adresse IP publique et qu’il soit connecté à Internet sans routeur ni passerelle, il est possible que certains services actifs ne soient pas accessibles aux clients accédant au serveur par Internet. Le test de l’accessibilité indique les services accessibles à ces clients.

Comment fonctionne le test de l’accessibilité ?

L’app Server se connecte de manière sécurisée à Apple et demande aux serveurs automatisés d’Apple de tenter de se connecter à votre serveur pour chacun des services activés. Étant donné que les serveurs Apple se trouvent en dehors de votre réseau local, ils peuvent simuler la connexion à votre serveur par des clients externes. Les résultats sont indiqués dans l’app Server.

Les serveurs Apple vérifient la présence des éléments suivants :

  • Une adresse IP accessible de manière externe.

  • Un nom d’hôte public.

  • Une connexion à chacun des services activés sur le serveur. Aucune information autre que les informations relatives à l’accessibilité de la connexion ne sont échangées.

  • Un transfert de port correctement configuré, le cas échéant.

Apple ne conserve aucune de vos données personnelles et transmet uniquement les informations relatives à votre configuration requises pour que les serveurs Apple sachent quels types de connexion rechercher.

Pourquoi utiliser le test de l’accessibilité ?

Ce service permet une vérification fiable de la connectivité de votre réseau à partir d’une source externe. Il peut être frustrant de penser que tout est configuré correctement, mais de n’avoir aucun moyen de tester les connexions. Ce service vous fournit des informations instantanées sur l’accessibilité externe permise par votre configuration.

Comment désactiver ce service ?

Pour désactiver le test de l’accessibilité, consultez la rubrique Désactiver les tests de l’accessibilité.
