Se connecter à votre serveur

Utilisez la sous-fenêtre « Connexion à votre serveur » pour configurer les noms dont les autres utilisateurs se servent pour se connecter à votre serveur.

Nom de l’ordinateur

Ce nom apparaît dans les sections AirDrop ou Partagés de la barre latérale du Finder sur d’autres ordinateurs Mac. Les autres ordinateurs peuvent utiliser le nom de l’ordinateur pour se connecter à votre serveur afin de partager des fichiers ou leur écran, si ces services sont activés.

Nom d’hôte

L’assistant réglages du serveur essaie de déterminer le nom d’hôte à partir du serveur DNS disponible sur le réseau.

Voici quelques raisons possibles si l’assistant réglages du serveur ne parvient pas à déterminer le nom d’hôte de votre serveur :

  • L’assistant ne trouve pas de serveur DNS (sur votre intranet ou sur Internet) configuré correctement pour votre serveur.

  • Votre serveur a reçu les mauvais réglages de connexion au réseau d’un serveur DHCP, qui peut être votre routeur Internet.

  • Votre intranet ne dispose d’aucun serveur DHCP ; vous devez saisir manuellement les réglages de connexion au réseau de votre réseau.

Si vous saisissez un nom d’hôte, respectez les règles suivantes :

Nom d’hôte d’un réseau local :

Pour permettre aux utilisateurs d’accéder à votre serveur uniquement depuis votre intranet, saisissez un nom d’hôte se terminant par .local, tel que server.local. Ce nom, également appelé nom d’hôte local, correspond à un réseau local.

Seuls les ordinateurs de votre sous-réseau IP local peuvent accéder à votre serveur à l’aide de son nom de réseau local. Habituellement, les ordinateurs du même sous-réseau possèdent des adresses IP commençant par les trois mêmes groupes de chiffres, par exemple 192.168.1. Les ordinateurs Windows doivent posséder Services d’impression Bonjour pour Windows, disponible à l’adresse http://support.apple.com/kb/DL999? viewlocale=fr_FR.

Si le nom d’hôte et le nom de réseau local d’un serveur sont identiques, ce dernier ne prend pas en charge Kerberos ou l’authentification unique.

Nom d’hôte d’un réseau privé :

Pour permettre aux utilisateurs d’accéder à ce serveur à l’aide d’un nom d’hôte depuis votre intranet ou à l’aide d’une connexion VPN depuis Internet, saisissez un nom d’hôte se terminant par .private, tel que serveur.exemple.private.

Si votre intranet possède un serveur DNS, il doit être configuré de façon à utiliser le nom d’hôte du serveur pour rechercher son adresse IP et vice versa. Cela signifie que votre serveur DNS doit posséder un enregistrement pour les recherches en avant (un enregistrement A) et un enregistrement pour les recherches inversées (enregistrement PTR). Demandez à l’administrateur de votre serveur DNS de configurer ces enregistrements pour votre serveur.

Si votre intranet ne dispose d’aucun serveur DNS, l’assistant réglages du serveur configure votre serveur afin de fournir un service DNS minimal. Les utilisateurs peuvent ainsi utiliser le nom d’hôte de votre serveur sur votre intranet.

Nom d’hôte sur Internet :

Pour que les utilisateurs puissent accéder à ce serveur à l’aide du même nom d’hôte sur votre intranet et sur Internet, saisissez le nom d’hôte utilisé pour l’enregistrement du nom de domaine.

Les serveurs DNS que vous utilisez pour Internet doivent être configurés de façon à utiliser le nom d’hôte du serveur pour rechercher son adresse IP et vice versa. Cela signifie que votre serveur DNS doit posséder un enregistrement pour les recherches en avant (un enregistrement A) et un enregistrement pour les recherches inversées (enregistrement PTR). Demandez à votre FAI ou à votre service d’hébergement DNS de configurer ces enregistrements DNS.

Si votre intranet dispose d’un serveur DNS, il doit être configuré avec des enregistrements pour la recherche en avant et pour la recherche inversée. Demandez à l’administrateur de votre serveur DNS intranet de configurer ces enregistrements pour votre serveur.

Si votre intranet ne dispose d’aucun serveur DNS, l’assistant réglages du serveur configure votre serveur afin de fournir un service DNS minimal. Les utilisateurs peuvent ainsi utiliser le nom d’hôte de votre serveur sur votre intranet.

Remarque : Soyez prudent dans le choix du nom d’hôte. Il sera difficile de le modifier par la suite (cela peut perturber les utilisateurs, entre autres). Par exemple, les utilisateurs doivent configurer à nouveau leurs ordinateurs et informer les autres que leur adresse électronique a changé.

Adresse réseau

L’adresse IP numérique qui identifie de façon unique votre serveur sur le réseau. L’adresse réseau peut être tout d’abord définie sur l’adresse IP que votre serveur reçoit du serveur DHCP sur votre réseau. Cliquez sur Changer de réseau pour modifier l’adresse réseau.

Votre serveur devrait toujours posséder la même adresse IP, afin que d’autres utilisateurs puissent s’y connecter de manière fiable. Si l’adresse IP de votre serveur change, il se peut que les utilisateurs ne puissent plus s’y connecter.

Voici plusieurs façons de conserver la même adresse IP :

  • Si votre serveur se connecte à Internet via un routeur DHCP, telle qu’une borne d’accès AirPort, vous pouvez configurer le DHCP afin qu’il attribue toujours la même adresse IP au serveur. Cette fonctionnalité est également appelée mappage statique ou réservations DHCP. Si votre réseau possède un serveur DHCP distinct, demandez à l’administrateur de configurer le mappage statique de votre serveur.

  • Si votre serveur possède une connexion directe à Internet, par exemple grâce à un modem DSL ou câble n’étant pas un routeur, obtenez une adresse IP statique (fixe) auprès de votre FAI. Au besoin, vous pouvez généralement en obtenir une moyennant des frais supplémentaires.

  • Si votre serveur n’est pas connecté à Internet et que votre intranet ne possède pas de serveur DHCP, cliquez sur Changer de réseau pour saisir manuellement une adresse IP statique.

Changer de réseau

Cliquez sur cette option pour modifier l’adresse réseau du serveur ou d’autres réglages de la connexion réseau.

Votre serveur possède également un nom d’hôte local, tel que serveur.local (non affiché dans cette sous-fenêtre), qui n’est enregistré auprès d’aucun serveur DNS. Les ordinateurs sur le même intranet (sous-réseau IP) que votre serveur peuvent utiliser son nom d’hôte local au lieu du nom d’hôte enregistré pour s’y connecter. Le nom d’hôte local est généralement dérivé du nom de l’ordinateur. Pour le modifier, utilisez l’app Server une fois la configuration du serveur terminée. (Sélectionnez votre serveur, puis choisissez la sous-fenêtre Réseau.)
