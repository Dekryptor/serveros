可连通性测试

可连通性测试让您知道服务器访问互联网的状况。

除非您的服务器有公共 IP 地址并在没有路由器或网关的情况下连接到互联网，否则可能不是所有活跃的服务都可用于互联网上的客户端。可连通性测试显示了可以为这些客户端所用的服务。

可连通性测试的工作原理是什么？

Server 应用安全连接到 Apple 并请求 Apple 的自动化服务器尝试连接到已启用每项服务的服务器。因为 Apple 服务器在本地网络外，所以它们可以模拟外部客户端以尝试连接到您的服务器。结果会显示在 Server 应用中。

Apple 服务器会检查：

  • 外部可访问的 IP 地址。

  • 公共主机名。

  • 与服务器上的每项已启用的服务握手。无信息（除连接可用性）交换。

  • 如果需要，需正确配置端口转发。

Apple 不会保留您的任何个人信息，它仅传输关于您的配置的信息，以让 Apple 的服务器知道要检查的连接类型。

为什么要使用可连通性测试？

此服务提供了一种可靠的方式，使外部来源检查您的网络连接。当您认为一切都正确配置，但无法测试连接时，会感觉沮丧。此服务会提供即时反馈，让您知道您的配置是否可真正进行外部访问。

如何停用可连通性测试？

若要停用可连通性测试，请参阅停用可连通性测试。
