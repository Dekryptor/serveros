Test di raggiungibilità

Il test di raggiungibilità ti consente di sapere quanto è accessibile il server a Internet.

A meno che il server non abbia un indirizzo IP pubblico e non sia connesso a Internet senza un router o un gateway, non tutti i servizi attivi potrebbero essere disponibili ai client su Internet. Il test di raggiungibilità mostra quali servizi sono disponibili a tali client.

Come funziona il test di raggiungibilità?

L'app Server si connette in maniera sicura a Apple e chiede a server automatizzati Apple di tentare di connettersi al tuo server per ogni servizio abilitato. Dato che i server Apple si trovano al di fuori del tuo network locale, possono simulare client esterni che tentano di connettersi al tuo server. I risultati vengono mostrati nell'app Server.

I server Apple verificano la presenza di:

  • Un indirizzo IP accessibile esternamente.

  • Uno nome host pubblico.

  • Un handshake con ciascun servizio abilitato sul server. Non viene scambiata nessun'altra informazione a parte la disponibilità della connessione.

  • Inoltro delle porte correttamente configurato, se necessario.

Apple non conserva alcuna informazione personale e trasmette solo le informazioni sulla configurazione necessarie a comunicare ai server Apple quali tipi di connessione verificare.

Perché dovrei utilizzare il test di raggiungibilità?

Il servizio ti fornisce un metodo affidabile per fare in modo che una sorgente esterna verifichi la tua connettività network. È frustrante quando si crede di aver configurato tutto in maniera corretta, ma non si ha alcun modo per verificare le connessioni. Il servizio ti fornisce un riscontro istantaneo e ti permette di sapere se la configurazione consente l'accesso esterno.

Come disabilito il servizio?

Per disabilitare il test di raggiungibilità, consulta Disabilitare i test di raggiungibilità.
