Acceso al servidor

El panel “Acceso al servidor” aparece para que pueda especificar si el nombre de host de su servidor funciona para todo Internet o solo en su intranet (red local). Especifique si los usuarios acceden a su servidor usando el nombre de host solo en su intranet o tanto en la intranet como en Internet. Puede introducir el nombre de host del servidor en el panel siguiente

Red local

Seleccione esta opción para permitir a los usuarios acceder al servidor solo desde la intranet. Los usuarios pueden acceder a su servidor usando su nombre de red local, también conocido como nombre de host local. Normalmente, se trata del nombre del ordenador del servidor con el sufijo .local.

Solo los ordenadores de su subred IP local pueden acceder a su servidor mediante su nombre de red local. Normalmente, los ordenadores de la misma subred tienen direcciones IP que comienzan por los tres mismos números; por ejemplo, 192.168.1.

Si el nombre de host del servidor es su nombre de red local, el servidor no admite Kerberos o la autenticación mediante inicio de sesión sencillo.

Red local y VPN

Seleccione esta opción para permitir a los usuarios acceder a este servidor usando su nombre de host en su intranet y usando una conexión VPN desde Internet. Para permitir a los usuarios de Internet acceder al servidor de manera segura a través de una conexión VPN, active el servicio VPN en la aplicación Server después de finalizar la configuración del servidor.

Si la intranet cuenta con un servidor DNS, deberá configurarse para usar el nombre de host del servidor para buscar su dirección IP y viceversa. Esto significa que el servidor DNS debe tener un registro para la búsqueda hacia adelante (registro A) y un registro para la búsqueda hacia atrás (registro PTR). Solicite al administrador del servidor DNS que configure estos registros DNS para el servidor.

Si la intranet no cuenta con un servidor DNS, el asistente de configuración del servidor configurará el servidor para proporcionar un servicio DNS mínimo de modo que los usuarios puedan usar el nombre de host de su servidor en la intranet.

Nombre de dominio

Seleccione esta opción para permitir a los usuarios acceder al servidor usando el mismo nombre de host en Internet y en su intranet.

Los servidores DNS que use para Internet deberán configurarse para usar el nombre de host del servidor para buscar su dirección IP y viceversa. Esto significa que el servidor DNS debe tener un registro para la búsqueda hacia adelante (registro A) y un registro para la búsqueda hacia atrás (registro PTR). Solicite a su proveedor de Internet o al servicio de alojamiento de DNS que configure estos registros DNS.

Si la intranet cuenta con un servidor DNS, también deberá configurarse con registros para búsqueda hacia adelante y hacia atrás. Solicite al administrador del servidor DNS de la intranet que configure estos registros DNS.

Si la intranet no cuenta con un servidor DNS, el asistente de configuración del servidor configurará el servidor para proporcionar un servicio DNS mínimo de modo que los usuarios puedan usar el nombre de host de su servidor en la intranet.
