서버에 연결하기

서버에 연결하기 패널을 이용하면 다른 사용자가 사용자의 서버에 연결하는 데 사용하는 이름을 설정할 수 있습니다.

컴퓨터 이름

이 이름은 다른 Mac 컴퓨터의 Finder 사이드바에 있는 AirDrop 또는 공유 부분에 나타납니다. 파일 공유나 화면 공유가 켜져 있는 경우 다른 컴퓨터가 사용자 서버에 연결하기 위해 이 컴퓨터 이름을 사용할 수 있습니다.

호스트 이름

서버 설정 지원은 네트워크 상에서 사용할 수 있는 DNS 서버로부터 사용자에 대한 호스트 이름을 정하려고 시도합니다.

다음은 서버 설정 지원이 사용자 서버의 호스트 이름을 정할 수 없는 몇 가지 경우입니다.

  • 지원은 사용자 서버에 대해 올바르게 구성된 (인트라넷 또는 인터넷에서) DNS 서버를 찾을 수 없습니다.

  • 사용자의 서버는 DHCP 서버(사용자의 인터넷 라우터일 수 있음)로부터 잘못된 네트워크 연결 설정을 받았습니다.

  • 사용자의 인트라넷에는 DHCP 서버가 없어서 서버의 네트워크 연결 설정을 수동으로 입력해야 했습니다.

호스트 이름을 입력하는 경우 다음 안내를 따르십시오.

로컬 네트워크의 호스트 이름:

다른 사용자들이 사용자의 인트라넷에서만 서버에 접근하도록 하려면 server.local과 같이 .local로 끝나는 호스트 이름을 입력하십시오. 이것은 로컬 호스트 이름이라고도 하는 로컬 네트워크 이름입니다.

로컬 IP 서브네트 상의 컴퓨터만 해당 로컬 네트워크 이름을 사용하여 사용자의 서버에 접근할 수 있습니다. 일반적으로 동일한 서브네트 상의 컴퓨터는 동일한 숫자 세 세트로 시작하는 IP 주소를 가집니다(예: 192.168.1). Windows 컴퓨터는 Windows용 Bonjour 프린트 서비스가 설치되어 있어야 합니다. support.apple.com/kb/DL999? viewlocale=ko_KR에서 다운로드할 수 있습니다.

서버의 호스트 이름이 해당 로컬 네트워크 이름인 경우 해당 서버는 Kerberos 또는 SSO(Single Sign-On) 인증을 지원하지 않습니다.

개인 네트워크용 호스트 이름:

인트라넷에서 해당 호스트 이름을 사용하거나 인터넷에서 VPN 연결을 사용하여 다른 사용자가 이 서버에 접근하도록 하려면 server.example.private과 같이 .private으로 끝나는 호스트 이름을 입력하십시오.

사용자의 인트라넷에 DNS 서버가 있는 경우 서버 호스트 이름을 사용하여 해당 IP 주소를 찾거나 그 반대로 찾을 수 있도록 구성되어야 합니다. 이는 DNS 서버에 정방향 조회(A 레코드) 및 역방향 조회(PTR 레코드)에 대한 레코드가 있어야 한다는 뜻입니다. DNS 서버 관리자에게 사용자의 서버에 대해 이러한 DNS 레코드를 구성하도록 요청하십시오.

사용자의 인트라넷에 DNS 서버가 없는 경우 서버 설정 지원이 최소 DNS 서비스를 제공하도록 사용자의 서버를 구성하여 다른 사용자들이 인트라넷에서 사용자 서버의 호스트 이름을 사용할 수 있습니다.

인터넷용 호스트 이름:

다른 사용자가 인터넷 및 사용자의 인트라넷 상에서 동일한 호스트 이름을 사용하여 사용자의 서버에 접근할 수 있도록 하려면 도메인 이름 등록을 통해 등록된 해당 호스트 이름을 입력하십시오.

인터넷용으로 사용하는 DNS 서버는 서버 호스트 이름을 사용하여 해당 IP 주소를 찾거나 그 반대로 찾을 수 있도록 구성되어야 합니다. 이는 DNS 서버에 정방향 조회용 레코드(A 레코드) 및 역방향 조회용 레코드(PTR 레코드)가 있어야 한다는 뜻입니다. ISP 또는 DNS 호스팅 서비스 제공업체에 이러한 DNS 레코드 구성을 요청하십시오.

사용자의 인트라넷에 DNS 서버가 있는 경우에도 정방향 조회 및 역방향 조회에 대한 레코드로 구성되어야 합니다. 사용자의 인트라넷 DNS 서버 관리자에게 이러한 DNS 레코드를 구성하도록 요청하십시오.

사용자의 인트라넷에 DNS 서버가 없는 경우 서버 설정 지원이 최소 DNS 서비스를 제공하도록 사용자의 서버를 구성하여 다른 사용자들이 인트라넷에서 사용자 서버의 호스트 이름을 사용할 수 있습니다.

참고:  호스트 이름을 신중히 선택하십시오. 나중에 호스트 이름을 변경하면 특히 다른 사용자들에게 지장을 줄 수 있습니다. 예를 들어 다른 사용자들은 해당 컴퓨터를 다시 구성해야 하며 다른 사람들에게 이메일 주소가 변경되었다고 알려야 합니다.

네트워크 주소

네트워크 상에서 사용자의 서버를 고유하게 확인하는 숫자 형식의 IP 주소입니다. 네트워크 주소는 처음에 네트워크 상의 DHCP 서버에서 사용자의 서버가 받는 IP 주소로 설정되어야 합니다. 네트워크 변경을 클릭하여 네트워크 주소를 변경하십시오.

사용자의 서버가 항상 동일한 IP 주소를 갖기 때문에 다른 사용자가 안정적으로 서버에 연결할 수 있습니다. 서버의 IP 주소가 변경되면 다른 사용자가 서버에 연결하지 못할 수 있습니다.

다음은 서버의 IP 주소 변경을 관리하는 방법입니다.

  • 사용자의 서버가 AirPort Base Station처럼 DHCP를 통해 라우터로부터 인터넷 연결을 얻는 경우 항상 동일한 IP 주소를 사용자 서버에 할당하도록 해당 라우터의 DHCP를 구성할 수 있습니다. 이 기능을 고정 매핑 또는 DHCP 예약이라고도 합니다. 사용자의 네트워크에 다른 DHCP 서버를 있는 경우 해당 관리자에게 사용자의 서버에 대해 고정 매핑을 구성하도록 요청하십시오.

  • 예를 들어 DSL 모뎀 또는 라우터가 아닌 케이블 모뎀을 통해서와 같이 사용자의 서버가 인터넷에 직접 연결되는 경우, 사용자의 ISP로부터 정적(고정 ) IP 주소를 얻습니다. 필요한 경우 일반적으로 요금을 지불하고 업그레이드할 수 있습니다.

  • 사용자의 서버가 인터넷에 연결되지 않았고 사용자의 인트라넷에 DHCP 서버가 없는 경우 네트워크 변경을 클릭하여 고정 IP 주소를 수동으로 입력하십시오.

네트워크 변경

클릭하면 해당 서버의 네트워크 주소 또는 다른 네트워크 연결 설정을 변경할 수 있습니다.

사용자의 Mac에는 로컬 호스트 이름도 있습니다. 이 이름은 server.local과 같이 이 패널에 표시되지 않고 DNS 서버로 등록되지 않습니다. 사용자의 서버와 동일한 인트라넷(IP 서브네트) 상의 컴퓨터는 등록된 호스트 이름 대신 해당 로컬 호스트 이름을 사용하여 사용자의 서버에 연결할 수 있습니다. 로컬 호스트 이름은 기본적으로 해당 컴퓨터 이름에서 파생됩니다. 변경하려면 서버 설정 완료 후 Server App을 사용하십시오. (사용자의 서버를 선택하고 네트워크 패널에서 선택하십시오.)
