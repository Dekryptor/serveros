Bereikbaarheid

Met een bereikbaarheidstest kunt u nagaan in hoeverre uw server toegankelijk is via het internet.

Tenzij uw server een openbaar IP-adres heeft en zonder router of gateway met het internet is verbonden, zijn mogelijk niet alle actieve voorzieningen beschikbaar voor clients op het internet. Een bereikbaarheidstest laat zien welke voorzieningen beschikbaar zijn voor deze clients.

De werking van een bereikbaarheidstest

Vanuit het programma Server wordt een beveiligde verbinding met Apple tot stand gebracht en worden de geautomatiseerde servers bij Apple gevraagd om voor elke ingeschakelde voorziening verbinding te maken met uw server. Omdat de Apple servers zich buiten uw lokale netwerk bevinden, kunnen ze zich voordoen als externe clients die een verbinding met uw server tot stand proberen te brengen. De resultaten worden in het programma Server weergegeven.

De Apple servers controleren de verbindingen op de volgende punten:

  • Een extern toegankelijk IP-adres.

  • Een openbare hostnaam.

  • Een handshake met elke voorziening die op de server is ingeschakeld. Er wordt geen informatie uitgewisseld, afgezien van informatie over de beschikbaarheid van de verbinding.

  • Correcte configuratie van poortkoppeling, indien nodig.

Apple bewaart hierbij geen persoonlijke gegevens en verzendt alleen die gegevens over uw configuratie die nodig zijn om de servers te laten weten welke verbindingen moeten worden gecontroleerd.

Het nut van een bereikbaarheidstest

Deze voorziening biedt een betrouwbare manier om uw netwerkverbindingen te laten controleren door een externe partij. Het is frustrerend als u denkt dat alles correct is geconfigureerd, maar u de verbindingen niet kunt testen. Deze voorziening geeft meteen aan of bij uw configuratie toegang van buitenaf mogelijk is.
