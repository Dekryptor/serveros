사용자의 서버에 접근하기

나타나는 사용자의 서버에 접근 중 패널에서 사용자 서버의 호스트 이름이 전체 인터넷에 대해 동작하도록 할지, 사용자의 인트라넷(로컬 네트워크)에서만 동작하도록 할지 지정할 수 있습니다. 다른 사용자가 인트라넷에서만 또는 인트라넷과 인터넷에서 해당 호스트 이름을 사용하여 사용자 서버에 접근하도록 할지 지정하십시오. 다음 패널에서 해당 서버의 호스트 이름을 입력할 수 있습니다.

로컬 네트워크

이 옵션을 선택하여 다른 사용자가 인트라넷에서만 사용자의 서버에 접근할 수 있도록 할 수 있습니다. 사용자는 로컬 호스트 이름이라고도 하는 로컬 네트워크 이름을 사용하여 사용자 서버에 접근할 수 있습니다. 일반적으로 해당 서버의 컴퓨터 이름에 .local이 추가된 이름입니다.

로컬 IP 서브네트 상의 컴퓨터만 해당 로컬 네트워크 이름을 사용하여 사용자의 서버에 접근할 수 있습니다. 일반적으로 동일한 서브네트 상의 컴퓨터는 동일한 숫자 세 개로 시작하는 IP 주소를 가집니다(예: 192.168.1).

서버의 호스트 이름이 해당 로컬 네트워크 이름인 경우 해당 서버는 Kerberos 또는 SSO(Single Sign-On) 인증을 지원하지 않습니다.

로컬 네트워크 및 VPN

인트라넷에서 호스트 이름을 사용하고 인터넷에서 VPN 연결을 사용하여 다른 사용자가 이 서버에 접근하도록 하려면 이 옵션을 선택하십시오. 인터넷 사용자가 VPN 연결을 통해 사용자 서버에 안전하게 접근하도록 하려면 서버 설정을 완료한 후 Server App에서 VPN 서비스를 켜십시오.

사용자의 인트라넷에 DNS 서버가 있는 경우 서버 호스트 이름을 사용하여 해당 IP 주소를 찾거나 그 반대로 찾을 수 있도록 구성되어야 합니다. 이는 DNS 서버에 정방향 조회용 레코드(A 레코드) 및 역방향 조회용 레코드(PTR 레코드)가 있어야 한다는 뜻입니다. DNS 서버 관리자에게 사용자의 서버에 대해 이러한 DNS 레코드를 구성하도록 요청하십시오.

사용자의 인트라넷에 DNS 서버가 없는 경우 서버 설정 지원이 최소 DNS 서비스를 제공하도록 사용자의 서버를 구성하여 다른 사용자들이 인트라넷에서 사용자 서버의 호스트 이름을 사용할 수 있습니다.

도메인 이름

다른 사용자가 인터넷 및 사용자의 인트라넷 상에서 동일한 호스트 이름을 사용하여 사용자의 서버에 접근할 수 있도록 하려면 이 옵션을 선택하십시오.

인터넷용으로 사용하는 DNS 서버는 서버 호스트 이름을 사용하여 해당 IP 주소를 찾거나 그 반대로 찾을 수 있도록 구성되어야 합니다. 이는 DNS 서버에 정방향 조회용 레코드(A 레코드) 및 역방향 조회용 레코드(PTR 레코드)가 있어야 한다는 뜻입니다. ISP 또는 DNS 호스팅 서비스 제공업체에 이러한 DNS 레코드 구성을 요청하십시오.

사용자의 인트라넷에 DNS 서버가 있는 경우에도 정방향 조회 및 역방향 조회에 대한 레코드로 구성되어야 합니다. 사용자의 인트라넷 DNS 서버 관리자에게 이러한 DNS 레코드를 구성하도록 요청하십시오.

사용자의 인트라넷에 DNS 서버가 없는 경우 서버 설정 지원이 최소 DNS 서비스를 제공하도록 사용자의 서버를 구성하여 다른 사용자들이 인트라넷에서 사용자 서버의 호스트 이름을 사용할 수 있습니다.
