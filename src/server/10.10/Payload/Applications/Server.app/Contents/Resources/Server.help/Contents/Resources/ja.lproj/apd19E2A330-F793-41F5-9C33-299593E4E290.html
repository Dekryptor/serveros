サーバに接続する

「サーバへの接続」パネルで、ほかのユーザがサーバに接続するために使用する名前を設定します。

コンピュータ名

これは、ほかの Mac コンピュータの Finder のサイドバーの「AirDrop」または「共有」セクションに表示されます。ほかのコンピュータでファイル共有や画面共有のサービスが入になっている場合は、ファイル共有または画面共有のために、そのコンピュータがこのコンピュータ名を使ってサーバに接続することができます。

ホスト名

サーバ設定アシスタントは、ネットワーク上の DNS サーバからホスト名を判別しようとします。

サーバ設定アシスタントがサーバのホスト名を判別できない原因のいくつかを、次に示します：

  • アシスタントが、イントラネットやインターネット上の、サーバ用に適切に構成されていない DNS サーバを見つけることができない。

  • サーバが DHCP サーバから正しくないネットワーク接続設定を受信して、それがインターネットルーターである可能性がある。

  • イントラネットに DHCP サーバが設置されていないため、サーバのネットワーク接続設定を手動で入力する必要がある。

ホスト名を入力する場合は、次のガイドラインに従ってください：

ローカルネットワーク用のホスト名：

ユーザがイントラネットからのみサーバにアクセスできるようにするには、server.local のような .local で終わるホスト名を入力します。これはローカルネットワーク名で、ローカルホスト名とも呼ばれます。

ローカル IP サブネット上のコンピュータのみが、そのローカルネットワーク名を使ってサーバにアクセスすることができます。通常、同じサブネット上のコンピュータは、同じ 3 組の数字（192.168.1 など）で始まる IP アドレスを持ちます。Windows コンピュータには、Bonjour Print Services for Windows がインストールされている必要があります。これは support.apple.com/kb/DL999?viewlocale=ja_JP で入手できます。

サーバのホスト名がそのローカルネットワーク名の場合、そのサーバでは Kerberos またはサインオン認証はサポートされません。

プライベートネットワーク用のホスト名：

ユーザがこのサーバに、イントラネット上ではそのホスト名を使って、インターネットからは VPN 接続を使ってアクセスできるようにするには、server.example.private のような .private で終わるホスト名を入力します。

イントラネットに DNS サーバが設置されている場合は、サーバのホスト名を使ってその IP アドレスを検索したり、逆に IP アドレスを使ってサーバのホスト名を検索するように構成する必要があります。これは、DNS サーバには前方参照用のレコード（A レコード）と逆引き参照用のレコード（PTR レコード）が必要であることを意味します。これらの DNS レコードをサーバ用に構成するときは、DNS サーバ管理者に依頼してください。

イントラネットに DNS サーバが設置されていない場合は、サーバ設定アシスタントによりサーバが最低限の DNS サービスを提供するように構成されるため、ユーザはイントラネット上のサーバのホスト名を使用することができます。

インターネット用のホスト名：

ユーザがインターネットとイントラネットで同じホスト名を使ってサーバにアクセスできるようにするには、ドメイン名登録業者に登録したホスト名を入力します。

インターネット用に使用する DNS サーバは、サーバのホスト名を使ってその IP アドレスを検索したり、逆に IP アドレスを使ってサーバのホスト名を検索したりするように構成する必要があります。これは、DNS サーバには前方参照用のレコード（A レコード）と逆引き参照用のレコード（PTR レコード）が必要であることを意味します。これらの DNS レコードを構成するときは、ISP または DNS ホスティングサービスに依頼してください。

イントラネットに DNS サーバが設置されている場合は、前方参照と逆引き参照用のレコードで構成する必要もあります。これらの DNS レコードを構成するときは、イントラネットの DNS サーバ管理者に依頼してください。

イントラネットに DNS サーバが設置されていない場合は、サーバ設定アシスタントによりサーバが最低限の DNS サービスを提供するように構成されるため、ユーザはイントラネット上のサーバのホスト名を使用することができます。

参考： ホスト名は慎重に選択してください。ホスト名を後で変更すると、特にユーザに対して、混乱を生じさせる可能性があります。たとえば、ユーザは自分のコンピュータを再構成して、自分のメールアドレスが変更されたことをほかのユーザに伝える必要があります。

ネットワークアドレス

ネットワーク上のサーバを一意に識別する数字の IP アドレスです。ネットワークアドレスは、最初は、サーバがネットワーク上の DHCP サーバから受信する IP アドレスに設定される場合があります。ネットワークアドレスを変更するには、「ネットワークを変更」をクリックします。

サーバでは常に同じ IP アドレスが使用されるので、ほかのユーザはサーバに確実に接続できます。サーバの IP アドレスが変更されると、ユーザがサーバに接続できなくなる場合があります。

サーバの IP アドレスが変更されないようにする方法は、次の通りです：

  • サーバが DHCP を使って AirMac ベースステーションなどのルーターからインターネットに接続する場合は、サーバに常に同じ IP アドレスを割り当てるようにルーターの DHCP を構成できます。この機能は、「静的マッピング」または「DHCP の予約」とも呼ばれています。ネットワークに別の DHCP サーバが設置されている場合は、サーバ用に静的マッピングを構成するように管理者に依頼してください。

  • サーバがルーターではない DSL モデムやケーブルモデムなどを経由してインターネットに直接接続する場合は、ISP から静的（固定）IP アドレスを取得してください。必要に応じて、通常は有料の静的 IP アドレスにアップグレードできます。

  • サーバがインターネットに接続されていない場合で、イントラネットに DHCP サーバが設置されていないときは、「ネットワークを変更」をクリックして静的 IP アドレスを手動で入力できます。

ネットワークを変更

サーバのネットワークアドレスなどのネットワーク接続設定を変更する場合にクリックします。

お使いの Mac にも server.local のようなローカルホスト名があります。これはこのパネルには表示されず、どの DNS サーバにも登録されません。サーバと同じイントラネット（IP サブネット）上のコンピュータは、登録済みのホスト名の代わりにローカルホスト名を使ってサーバに接続できます。ローカルホスト名は、デフォルトではコンピュータ名から付けられます。変更するには、サーバ設定の終了後に Server アプリケーションを使用します。（サーバを選択し、「ネットワーク」パネルを選択します。）
