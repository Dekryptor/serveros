访问服务器

“访问服务器”面板出现后，您可以指定服务器的主机名是在整个互联网上使用还是仅在内部网（本地网络）上使用。指定用户是只能在内部网上通过服务器的主机名来访问服务器，还是在内部网和互联网上都可以通过服务器的主机名来访问服务器。您可以在下一个面板中输入服务器的主机名。

本地网络

如果只想让用户从内部网访问服务器，请选择此选项。用户可以使用您的服务器的本地网络名称（也称为“本地主机名”）来访问它。通常是服务器的电脑名称附加 .local。

只有在您的本地 IP 子网上的电脑可以使用您服务器的本地网络名称访问您的服务器。通常情况下，同一个子网上的电脑的 IP 地址会以相同的三个数字开头，如 192.168.1。

如果服务器的主机名称是它的本地网络名称，则该服务器不支持 Kerberos 或单点登录鉴定。

本地网络和 VPN

如果想要让用户使用服务器在内部网上的主机名以及从互联网使用 VPN 连接来访问此服务器，请选择此选项。若要让互联网用户通过 VPN 连接安全地访问您的服务器，请在完成服务器设置后打开 Server 应用程序中的 VPN 服务。

如果您的内部网有 DNS 服务器，它必须经过配置以使用服务器的主机名来查找它的 IP 地址以及使用它的 IP 地址来查找主机名。这意味着 DNS 服务器必须有用于正向查找的记录（A 记录）以及用于反向查找的记录（PTR 记录）。请要求您的 DNS 服务器管理员为您配置这些 DNS 记录。

如果您的内部网没有 DNS 服务器，服务器设置助理会配置您的服务器以提供最小化 DNS 服务，以便用户可以使用您的服务器在内部网上的主机名称。

域名称

如果想要让用户使用服务器在互联网和内部网上的同一主机名访问您的服务器，请选择此选项。

您用于互联网的 DNS 服务器必须经过配置以使用服务器的主机名来查找它的 IP 地址以及使用它的 IP 地址来查找主机名。这意味着 DNS 服务器必须有用于正向查找的记录（A 记录）以及用于反向查找的记录（PTR 记录）。请要求您的 ISP 或 DNS 托管服务为您配置这些 DNS 记录。

如果您的内部网具有 DNS 服务器，它也必须使用记录进行配置以用于正向查找和反向查找。请要求您的内部网 DNS 服务器管理员为您配置这些 DNS 记录。

如果您的内部网没有 DNS 服务器，服务器设置助理会配置您的服务器以提供最小化 DNS 服务，以便用户可以使用您的服务器在内部网上的主机名称。
