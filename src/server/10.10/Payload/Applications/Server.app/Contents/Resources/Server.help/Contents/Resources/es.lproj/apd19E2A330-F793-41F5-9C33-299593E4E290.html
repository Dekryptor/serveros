Conexión con su servidor

Use el panel “Conectando con su servidor” para configurar los nombres que usan otros para conectarse con su servidor.

Nombre del ordenador

Aparece en las secciones AirDrop o Compartir de la barra lateral del Finder en otros ordenadores Mac. Otros ordenadores pueden usar el nombre del ordenador para conectarse a su servidor para compartir archivos o la pantalla si tiene activados estos servicios.

Nombre de host

El asistente de configuración del servidor trata de determinar el nombre de host a partir de un servidor DNS disponible en la red.

Algunas razones por las que el asistente de configuración del servidor podría no ser capaz de determinar el nombre de host del servidor:

  • El asistente no encuentra un servidor DNS (en la intranet o en Internet) correctamente configurado para el servidor.

  • El servidor ha recibido unos ajustes de conexión de red erróneos de un servidor DHCP, que podría ser su router de Internet.

  • Su intranet no tiene un servidor DHCP, por lo que debe introducir manualmente los ajustes de la conexión de red del servidor.

Si introduce un nombre de host, siga estas directrices:

Nombre de host para la red local:

Para que otros usuarios puedan acceder a su servidor solo desde su intranet, introduzca un nombre de host que termine en .local, como servidor.local. Se trata de un nombre de red local, conocido también como nombre de host local.

Solo los ordenadores de su subred IP local pueden acceder a su servidor mediante su nombre de red local. Normalmente, los ordenadores de la misma subred tienen direcciones IP que comienzan por los tres mismos números; por ejemplo, 192.168.1. Los ordenadores Windows deben tener instalado los Servicios de impresión Bonjour para Windows. Se encuentra disponible en http://support.apple.com/kb/DL999?viewlocale=es_ES.

Si el nombre de host del servidor es su nombre de red local, el servidor no admite Kerberos o la autenticación mediante inicio de sesión sencillo.

Nombre de host para la red privada:

Para permitir a los usuarios que accedan a este servidor usando su nombre de host en su intranet y usando una conexión VPN desde Internet, introduzca un nombre de host que termine en .private, como servidor.example.private.

Si la intranet cuenta con un servidor DNS, deberá configurarse para usar el nombre de host del servidor para buscar su dirección IP y viceversa. Esto significa que el servidor DNS debe tener registros para la búsqueda hacia adelante (registro A) y hacia atrás (registro PTR). Solicite al administrador del servidor DNS que configure estos registros DNS para el servidor.

Si la intranet no cuenta con un servidor DNS, el asistente de configuración del servidor configurará el servidor para proporcionar un servicio DNS mínimo de modo que los usuarios puedan usar el nombre de host de su servidor en la intranet.

Nombre de host para Internet:

Para permitir a los usuarios que accedan al servidor usando el mismo nombre de host en Internet y en su intranet, introduzca el nombre de host que ha registrado en un registrador de nombres de dominio.

Los servidores DNS que use para Internet deberán configurarse para usar el nombre de host del servidor para buscar su dirección IP y viceversa. Esto significa que el servidor DNS debe tener un registro para la búsqueda hacia adelante (registro A) y un registro para la búsqueda hacia atrás (registro PTR). Solicite a su proveedor de Internet o al servicio de alojamiento de DNS que configure estos registros DNS.

Si la intranet cuenta con un servidor DNS, también deberá configurarse con registros para búsqueda hacia adelante y hacia atrás. Solicite al administrador del servidor DNS de la intranet que configure estos registros DNS.

Si la intranet no cuenta con un servidor DNS, el asistente de configuración del servidor configurará el servidor para proporcionar un servicio DNS mínimo de modo que los usuarios puedan usar el nombre de host de su servidor en la intranet.

Nota:  Seleccione con cuidado el nombre del host. Si cambia el nombre del host más adelante, podría resultar perjudicial, sobre todo para los usuarios. Por ejemplo, los usuarios deberían reconfigurar sus ordenadores y decirle a otras personas que han cambiado de dirección de correo electrónico.

Dirección de red

La dirección IP numérica que identifica su servidor en la red de manera exclusiva. La dirección de red puede configurarse inicialmente en la que recibe su servidor del servidor DHCP de la red. Haga clic en “Cambiar de red” para cambiar la dirección de red.

El servidor tiene siempre la misma dirección IP, de modo que los demás usuarios pueden conectarse a él con plena fiabilidad. Si cambia la dirección IP del servidor, es posible que los usuarios no pudieran conectarse a él.

Algunas formas de impedir que cambie la dirección IP del servidor:

  • Si su servidor obtiene su conexión a Internet de un router con DHCP, tal como una estación base AirPort, puede configurar el DHCP del router para que siempre asigne la misma dirección IP al servidor. Esta característica se conoce también como asignación estática o reserva de DHCP. Si su red tiene un servidor DHCP distinto, pida al administrador que configure la asignación estática para su servidor.

  • Si su servidor tiene una conexión directa a Internet (por ejemplo, a través de un módem DSL o un módem por cable que no sea un router), solicite una dirección IP estática (fija) a su proveedor de Internet. En caso necesario, normalmente podrá hacerlo pagando una cuota.

  • Si su servidor no está conectado a Internet y su intranet no tiene servidor DHCP, Haga clic en “Cambiar de red” para introducir manualmente una dirección IP estática.

Cambiar de red

Haga clic para cambiar la dirección de red del servidor u otros ajustes de la conexión de red.

Su Mac también tiene un nombre de host local, como servidor.local, que no aparece en este panel y no está registrado con ningún servidor DNS. Los equipos de la misma intranet (subred de IP) que su servidor pueden usar su nombre de host local en lugar del nombre de host registrado para conectarse a su servidor. El nombre de host local deriva por omisión del nombre del ordenador. Para cambiarlo, use la aplicación Server cuando finalice la configuración del servidor. (Seleccione el servidor y elija el panel Red.)
