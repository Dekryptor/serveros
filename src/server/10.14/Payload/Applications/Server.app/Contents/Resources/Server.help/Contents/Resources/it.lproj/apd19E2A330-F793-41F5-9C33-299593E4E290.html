Connettersi al server

Utilizza il pannello “Connettersi al server” per configurare i nomi che gli altri utilizzano per connettersi al server.

Nome computer

Questo appare nelle sezioni AirDrop o Condivisi della barra laterale del Finder su altri computer Mac. Altri computer possono utilizzare il nome del computer per connettersi al server per la condivisione dei file o dello schermo, se ha questi servizi attivi.

Nome host

L'assistente di configurazione del server prova a determinare il nome host da un server DNS disponibile sulla rete.

Di seguito alcune ragioni per cui l'assistente di configurazione del server potrebbe non essere in grado di determinare il nome host del server:

  • L'assistente non riesce a trovare un server DNS (sulla intranet o su Internet) configurato correttamente per il server.

  • Il server ha ricevuto le impostazioni di connessione di rete non corrette da un server DHCP, che potrebbe essere il router Internet.

  • L'intranet non dispone di un server DHCP, quindi devi inserire manualmente le impostazioni di connessione di rete del server.

Se inserisci un nome host, segui queste linee guida:

Nome host per una rete locale:

Peri consentire agli utenti di accedere al server solo dalla intranet, inserisci un nome host che termina con .local, ad esempio server.local. Questo è un nome di rete locale, anche chiamato nome host locale.

Solo i computer nella sottorete IP possono accedere al server usando il suo nome di rete locale. Normalmente, i computer nella stessa sottorete hanno indirizzi IP che cominciano con le stesse tre serie di numeri, ad esempio, 192.168.1. I computer Windows devono avere installato i Servizi di stampa Bonjour per Windows. Sono disponibili all'indirizzo support.apple.com/kb/DL999?viewlocale=it_IT.

Se il nome host del server corrisponde al nome della rete locale, il server non supporta Kerberos o l'autenticazione Single Sign-On.

Nome host per Internet:

Per permettere agli utenti di accedere al server usando lo stesso nome host su Internet e sull'intranet, inserisci il nome host che hai registrato con un sito di registrazione di nomi domino.

I server DNS che utilizzi per Internet devono essere configurati per utilizzare il nome host del server per cercare il suo indirizzo IP e viceversa. Ciò significa che il server DNS deve disporre di un record per la ricerca in avanti (un record A) e per la ricerca inversa (un record PTR). Chiedi al servizio di hosting DNS o all'ISP di configurare per te questi record DNS.

Se l'intranet dispone di un server DNS, anche questo deve essere configurato con i record per la ricerca in avanti e inversa. Chiedi all'amministratore del server DNS intranet di configurare questi record DNS per te.

Se l'intranet non dispone di un server DNS, l'assistente configurazione server configura il server per fornire un servizio DNS minimo in modo che gli utenti possano utilizzare il nome host del server sull'intranet.

Nota: Seleziona un nome host con attenzione. Modificare il nome host in un secondo momento potrebbe non essere conveniente, in particolare per gli utenti. Per esempio, gli utenti devono riconfigurare i loro computer e informare altri che i loro indirizzi e-mail sono cambiati.

Indirizzo di rete

L'indirizzo IP numerico che identifica in modo univoco il server sulla rete. L'indirizzo di rete potrebbe essere impostato inizialmente come l'indirizzo IP che il server riceve dal server DHCP sulla rete. Fai clic su “Modifica rete” per cambiare l'indirizzo di rete.

Il server dovrebbe utilizzare sempre lo stesso indirizzo IP, in modo che gli altri utenti possano connettersi al server in modo affidabile. Se l'indirizzo IP del server cambia, gli utenti non sono in grado di connettersi ad esso.

Di seguito i modi per mantenere immutato l'indirizzo IP del server:

  • Se il server riceve la connessione Internet da un router con DHCP, puoi configurare il DHCP del router per assegnare sempre lo stesso indirizzo IP al server. Questa caratteristica viene anche chiamata mappatura statica o prenotazioni DHCP. Se la rete dispone di un altro server DHCP, chiedi all'amministratore di configurare la mappatura statica per il server.

  • Se il server dispone di una connessione Internet diretta, ad esempio tramite un modem DSL o via cavo diverso da un router, ottieni un indirizzo IP statico (fisso) dal tuo ISP. Se necessario, puoi generalmente acquistarne uno.

  • Se il server non è connesso a Internet e la Intranet non dispone di un server DHCP, fai clic su “Modifica rete” per inserire manualmente un indirizzo IP statico.

Modifica rete

Fai clic per modificare l'indirizzo di rete del server o altre impostazioni di connessione di rete.

Il server ha anche un nome host locale, come server.local, che non viene mostrato in questo pannello e non viene registrato con nessun server DNS. I computer sulla stessa Intranet (sottorete IP) come il server, possono utilizzare il suo nome host locale anziché il nome host registrato per connettersi al server. Solitamente il nome host locale deriva dal nome del computer. Per modificarlo, utilizza Server app dopo aver terminato la configurazione del server. Seleziona il server, quindi scegli il pannello Rete.
