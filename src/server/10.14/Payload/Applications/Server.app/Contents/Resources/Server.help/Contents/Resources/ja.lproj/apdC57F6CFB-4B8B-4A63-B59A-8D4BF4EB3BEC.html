サーバにアクセスする

「サーバへのアクセス」パネルでは、サーバのホスト名をインターネット全体で使用するか、イントラネット（ローカルネットワーク）上のみで使用するかを指定できます。ユーザがイントラネット上のホスト名のみを使ってサーバにアクセスするか、イントラネットとインターネット上のホスト名を使ってサーバにアクセスするかを指定します。次のパネルで、サーバのホスト名を入力できます。

ローカルネットワーク

ユーザがイントラネットからしかサーバにアクセスできないようにするには、このオプションを選択します。ユーザは、ローカルネットワーク名（「ローカルホスト名」とも呼ばれる）を使用してサーバにアクセスできます。これは、通常は .local が付いたサーバのコンピュータ名です。

ローカル IP サブネット上のコンピュータのみが、そのローカルネットワーク名を使ってサーバにアクセスすることができます。通常、同じサブネット上のコンピュータは、同じ 3 つの数字（192.168.1 など）で始まる IP アドレスを持ちます。

サーバのホスト名がそのローカルネットワーク名の場合、そのサーバでは Kerberos またはサインオン認証はサポートされません。

ドメイン名

ユーザがインターネットとイントラネットで同じホスト名を使ってサーバにアクセスできるようにするには、このオプションを選択します。

インターネット用に使用する DNS サーバは、サーバのホスト名を使ってその IP アドレスを検索したり、逆に IP アドレスを使ってサーバのホスト名を検索したりするように構成する必要があります。これは、DNS サーバには前方参照用のレコード（A レコード）と逆引き参照用のレコード（PTR レコード）が必要であることを意味します。これらの DNS レコードを構成するときは、ISP または DNS ホスティングサービスに依頼してください。

イントラネットに DNS サーバが設置されている場合は、前方参照と逆引き参照用のレコードで構成する必要もあります。これらの DNS レコードを構成するときは、イントラネットの DNS サーバ管理者に依頼してください。

イントラネットに DNS サーバが設置されていない場合は、サーバ設定アシスタントによりサーバが最低限の DNS サービスを提供するように構成されるため、ユーザはイントラネット上のサーバのホスト名を使用することができます。
