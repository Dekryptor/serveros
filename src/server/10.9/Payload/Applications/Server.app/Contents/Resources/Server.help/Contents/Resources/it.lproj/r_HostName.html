Accedere al server

Viene visualizzato il pannello di accesso al server dove occorre specificare se il nome host del server funziona per l'intera Internet o solo per l'intranet (network locale). Specifica se gli utenti accedono al server usando il suo nome host solo sull'intranet o sull'intranet e su Internet Puoi inserire il nome host del server nel pannello successivo.
OpzioneDescrizione

Network locale

Seleziona questa opzione se vuoi permettere agli utenti di accedere al server solo dall'intranet, seleziona questa opzione. Gli utenti possono accedere al server utilizzando il nome del network locale, noto anche come “nome host locale”. Normalmente è il nome del computer seguito da .local.

Solo i computer nella sottorete IP possono accedere al server usando il suo nome network locale. Normalmente, i computer nella stessa sottorete hanno indirizzi IP che cominciano con gli stessi tre numeri, ad esempio, 192.168.1. I computer Windows devono avere installato i Servizi di stampa Bonjour per Windows. Sono disponibili all'indirizzo support.apple.com/kb/DL999?viewlocale=it_IT.

Se il nome host del server corrisponde al suo nome network locale, il server non supporta Kerberos o l'autenticazione ad accesso singolo.

Network locale e VPN

Seleziona questa opzione se vuoi permettere agli utenti di accedere a questo server usando il suo nome host dall'intranet e usando una connessione VPN da Internet. Per permettere agli utenti Internet di accedere al server in modo sicuro tramite una connessione VPN, attiva il servizio VPN in Server app dopo aver terminato la configurazione del server.

Se l'intranet dispone di un server DNS, questo deve essere configurato per utilizzare il nome host del server per cercare il suo indirizzo IP e viceversa. Ciò significa che il server DNS deve disporre di record per la ricerca in avanti (un record A) e per la ricerca inversa (un record PTR). Chiedi all'amministratore del server DNS di configurare questi record DNS sul server.

Se l'intranet non dispone di un server DNS, l'assistente configurazione server configura il server per fornire un servizio DNS minimo in modo che gli utenti possano utilizzare il nome host del server sull'intranet.

Nome dominio

Seleziona questa opzione se vuoi permettere agli utenti di accedere al server usando lo stesso nome host su Internet e sull'intranet.

I server DNS che utilizzi per Internet devono essere configurati per utilizzare il nome host del server per cercare il suo indirizzo IP e viceversa. Ciò significa che il server DNS deve disporre di record per la ricerca in avanti (un record A) e per la ricerca inversa (un record PTR). Chiedi al servizio di hosting DNS o all'ISP di configurare per te questi record DNS.

Se l'intranet dispone di un server DNS, anche questo deve essere configurato con i record per la ricerca in avanti e inversa. Chiedi all'amministratore del server DNS intranet di configurare questi record DNS per te.

Se l'intranet non dispone di un server DNS, l'assistente configurazione server configura il server per fornire un servizio DNS minimo in modo che gli utenti possano utilizzare il nome host del server sull'intranet.
