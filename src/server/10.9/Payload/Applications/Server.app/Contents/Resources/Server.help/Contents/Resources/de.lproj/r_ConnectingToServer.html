Herstellen der Verbindung zum Server

Richten Sie die Adresse und die Namen ein, die Ihren Server im Netzwerk identifizieren.

Richten Sie im Bereich „Mit Ihrem Server verbinden“ Namen ein, über die andere Benutzer die Verbindung zu Ihrem Server herstellen.

OptionBeschreibung

Computername

Wird in den Abschnitten „AirDrop“ oder „Freigaben“ in der Finder-Seitenleiste auf anderen Mac-Computern angezeigt. Andere Computer können mithilfe des Computernamens die Verbindung zu Ihrem Server herstellen und die Dateifreigabe oder die Bildschirmfreigabe nutzen, wenn diese dort aktiviert sind.

Hostname

Der Assistent für die Serverkonfiguration versucht, den Hostnamen von einem verfügbaren DNS-Server im Netzwerk abzurufen.

In folgenden Fällen kann der Assistent möglicherweise den Hostnamen Ihres Servers nicht ermitteln:
  • Der Assistent findet keinen DNS-Server im Intranet oder im Internet, der für Ihren Server korrekt konfiguriert ist.
  • Ihr Server hat fehlerhafte Netzwerkverbindungseinstellungen von einem DHCP-Server erhalten, bei dem es sich um Ihren Internet-Router handeln kann.
  • In Ihrem Intranet gibt es keinen DHCP-Server, daher müssen Sie die Netzwerkverbindungseinstellungen Ihres Servers manuell eingeben.
Weitere Informationen zur richtigen Konfiguration der DNS-Server finden Sie im Folgenden unter den jeweiligen Optionen. Wenn Ihr Server fehlerhafte Netzwerkverbindungseinstellungen von einem DHCP-Server erhalten hat oder es im Intranet keinen DHCP-Server gibt, können Sie einen solchen einrichten oder die Netzwerkverbindungseinstellungen Ihres Servers im Bereich „Netzwerk wechseln“ ändern.

Beachten Sie folgende Richtlinien bei der Eingabe eines Hostnamens:

Hostname für ein lokales Netzwerk:

Wenn nur Benutzer im Intranet die Möglichkeit haben sollen, auf Ihren Server zuzugreifen, geben Sie einen Hostnamen ein, der auf „.local“ endet, etwa „server.local“. Hierbei handelt es sich um einen lokalen Netzwerknamen bzw. lokalen Hostnamen.

Nur Computer im lokalen IP-Teilnetz können anhand des lokalen Netzwerknamens auf Ihren Server zugreifen. Normalerweise haben Computer im selben Teilnetz IP-Adressen, die mit denselben drei Zahlen beginnen, z. B. 192.168.1. Auf Windows-Computern müssen die Bonjour-Druckdienste für Windows installiert sein. Diese sind unter „support.apple.com/kb/DL999?viewlocale=de_DE“ zu finden.

Wenn der Hostname des Servers mit dessen Namen im lokalen Netzwerk identisch ist, unterstützt der Server weder die Authentifizierung über Kerberos noch die Gesamtauthentifizierung (Single Sign-On, SSO).

Hostname für ein privates Netzwerk:

Wenn es Benutzern möglich sein soll, auf diesen Server über dessen Hostnamen im Intranet oder über eine VPN-Verbindung im Internet zuzugreifen, geben Sie einen Hostnamen ein, der auf „.private“ endet, etwa „server.example.private“.

Wenn es im Intranet einen DNS-Server gibt, muss dieser so konfiguriert werden, dass die IP-Adresse des Servers anhand dessen Hostnamen ermittelt wird und umgekehrt. Das bedeutet, dass der DNS-Server über Datensätze für eine Vorwärtssuche (A-Datensatz) und für eine Rückwärtssuche (PTR-Datensatz) verfügen muss. Bitten Sie Ihren DNS-Serveradministrator, diese DNS-Datensätze für Ihren Server zu konfigurieren.

Gibt es keinen DNS-Server im Intranet, konfiguriert der Assistent für die Serverkonfiguration Ihren Server so, dass dieser minimale DNS-Dienste bereitstellt, damit die Benutzer den Hostnamen des Servers im Intranet verwenden können.

Hostname für das Internet:

Wenn es möglich sein soll, dass Benutzer anhand desselben Hostnamens im Internet und im Intranet auf Ihren Server zugreifen, geben Sie den Hostnamen ein, den Sie bei einer Registrierungsstelle für Domain-Namen registriert haben.

Die DNS-Server für das Internet müssen so konfiguriert werden, dass die IP-Adresse des Servers anhand dessen Hostnamen ermittelt wird und umgekehrt. Das bedeutet, dass der DNS-Server über Datensätze für eine Vorwärtssuche (A-Datensatz) und für eine Rückwärtssuche (PTR-Datensatz) verfügen muss. Bitten Sie Ihren Internetanbieter oder DNS-Hostingdienst, diese DNS-Datensätze für Sie zu konfigurieren.

Gibt es im Intranet einen DNS-Server, muss auch dieser mit den entsprechenden Datensätzen konfiguriert werden. Bitten Sie Ihren Intranet-DNS-Serveradministrator, diese DNS-Datensätze für Sie zu konfigurieren.

Gibt es keinen DNS-Server im Intranet, konfiguriert der Serverassistent Ihren Server so, dass dieser minimale DNS-Dienste bereitstellt, damit die Benutzer den Hostnamen des Servers im Intranet verwenden können.

Anmerkung: Wählen Sie den Hostnamen mit Bedacht. Eine spätere Änderung des Namens kann vor allem für die Benutzer sehr störend sein und zu Betriebsunterbrechungen führen. Die Benutzer müssen dann möglicherweise ihre Computer neu konfigurieren und andere darüber informieren, dass sich ihre E-Mail-Adresse geändert hat.

Netzwerkadresse

Dies ist die numerische IP-Adresse, die Ihren Server eindeutig im Netzwerk identifiziert. Die Netzwerkadresse kann anfangs auf die IP-Adresse eingestellt sein, die Ihr Server vom DHCP-Server im Netzwerk erhält. Klicken Sie auf „Netzwerk wechseln“, um die Netzwerkadresse zu ändern.

Ihr Server muss immer dieselbe IP-Adresse haben, damit andere Benutzer zuverlässig eine Verbindung zum Server herstellen können. Wenn sich die IP-Adresse des Servers ändert, können die Benutzer möglicherweise keine Verbindung mehr zum Server herstellen.

Eine Änderung der IP-Adresse des Servers können Sie wie folgt verhindern:

  • Wenn Ihr Server über einen Router mit DHCP, etwa eine AirPort-Basisstation, die Verbindung zum Internet herstellt, können Sie den DHCP-Dienst des Routers so konfigurieren, dass dem Server stets dieselbe IP-Adresse zugewiesen wird. Diese Vorgehensweise wird als „statische Zuordnung“ oder „DHCP-Reservierung“ bezeichnet. Wenn Ihr Netzwerk einen anderen DHCP-Server nutzt, bitten Sie den Administrator, eine statische Zuordnung für Ihren Server zu konfigurieren.
  • Wenn Ihr Server direkt mit dem Internet verbunden ist, etwa über ein DSL-Modem oder ein Kabelmodem ohne Router, fordern Sie bei Ihrem Internetanbieter eine statische (feste) IP-Adresse an. Falls nötig, können Sie ein kostenpflichtiges Upgrade durchführen.
  • Wenn Ihr Server nicht mit dem Internet verbunden ist und es im Intranet keinen DHCP-Server gibt, klicken Sie dazu auf „Netzwerk wechseln“, um manuell eine statische IP-Adresse einzugeben.

Netzwerk wechseln

Klicken Sie auf diese Option, um die Netzwerkadresse des Servers oder andere Netzwerkverbindungseinstellungen zu ändern.

Ihr Mac hat auch einen lokalen Hostnamen wie „server.local“, der in diesem Bereich nicht angezeigt wird und bei keinem DNS-Server registriert ist. Computer im selben Intranet (IP-Teilnetz) wie Ihr Server können den lokalen Hostnamen statt des registrierten Hostnamens verwenden, um eine Verbindung zu Ihrem Server herzustellen. Der lokale Hostname wird standardmäßig vom Computernamen abgeleitet. Sie können ihn mit der Server-App nach dem Konfigurieren des Servers ändern. (Wählen Sie Ihren Server und dann den Bereich „Netzwerk“ aus.)
