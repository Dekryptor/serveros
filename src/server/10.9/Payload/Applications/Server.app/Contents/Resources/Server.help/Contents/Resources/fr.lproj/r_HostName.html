Accès à votre serveur

La sous-fenêtre Accès à votre serveur apparaît, dans laquelle vous pouvez définir si le nom d’hôte de votre serveur fonctionne pour Internet ou seulement sur votre intranet (réseau local). Indiquez si les utilisateurs accèdent au serveur en utilisant son nom d’hôte sur votre intranet seulement, ou sur intranet et Internet. Entrez le nom d’hôte du serveur dans la sous-fenêtre suivante.
OptionDescription

Réseau local

Sélectionnez cette option pour permettre aux utilisateurs d’accéder à votre serveur uniquement depuis votre intranet. Les utilisateurs peuvent accéder à votre serveur en utilisant son nom de réseau local, aussi appelé « nom d’hôte local ». Il correspond généralement au nom de l’ordinateur du serveur, suivi de .local.

Seuls les ordinateurs de votre sous-réseau IP local peuvent accéder à votre serveur à l’aide de son nom de réseau local. Habituellement, les ordinateurs du même sous-réseau possèdent des adresses IP commençant par les trois mêmes chiffres, par exemple 192.168.1. Les ordinateurs Windows doivent posséder Services d’impression Bonjour pour Windows, disponible à l’adresse http://support.apple.com/kb/DL999?viewlocale=fr_FR.

Si le nom d’hôte et le nom de réseau local d’un serveur sont identiques, ce dernier ne prend pas en charge Kerberos ou l’authentification unique.

Réseau local et VPN

Sélectionnez cette option pour permettre aux utilisateurs d’accéder à ce serveur à l’aide d’un nom d’hôte depuis votre intranet ou à l’aide d’une connexion VPN depuis Internet. Pour permettre aux utilisateurs d’Internet d’accéder à votre serveur de manière sécurisée par le biais d’une connexion VPN, activez le service VPN dans l’app Server une fois la configuration du serveur terminée.

Si votre intranet possède un serveur DNS, il doit être configuré de façon à utiliser le nom d’hôte du serveur pour rechercher son adresse IP et vice versa. Cela signifie que votre serveur DNS doit posséder des enregistrements pour des recherches en avant (un enregistrement A) et inversées (enregistrement PTR). Demandez à l’administrateur de votre serveur DNS de configurer ces enregistrements pour votre serveur.

Si votre intranet ne dispose d’aucun serveur DNS, l’assistant réglages du serveur configure votre serveur afin de fournir un service DNS minimal. Les utilisateurs peuvent ainsi utiliser le nom d’hôte de votre serveur sur votre intranet.

Nom de domaine

Sélectionnez cette option pour permettre aux utilisateurs d’accéder à votre serveur à l’aide du même nom d’hôte sur votre intranet et sur Internet.

Les serveurs DNS que vous utilisez pour Internet doivent être configurés de façon à utiliser le nom d’hôte du serveur pour rechercher son adresse IP et vice versa. Cela signifie que votre serveur DNS doit posséder des enregistrements pour des recherches en avant (un enregistrement A) et inversées (enregistrement PTR). Demandez à votre FAI ou à votre service d’hébergement DNS de configurer ces enregistrements DNS.

Si votre intranet dispose d’un serveur DNS, il doit être configuré avec des enregistrements pour la recherche en avant et inversée. Demandez à l’administrateur de votre serveur DNS intranet de configurer ces enregistrements pour votre serveur.

Si votre intranet ne dispose d’aucun serveur DNS, l’assistant réglages du serveur configure votre serveur afin de fournir un service DNS minimal. Les utilisateurs peuvent ainsi utiliser le nom d’hôte de votre serveur sur votre intranet.
