Zugreifen auf Ihren Server

Der Bereich „Zugreifen auf Ihren Server“ wird eingeblendet, damit Sie angeben können, ob der Hostname des Servers für das gesamte Internet gilt oder nur für Ihr Intranet (lokales Netzwerk). Geben Sie an, ob die Benutzer nur im Intranet oder aber im Intranet und im Internet über den Hostnamen auf Ihren Server zugreifen. Sie können den Hostnamen des Servers im nächsten Bereich eingeben.
OptionBeschreibung

Lokales Netzwerk

Wählen Sie diese Option aus, damit Benutzer nur im Intranet auf den Server zugreifen können. Benutzer können auf Ihren Server durch Eingabe seines lokalen Netzwerknamens (auch als „lokaler Hostname“ bezeichnet) zugreifen. Er entspricht normalerweise dem Namen des Computers, an den „.local“ angehängt wird.

Nur Computer im lokalen IP-Teilnetz können anhand des lokalen Netzwerknamens auf Ihren Server zugreifen. Normalerweise haben Computer im selben Teilnetz IP-Adressen, die mit denselben drei Zahlen beginnen, z. B. 192.168.1. Auf Windows-Computern müssen die Bonjour-Druckdienste für Windows installiert sein. Diese sind unter „support.apple.com/kb/DL999?viewlocale=de_DE“ zu finden.

Wenn der Hostname des Servers mit dessen Namen im lokalen Netzwerk identisch ist, unterstützt der Server weder die Authentifizierung über Kerberos noch die Gesamtauthentifizierung (Single Sign-On, SSO).

Lokales Netzwerk und VPN

Wählen Sie diese Option aus, damit Benutzer auf diesen Server über dessen Hostnamen im Intranet und über eine VPN-Verbindung im Internet zugreifen können. Aktivieren Sie nach der Installation des Servers den VPN-Dienst in der Server-App, damit Internetbenutzer über eine VPN-Verbindung sicher auf den Server zugreifen können.

Wenn es im Intranet einen DNS-Server gibt, muss dieser so konfiguriert werden, dass die IP-Adresse des Servers anhand dessen Hostnamen ermittelt wird und umgekehrt. Das bedeutet, dass der DNS-Server über Datensätze für eine Vorwärtssuche (A-Datensatz) und für eine Rückwärtssuche (PTR-Datensatz) verfügen muss. Bitten Sie Ihren DNS-Serveradministrator, diese DNS-Datensätze für Ihren Server zu konfigurieren.

Gibt es keinen DNS-Server im Intranet, konfiguriert der Serverassistent Ihren Server so, dass dieser minimale DNS-Dienste bereitstellt, damit die Benutzer den Hostnamen des Servers im Intranet verwenden können.

Domain-Name

Wählen Sie diese Option aus, damit Benutzer mit demselben Hostnamen im Internet und im Intranet auf Ihren Server zugreifen können.

Die DNS-Server für das Internet müssen so konfiguriert werden, dass die IP-Adresse des Servers anhand dessen Hostnamen ermittelt wird und umgekehrt. Das bedeutet, dass der DNS-Server über Datensätze für eine Vorwärtssuche (A-Datensatz) und für eine Rückwärtssuche (PTR-Datensatz) verfügen muss. Bitten Sie Ihren Internetanbieter oder DNS-Hostingdienst, diese DNS-Datensätze für Sie zu konfigurieren.

Gibt es im Intranet einen DNS-Server, muss auch dieser mit den entsprechenden Datensätzen konfiguriert werden. Bitten Sie Ihren Intranet-DNS-Serveradministrator, diese DNS-Datensätze für Sie zu konfigurieren.

Gibt es keinen DNS-Server im Intranet, konfiguriert der Serverassistent Ihren Server so, dass dieser minimale DNS-Dienste bereitstellt, damit die Benutzer den Hostnamen des Servers im Intranet verwenden können.
