連接您的伺服器

「連接您的伺服器」面板會顯示，因此您指定伺服器的主機名稱是適用於整個 Internet 或僅適用於您的內部網路（區域網路）。 指定使用者是否在您的內部網路上，或在您的內部網路和 Internet 上，使用伺服器的主機名稱來進行連接。 您可以在下一個面板中輸入伺服器的主機名稱。
選項說明

區域網路

選擇此選項來只讓使用者從內部網路連接您的伺服器。使用者可以使用此區域網路名稱（也稱為本機主機名稱）來連接您的伺服器。其通常是伺服器的電腦名稱，後面附加 .local。

只有在您區域 IP 子網路上的電腦可使用伺服器的區域網路名稱來進行連接。通常，電腦若位於相同的子網路上，其 IP 位址會有相同的前三個數字，例如 192.168.1。Windows 電腦必須已安裝「Windows 版 Bonjour 列印服務」。其可在 support.apple.com/kb/DL999?viewlocale=zh_TW 取得。

如果伺服器的主機名稱是其區域網路名稱，伺服器便不支援 Kerberos 或單一登入認證。

區域網路和 VPN

若要讓使用者在您的內部網路上使用伺服器的主機名稱和從 Internet 使用 VPN 連線來進行連接，請選擇此選項。若要讓 Internet 使用者安全地透過 VPN 連線來連接伺服器，請在完成伺服器設定後，在 Server App 中啟用 VPN 服務。

如果您的內部網路有 DNS 伺服器，其必須設定為使用伺服器的主機名稱來查詢其 IP 位址，反之亦然。這表示 DNS 伺服器必須有向前查詢（A 記錄）和反向查詢（PTR 記錄）的記錄。請您的 DNS 伺服器管理者為伺服器設定這些 DNS 記錄。

如果您的內部網路沒有 DNS 伺服器，伺服器設定輔助程式會設定您的伺服器以提供最精簡的 DNS 服務，讓使用者可在您的內部網路上使用伺服器的主機名稱。

網域名稱

若要讓使用者在 Internet 和內部網路上使用相同的主機名稱來連接您的伺服器，請選擇此選項。

您用於 Internet 的 DNS 伺服器必須設定為使用伺服器的主機名稱來查詢其 IP 位址，反之亦然。這表示 DNS 伺服器必須有向前查詢（A 記錄）和反向查詢（PTR 記錄）的記錄。請您的 ISP 或 DNS 託管服務為您設定這些 DNS 記錄。

如果您的內部網路有 DNS 伺服器，其也必須使用向前查詢和反向查詢的記錄來加以設定。請您的內部網路 DNS 伺服器管理者為您設定這些 DNS 記錄。

如果您的內部網路沒有 DNS 伺服器，伺服器設定輔助程式會設定您的伺服器以提供最精簡的 DNS 服務，讓使用者可在您的內部網路上使用伺服器的主機名稱。
