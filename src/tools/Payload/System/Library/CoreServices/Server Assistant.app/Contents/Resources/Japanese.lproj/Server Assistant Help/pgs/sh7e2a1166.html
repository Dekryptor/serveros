ユーザとグループ

「ユーザとグループ」パネルで選択するオプションによって、サーバでユーザの認証に使われるユーザおよびグループアカウントの管理と、アクセスを許可するサービスの決定を、どのように行うかを指定します。

ユーザとグループを管理するためのオプション
ユーザとグループを作成

既存のディレクトリサーバがない小規模な組織用に単独のサーバを設定します。サーバはそれ自体のディレクトリサービスを Open Directory のマスターとして提供するため、サーバにはそれ自体のユーザおよびグループアカウントがあります。次のパネルで、設定したい基本的なサービスを選択できます。

ユーザとグループを読み込む

既存のディレクトリサーバがある組織のワークグループ用にサーバを設定します。サーバはそのディレクトリサーバからユーザを読み込むことができます。ユーザを個別に読み込むことができ、グループのメンバーを自動的に読み込むことができます。次のパネルで、サーバをディレクトリサーバに接続（バインド）します。また、サーバはそれ自体のディレクトリサービスを Open Directory のマスターとして提供するため、それ自体のユーザとグループのアカウントを持つことができます。後の方のパネルで、設定する基本的なサービスを選択できます。

手動で設定

多種多様なビジネスのニーズに対応できるよう、経験のある管理者がディレクトリサービス構成を完全に制御できるようにします。

このパネルの後で、サーバをディレクトリサーバに接続（バインド）するための情報や、Open Directory のマスターを作成してこれをディレクトリサーバにしたりするための情報を入力することができます。それらのオプションのいずれかを選択した場合は、設定する基本的なサービスを選択することもできます。ディレクトリサーバへの接続や Open Directory のマスターの作成を行わない場合は、設定後にディレクトリサービスとほかのすべてのサービスをユーザ自身が構成します。

注記： サーバがネットワークに接続されていない場合は、「手動で設定」を選択します。ほかの 2 つのオプションは、サーバのディレクトリサービスを構成します。これにはネットワーク接続が必要です。

「ユーザとグループ」パネルにこれらのオプションが表示されない場合は、Xsan プライマリ・メタデータ・コントローラ用の Mac OS X Server を設定しています。 「ユーザとグループ（Xsan）」を参照してください。
