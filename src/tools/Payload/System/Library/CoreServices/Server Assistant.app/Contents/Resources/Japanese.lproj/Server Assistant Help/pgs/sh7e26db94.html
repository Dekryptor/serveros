ホーム
ネットワーク

「ネットワーク」パネルでは、サーバのネットワークインターフェイスの接続設定を指定します。各インターフェイスの設定のグループが、ネットワーク接続サービスです。ネットワークインターフェイスはポートとも呼ばれ、プラグやケーブルを接続するインターフェイスの一部です。

サーバには、ユーザのコンピュータが接続するローカルネットワーク用のネットワーク接続サービスが必要です。これは通常は Ethernet ネットワークですが、AirMac ネットワークであることもあります。ISP（インターネット・サービス・プロバイダ）が PPPoE 経由の接続を必要としている場合は、PPPoE ネットワークサービス接続を構成する必要があります。サーバが複数のネットワークに接続している場合は、サーバが接続するネットワークごとにネットワーク接続サービスを構成する必要があります。

ネットワークサービス接続を構成するには：

  • リスト内のネットワーク接続サービスを選択してから、設定を指定します。さまざまな種類のネットワーク接続サービスの設定について詳しくは、以下のトピックを参照してください：

    Ethernet ネットワーク接続サービスを構成する
    AirMac ネットワーク接続サービスを構成する
    FireWire ネットワーク接続サービスを構成する
    PPPoE ネットワーク接続サービスを構成する
    LOM ポートのネットワーク接続サービスを構成する
    ネットワーク接続サービスを無効にする

  • 「追加」（＋）ボタンをクリックすることによって、ネットワーク接続サービスを追加します。操作方法については、「ネットワーク接続サービスを追加する」を参照してください。

  • ネットワーク接続サービスを選択し、「削除」（－）ボタンをクリックして、そのネットワークサービスを削除します。

  • アクション（歯車）ボタンがある場合は、それを使って、よく使うネットワーク接続サービスのみを表示するか、すべてのネットワーク接続サービスを表示します。

  • ポインタをリスト内のネットワーク接続サービスの上に移動して、ネットワーク接続サービスのハードウェア・アドレスとインターフェイス名を表示します。

  • リスト内でドラッグして、ネットワーク接続サービスの優先順位を変更します。ネットワーク上で通信を試みると、サーバはネットワーク接続サービスをリストされた順に使用します。

    最初のアクティブなネットワーク接続サービスが、サーバのプライマリネットワーク接続です。サーバは、ローカルネットワークの外でのすべての通信にこのサービスを使用します。

「ネットワーク」パネルには、最初はサーバの各 Ethernet インターフェイスのネットワーク接続サービスのリストが表示されます。サーバのネットワーク接続では Ethernet 接続が一般的なため、使用可能なほかのネットワークインターフェイスはリストに示されません。すべてのネットワークインターフェイスを表示するには、アクション（歯車）ボタンを使用します。サーバに Ethernet インターフェイスがない場合、「ネットワーク」パネルには使用可能なすべてのネットワークインターフェイスのリストが表示されます。 後でサーバを自動的に設定するためにプロファイルを作成する場合、最初は「ネットワーク」パネルにサービスが表示されません。

ほとんどの Macintosh コンピュータには、Ethernet ポートが内蔵されています。ワイヤレスネットワーク接続用の AirMac インターフェイス、ネットワーク接続に使用できる FireWire ポート、2 つ目の Ethernet ポートなど、追加のインターフェイスが内蔵されていたり追加されていたりする場合があります。

Intel ベースの Xserve システムには、LOM（Lights-out Management）プロセッサ用のインターフェイスもあります。このインターフェイスにはそれ自体の IP アドレスがあり、サーバがシステム終了しているときでも、「サーバモニタ」アプリケーションを使ってサーバに接続することができます。

インターネットゲートウェイ用のネットワーク接続サービス

「ネットワーク」パネルの前に「インターネットゲートウェイ」パネルが表示され、このサーバをインターネットゲートウェイにするよう選択した場合、Ethernet 接続の設定をローカルネットワークに変更することはできません。これらの設定は自動的に構成されます。

複数のサーバ用のネットワーク接続サービス

同時に複数のサーバを設定する場合は、「サーバアシスタント」で順番に各サーバの「ネットワーク」パネルが表示されます。パネルの上部付近にあるテキストは、現在構成しているサーバの DNS 名と IP アドレスを示しています。

Xsan メタデータコントローラ用のネットワーク接続サービス

以前に「Xsan」パネルが表示され、サーバを Xsan メタデータコントローラにするよう選択した場合：

  • Xsan の設定中にプライベート・メタデータ・ネットワークが自動的に構成されるようにする予定の場合は、プライベート・メタデータ・ネットワークのネットワーク接続サービスが無効になっていることを確認します。操作方法については、「ネットワーク接続サービスを無効にする」を参照してください。

  • ほかのすべてのネットワーク接続サービスのそれぞれに固定（静的）IP アドレスがあることを確認します。たとえば、「IPv4 の構成」オプションを「手入力」または「DHCP サーバを使用（アドレスは手入力）」に設定します。

    BootP、PPPoE、および DHCP ネットワーク構成オプションは Xsan ネットワークには適していないため、選択することはできません。

自動設定プロファイル用のネットワーク接続サービス

後でサーバを自動的に設定するためにプロファイルを作成する場合、このパネルで構成するネットワーク接続サービスは次のように使用されます。サーバを自動的に設定している間、「サーバアシスタント」は、ハードウェア・アドレス（Mac アドレスまたは Ethernet ID とも呼ばれます）を比較することによって、保存したプロファイル内のネットワーク接続サービスをサーバのネットワークインターフェイスと一致させようとします。これに失敗すると、保存したネットワーク接続サービス（en0、en1、fw0 など）のインターフェイス名（BSD ポート名とも呼ばれます）を、サーバのネットワークインターフェイスと一致させようとします。これに失敗すると、保存したネットワーク接続サービスは使用されません。

さまざまなサーバ上のネットワークインターフェイスを一致させる、汎用のネットワーク接続サービスを設定することができます。2 つの Ethernet サービスと 1 つのAirMac サービスを設定し、それぞれの「IPv4 の構成」オプションを「DHCP サーバを参照」に設定します。自動サーバ設定を始める前に、DHCP サービスが、自動的に設定される各サーバの各ネットワークインターフェイスに固定（静的）IP サービスを割り当てるように構成されていることを確認します。特定のサーバに 2 つの Ethernet インターフェイスまたは 1 つのAirMac インターフェイスがない場合、「サーバアシスタント」は対応する保存済みのネットワーク接続サービスを使用しません。
