Ermitteln eines Schnittstellennamens oder einer Hardwareadresse

Wenn Sie ein Profil für die automatische Serverkonfiguration erstellen und einen Netzwerkverbindungsdienst im Fenster "Netzwerk" hinzufügen wollen, aber für einen mit dem Netzwerk verbundenen Server weder den Schnittstellennamen (BSD-Port) noch die Hardwareadresse (MAC-Adresse) kennen, können Sie diese Informationen mithilfe von Befehlszeilenprogrammen ermitteln. Sie müssen die IP-Adresse des entfernten Servers oder den DNS-Namen und seine Hardwareseriennummer kennen. Melden Sie sich mit ssh im Programm "Terminal" am entfernten Server an. Rufen Sie dann mit networksetup die Netzwerkanschlussdaten ab.

Gehen Sie wie folgt vor, um die Netzwerkschnittstellendaten für einen entfernten Server abzurufen:

  1. Stellen Sie sicher, dass der entfernte Server gestartet wurde und mit dem Netzwerk verbunden ist.

    Wenn Mac OS X Server noch nicht installiert ist, können Sie den entfernten Server mit der Installations-DVD für Mac OS X Server starten.

  2. Öffnen Sie das Programm "Terminal" auf Ihrem Mac und melden Sie sich dann durch Eingabe des folgenden Befehls und Drücken des Zeilenschalters am entfernten Server an:

    ssh root@serveradresse

    Ersetzen Sie serveradresse durch die IP-Adresse oder den DNS-Namen des entfernten Servers.

  3. Geben Sie yes ein, wenn Sie gefragt werden, ob Sie sicher sind, dass Sie die Verbindung herstellen wollen, obwohl die Identität des entfernten Servers nicht bekannt ist.

    Sie werden dies gefragt, wenn Sie sich das erste Mal an einem entfernten Server anmelden.

  4. Wenn Sie zur Eingabe des Kennworts aufgefordert werden, geben Sie die ersten acht Ziffern der Seriennummer des entfernten Servers ein.

  5. Geben Sie den folgenden Befehl ein und drücken Sie den Zeilenschalter:

    networksetup -listallhardwareports

    In der Ausgabe dieses Befehls werden Schnittstellennamen unter "Device" und Hardwareadressen unter "Ethernet Address" angegeben.
