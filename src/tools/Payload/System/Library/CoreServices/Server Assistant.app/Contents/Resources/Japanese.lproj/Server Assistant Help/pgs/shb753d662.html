構成を保存

「構成を保存」パネルでは、入力した設定情報が含まれている自動設定プロファイルを保存します。サーバがアクセスできるディスク上に保存できます。たとえば、リムーバブルドライブや CD に保存することができます。サーバを起動して「ようこそ」パネルが表示されたら、ドライブを接続するか CD を挿入して、自動設定を始めることができます。

「保存」をクリックすると、.plist 拡張子を付ければ、プロファイルに好きなファイル名を付けることができます。「戻る」をクリックして一部の設定を変更し、「続ける」をクリックしてこのパネルに戻ってから、再度「保存」をクリックすることで、変更したプロファイルを保存することができます。

自動設定プロファイルの設定
使用を特定のコンピュータに制限

この自動設定プロファイルを使って、自動設定するサーバの条件を指定するには、このオプションを選択して「編集」をクリックします。

以下のいずれかの条件を含めることができ、それぞれ複数追加できます：

  • シリアル番号： ハードウェアシリアル番号

  • ホスト名： DNS 名のすべてまたは一部

  • IP アドレス

  • MAC アドレス： ネットワークインターフェイスのハードウェア・アドレス（Ethernet ID）

また、条件がコンピュータの適格属性とどのように比較されるかを選択することもできます。この比較は、以下のものを除けば、見ればすぐに分かる内容です：

  • である： コンピュータの適格属性は、条件と完全に一致する必要がある

  • と似ている： 条件の中の ? と * はワイルドカード文字であることを意味する。? はコンピュータの適格属性の任意の 1 文字と一致し、* はゼロ以上の文字と一致する

  • と一致する： サーバの適格属性には、条件を正規表現として解釈することで指定された単語または文字が含まれている必要があります（詳しくは、Web サイト www.icu-project.org/userguide/regexp.html にある「ICU User Guide」のセクション「Regular Expressions」を参照してください）。

複数の条件を指定すると、プロファイルはそれらのいずれかを満たすコンピュータに適用されます。

適格なコンピュータに複数の DNS 名、IP アドレス、または MAC アドレスがある場合は、すべて対応する条件と比較されます。 たとえば、ハードウェア・アドレスを指定する条件は、コンピュータのすべてのネットワークポートと比較されます。

暗号化

自動サーバ設定プロファイルを暗号化された形式または標準テキストのどちらで保存するかを選択します。

  • 「パスフレーズ暗号化」を選択する場合は、自動設定プロファイルの暗号化と復号化のためのパスフレーズを入力します。

    設定を始めるときにパスフレーズを再度入力するか、パスフレーズが含まれているテキストファイルを自動設定プロファイルと同じフォルダ内に入れて、パスフレーズファイルに自動設定プロファイルと同じ名前（ただし、拡張子は .pass）を付けることができます。

  • 「なし」を選択すると、プロファイルは標準テキストとして保存されます。

重要： プロファイルにだれでもアクセスできるようになる可能性がある場合は、「なし」を選択しないでください。プロファイルにはサーバの管理者アカウントのパスワードが含まれており、だれかがこれを使って、プロファイルを使用して設定したサーバを制御する可能性があります。

保存したプロファイルを使用するには、そのプロファイルを、自動的に設定したいコンピュータに接続されたボリュームの最上位（ルート）レベルにある「自動サーバ設定」という名前のフォルダに入れます。ボリュームは、コンピュータのハード・ドライブ、またはコンピュータに接続されている iPod、DVD、CD、FireWire ドライブ、USB ドライブなどの記憶装置上にあります。

Mac OS X Server がインストールされていても、設定はされていないコンピュータを起動すると、自動的に「自動サーバ設定」フォルダ内で有効な自動設定プロファイルに使用できるすべてのディスクが検索されます。複数のプロファイルが見つかった場合は、設定しているコンピュータと最も正確に一致するプロファイルが使用されます。最も正確に一致するものを判別するため、条件がハードウェアシリアル番号、ネットワークインターフェイス MAC アドレス（ハードウェア・アドレス）、IP アドレス、DNS 名、コンピュータ名、条件なしの順に比較されます。 プロファイルに複数の条件が含まれている場合は、それらのいずれかを満たすコンピュータに適用されます。
