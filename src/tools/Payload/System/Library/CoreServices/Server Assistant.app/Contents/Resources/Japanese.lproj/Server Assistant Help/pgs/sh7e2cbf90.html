LOM ポートのネットワーク接続サービスを構成する

Intel プロセッサを搭載した Xserve システムには別個の LOM（Lights-out Management）プロセッサが組み込まれており、これにより電源が入っていないときでもサーバと通信することができます。LOM プロセッサには、サーバの内蔵 Ethernet ポート用の独自のネットワーク設定があります。

LOM ポートのネットワーク接続サービスを構成するには：

  1. 左側のリストで LOM ポートのサービスを選択します。

  2. 必要に応じて、LOM 管理者の名前とパスワードを入力します。

    LOM 管理者アカウントは Mac OS X Server 管理者アカウントとは異なりますが、必要に応じて同じ名前とパスワードを使用することもできます。

  3. 「構成」をクリックし、構成方法を選択して、下の情報を参考にして構成の詳細を入力します。

    • IP アドレス： 
      ネットワーク上でサーバを一意に識別する数字アドレス。ピリオドで区切られた 4 つの数字として表されます（たとえば、192.0.2.123）。

      LOM インターフェイスにそれ自体の IP アドレスを指定します。Xserve の背面にある同じ物理コネクタ（ポート 1 または 2）を使用する場合でも、内蔵 Ethernet 1 または Ethernet 2 に割り当てたものと同じアドレスは使用しないでください。

    • サブネットマスク： 
      サーバの IP アドレスのネットワーク部分を識別する、4 つの番号のグループ。IP アドレスの残りの部分は、ネットワーク上のサーバを一意に識別します。
    • ルーター： 
      サーバのローカルネットワークのルーターの IP アドレス（たとえば、192.168.99.1 または 10.0.7.1）。ルーターは、データパケットを、ローカルネットワークからほかのネットワークまたはインターネット上の宛先に転送する装置です。ネットワークにルーターがない場合は、代わりにサーバの IP アドレスを入力します。

LOM 管理者の名前またはパスワードは、「管理者」アカウントをクリックすることによって変更できます。

サーバの設定後に、「サーバモニタ」を使用して、LOM ポートのネットワーク設定を変更したり、LOM 管理者アカウントを追加または変更することができます。操作方法については、「サーバモニタ」（「/アプリケーション/サーバ/」にあります）を開いて、「ヘルプ」メニューを参照してください。また、「ターミナル」で IPMITool を使用して、コマンドラインからこれらの設定を変更することもできます。詳しくは、IPMITool のマニュアルページ（man で表示）を参照してください。
