Connect to a Directory Server

Use the “Connect to a Directory Server” pane to specify the directory server you’ll import user accounts from. You can import existing user accounts from your organization’s Open Directory server or Active Directory server. Your server uses the account name, password, and other attributes that already exist in the imported user accounts. Your server can augment the existing attributes with attributes needed for its services. For example, imported users can be members of a group you create on your server.

Settings for importing from a directory server
Server Address

Select the directory server from the pop-up menu, or enter the directory server’s DNS name. If you’re connecting to an Open Directory server, you can enter its IP address.

Server Type

Can be Open Directory or Active Directory.

If you’re setting up a server now (not creating an auto setup profile for later use), Server Type is determined automatically.
User account name and password

These fields appear if the directory server allows or requires authentication.

  • For an Open Directory server, you can enter the short name and the password of a standard user account; you don’t need to use a directory administrator account.

  • For an Active Directory server, you can enter the name and password of an Active Directory administrator account or a standard user account that has the “Add workstations to domain” privilege.
