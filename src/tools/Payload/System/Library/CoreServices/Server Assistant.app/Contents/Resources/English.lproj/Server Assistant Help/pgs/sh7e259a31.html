Administrator Account

Use the Administrator Account pane to set up a user account that has server administration privileges.

Administrator account settings
Name

The full name you want used for the primary administrator account—for example, Server Administrator. This name can contain up to 255 characters (as few as 85 Japanese characters). It can include spaces. It can’t be the same as any predefined user name, such as System Administrator.

Short Name

This is an abbreviation of the full name. The short name typically is eight or fewer characters, but can be up to 255 Roman characters, typically eight or fewer. Use only a through z, A through Z, 0 through 9, period (.), _ (underscore), or - (hyphen). You can’t use a predefined user’s short name, such as root. After the account is created, you can’t change the short name.

Password

Should be hard to guess and kept secret. Anyone who knows an administrator name and password can change server settings and manage user accounts.

Password Hint

Can be anything that will help you recall your password (except the password itself).
Enable administrators to log in remotely using SSH

Allows server administrators to use the SSH (Secure Shell) network protocol to log in to the server from another location.
Enable administrators to manage this server remotely

Allows server administrators to observe and control your server over the network by using the display, keyboard, and mouse of a Mac with Mac OS X screen sharing or Apple Remote Desktop software, which you can purchase separately.

Other administrator accounts

The password you enter here is also used for the System Administrator account, whose short name is root. If your server will have its own users and groups, this password will also be used for a directory administrator account, whose name is Directory Administrator and short name is diradmin. You can enter a different directory administrator name and short name if you choose the “Configure manually” option in the “Users and Groups” pane later.

The System Administrator account has full access to the system, including system files. Protecting this root user password is very important, so it should not be the same as another account’s password. Server administrators don’t need root user privileges.

Important:  After setting up the server, you should either change the password of the account you create here and the directory administrator account, or change the password of the System Administrator (root) account.

To change the password or other attributes of the administrator account you create here, use the Accounts pane of System Preferences after logging in to the server.

To change the password or other attributes of the Directory Administrator account, use the Workgroup Manager application after setting up the server.

To change the password of the System Administrator (root) account, use the Login Options section of the Accounts pane of System Preferences to open the Directory Utility application after setting up the server.

Keeping your server secure

For security, you should create a standard user account (not an administrator account) and routinely use it to log in. Use an administrator account with Server Preferences, Server Admin, Workgroup Manager, or other administration applications and tools when you need to manage users, groups, or services.

You can create standard user accounts by using Server Preferences or Workgroup Manager.
