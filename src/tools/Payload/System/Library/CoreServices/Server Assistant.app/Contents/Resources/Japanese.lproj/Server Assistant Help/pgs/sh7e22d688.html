Web サービスについて

Web サービスは、Wiki、ブログ、Web カレンダー、メールへの Web アクセス、およびカスタムの Web サイトを提供することができます。

Web サービスがオンになっている場合は、サーバ上のユーザアカウントを持っているすべてのユーザが、「マイページ」と呼ばれる Wiki ポータルを使って、Wiki とブログの表示と作成、Web カレンダーの使用、Wiki アップデートの追跡、Web メールへのアクセスを簡単に行うことができます。

すべてのユーザが簡単に Wiki を作成でき、その Wiki コンテンツをだれが表示、検索、編集できるかを制御できます。付属のテンプレートを使用したり、独自のテンプレートを作成したりして、マークアップコードや特殊な構文の知識がなくても、ユーザが無理なくコンテンツの追加、削除、編集、および書式設定を実行できます。 2、3 回クリックするか、ドラッグとドロップを行うだけで、ファイルやイメージの添付、Podcast の公開、キーワードの割り当て、ほかの Wiki ページまたは Web サイトへのリンクを実行できます。また、Wiki の変更履歴全体を表示したり、前のページに戻ったりすることもできます。

Web カレンダーでは、ユーザが任意の Web ブラウザを使用して予定や会議などのイベントを管理できます。ユーザは、イベントへの案内を送受信できます。イベントの案内をするとき、ユーザはタイムラインで各人のスケジュールを確認できます。各ユーザの「マイページ」ポータルには個人用 Web カレンダーがあり、それぞれの Wiki には、その Wiki へのアクセス権を持つすべてのユーザが使用できるカレンダーを含めることもできます。ユーザは、自分の個人用 Web カレンダー上と、アクセス権を持っている Wiki グループカレンダー上に、複数のカレンダーを作成できます。Snow Leopard を使用しているユーザは、「iCal」アプリケーションを使用して、Web カレンダーに表示されるイベントを表示、変更、作成することもできます。

ブログを使用すると、技術者以外のユーザでも、プロジェクト、自分が作業中のファイル、ピクチャ、Podcast の最新情報をほかの社員に提供できます。一連のプロフェッショナルな内蔵テンプレートを使って、ドラッグ・アンド・ドロップだけで簡単に、ユーザは自分のブログを公開できます。

Web メールを使用すると、ユーザはインターネットに接続している場所ならどこでも Web ブラウザを使用してメールを送受信できます。ユーザのコンピュータ上で Mac OS X の Mail またはほかのメールアプリケーションを使用しているかのように、すべてのメールにアクセスできます。

これらの Web サービスに加え、Web サイト開発ソフトウェアを使用して、（自分または自分以外の人が作成した）カスタムの Web サイトを公開することもできます。各 Web サイトへのアクセス権を特定のグループに制限したり、Web サイトの一部を特定のグループに制限したりできます。また、各 Web サイトの IP アドレス、アクセスポート、およびサーバ上で Web サイトファイルを保管するフォルダを指定することもできます。カスタムの Web サイトは仮想ホストとも呼ばれています。
