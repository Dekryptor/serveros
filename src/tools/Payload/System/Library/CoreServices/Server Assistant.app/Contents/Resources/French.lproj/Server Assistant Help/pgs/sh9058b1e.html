Gestion AirPort

La sous-fenêtre Gestion AirPort permet d’utiliser Mac OS X Server pour gérer une borne d’accès AirPort Extreme (802.11n) ou une Time Capsule de telle sorte qu’elle protège votre réseau local tout en autorisant l’accès à certains services depuis Internet. Pour activer la gestion AirPort :

  • Vous devez saisir le mot de passe de la borne d’accès ou de la Time Capsule (et non le mot de passe du réseau sans fil).

  • Si votre serveur peut se connecter à d’autres périphériques que la borne d’accès AirPort Extreme ou la Time Capsule, vous devez connaître le nom de celui que doit gérer votre serveur.

Une fois la configuration initiale terminée, vous pouvez indiquer dans les Préférences du serveur les différents services auxquels vous souhaitez pouvoir accéder hors de votre réseau local. Mac OS X Server configurera votre borne d’accès AirPort Extreme ou votre Time Capsule de sorte que les demandes reçues pour ces services soient envoyées à votre serveur.

Si votre serveur fournit, par l’intermédiaire de votre borne d’accès AirPort Extreme ou de votre Time Capsule, un service VPN aux utilisateurs qui disposent de leur propre borne d’accès AirPort, Time Capsule ou autre routeur configuré pour partager une connexion à Internet, l’adresse IP de votre serveur ne peut pas commencer par les trois chiffres des adresses IP des utilisateurs VPN, tels que 10.0.1 ou 192.168.1. Pour savoir comment éviter les conflits d’adresses IP, consultez le manuel Premiers contacts livré avec Mac OS X Server.

La sous-fenêtre Gestion AirPort s’affiche si l’interface réseau principale de votre serveur est connectée à Internet par l’intermédiaire de votre borne d’accès AirPort Extreme ou de votre Time Capsule. De plus, l’option Partage de connexion de votre borne d’accès AirPort Extreme ou de votre Time Capsule doit être définie sur « Partager une adresse IP publique » (c’est-à-dire une connexion Internet).

La sous-fenêtre Gestion AirPort ne s’affiche pas pendant la configuration à distance de plusieurs serveurs.
