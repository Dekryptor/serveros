Utilisateurs et groupes (Xsan)

Dans la sous-fenêtre Utilisateurs et groupes d’un contrôleur de métadonnées Xsan principal, l’option sélectionnée indique la façon dont vous gérez les comptes d’utilisateur et de groupe, qui permettent d’authentifier les utilisateurs des ordinateurs du SAN et de déterminer les services auxquels ils peuvent accéder. Les comptes d’utilisateur et de groupe permettent de contrôler l’accès aux fichiers et dossiers du SAN en attribuant des autorisations d’accès aux différents utilisateurs et groupes.

Options de gestion des utilisateurs et des groupes
Gérer les utilisateurs et les groupes avec Xsan Admin

Permet de configurer le contrôleur de métadonnées principal comme maître Open Directory, de manière à ce qu’il dispose de son propre service d’annuaire pour les comptes d’utilisateur et de groupe. Utilisateurs et groupes peuvent être créés et gérés à l’aide de Xsan Admin, une fois la configuration terminée. Xsan Admin configure automatiquement les ordinateurs du SAN pour qu’ils puissent se connecter au service Open Directory.

Cette option est recommandée si vous ne disposez pas déjà d’un serveur d’annuaire et si vous ne pensez pas avoir plus de 20 utilisateurs SAN. Ces utilisateurs disposeront de dossiers de départ locaux sur leur ordinateur mais pas de dossiers de départ réseau sur le serveur.

Important :  si vous sélectionnez cette option, assurez-vous de bien utiliser Xsan Admin (et non l’option Préférences du serveur) pour créer des utilisateurs et des groupes. En effet, contrairement à l’option Préférences du serveur, Xsan Admin crée un dossier de départ pour le nouveau compte d’utilisateur.
Important :  cette option ne peut être sélectionnée que si vous utilisez cet Assistant réglages pour configurer Mac OS X Server sur le contrôleur de métadonnées principal. Vous ne pourrez plus configurer cette option par la suite.
Utiliser les utilisateurs et groupes provenant d’un serveur Open Directory

Cette option vous permet de connecter le contrôleur de métadonnées principal à un serveur Open Directory existant, dont vous indiquez le nom DNS ou l’adresse IP dans la sous-fenêtre suivante. Xsan Admin configure automatiquement les ordinateurs du SAN pour qu’ils puissent se connecter au serveur Open Directory. La gestion des utilisateurs et des groupes est assurée par un administrateur du serveur Open Directory.

Se connecter à un serveur d’annuaire ultérieurement

Cette option vous permet de ne réaliser la connexion à un serveur Open Directory existant qu’une fois la configuration terminée, ou de vous connecter à un autre type de serveur d’annuaire, tel que Active Directory, une fois la configuration terminée. Pour configurer des connexions à un service d’annuaire sur des ordinateurs du SAN avec Snow Leopard, consultez la rubrique Options - Se connecter, dans la sous-fenêtre Comptes des Préférences Système. Pour configurer des connexions à un service d’annuaire sur des ordinateurs du SAN avec Leopard, utilisez l’application Utilitaire d’annuaire.

Si ces options n’apparaissent pas, c’est que vous n’êtes pas en train de configurer un contrôleur de métadonnées Xsan principal. Voir Utilisateurs et groupes.
