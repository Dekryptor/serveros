インターネットゲートウェイ

「インターネットゲートウェイ」パネルでは、ローカルネットワークをインターネットにリンクして、インターネット接続をローカルネットワーク上のほかのコンピュータと共有するようにサーバを設定します。

「インターネットゲートウェイ」パネルが表示されるのは、サーバの 2 つのEthernet ポートが以下のように構成されている場合のみです：

  • 1 つの Ethernet ポートは、インターネット上のパブリック IP アドレス（10.0.0.1 や 192.168.1.1 のようなプライベート IP アドレスではなく）を持っている必要があります。このポートは、DSL モデム、ケーブルモデム、またはその他のインターネットソースに接続されます。通常は、サーバの 1 つ目の内蔵 Ethernet ポートをこのために使用します。

  • もう 1 つの Ethernet ポートは、ローカル・プライベート・ネットワーク内の動作しているネットワークスイッチまたはハブに接続する必要があります。この Ethernet ポートは、未構成か、手入力の IP アドレスを持っているか、169.254 で始まる自己割り当ての IP アドレスを持っている必要があります。このローカルネットワークに接続されているほかのコンピュータは、サーバのインターネット接続を共有します。

サーバをインターネットゲートウェイとして設定するには：

「このサーバをインターネットゲートウェイとして構成」を選択します。

「サーバアシスタント」により、どの Ethernet ポートがパブリックインターフェイスで、どれがローカルインターフェイスかが自動的に決定されます。

パブリックインターフェイスまたはローカルインターフェイス用に複数の Ethernet ポートを使用できる場合は、ポップアップメニューから使用したいポートを選択します。

サーバをゲートウェイとして設定するには、以下のように操作します：

  • ローカルネットワークに接続されている Ethernet ポートに、プライベート IP アドレス 192.168.1.1 を割り当てます。

  • DHCP サービスを有効にして、ローカルネットワーク上のコンピュータに IP アドレス 192.168.1.100 〜 192.168.1.199 を提供するように構成します。DHCP サービスはこれらのアドレスを、Ethernet ポートが「DHCP サーバを参照」オプションによって構成されているコンピュータに割り当てます。

    また、ユーザにアドレス 192.168.1.2 〜 192.168.1.99 を提供し、「DCHP サーバを使用（アドレスは手入力）」オプションで構成するために、それらの Ethernet ポートを使用することもできます。アドレス 192.168.1.200 〜 192.168.1.220 はサーバの VPN サービス用に予約されています。

  • サーバのインターネット接続をローカルネットワーク上のコンピュータと共有するように、NAT サービスを設定します。

  • インターネット上のコンピュータから来る受信接続をブロックするように、サーバのファイアウォールを設定します。ファイアウォールはローカルネットワーク上のコンピュータからの送信接続を許可します。また、ローカルコンピュータの送信接続に応答している受信接続を許可します。設定後に、「サーバ環境設定」の「セキュリティ」パネルで、特定のサービスに対するファイアウォール経由の着信要求を許可することができます。

  • ローカルネットワーク用の DNS サービスを設定して、ローカルネットワークのパフォーマンスを向上させるために DNS 名のルックアップをキャッシュするように構成します。
