ブラックリストに記載された送信者からのメールを拒否する

DNS ブラックリスト（DNSBL）サーバによってオープンリレーとしてブラックリストに組み入れられたほかのメールサーバからのメールを、お使いのサーバのメールサービスで拒否することができます。お使いのメールサーバは、Spamhaus Project（www.spamhaus.org）が運営する zen.spamhaus.org という DNSBL サーバを使用しています。別の DNSBL サーバを指定することもできます。DNSBL は、リアルタイムブラックリスト（RBL）サーバまたはブラックホールサーバとして知られています。

ブラックリストに記載された送信者からの迷惑メールは、正確にブロックされないことがあります。有効なメールであっても、正しく設定されていないメールサーバから送信されたものは、受信されない場合があります。

メールサービスに使用したい DNSBL サーバを指定するには：

「サーバ環境設定」の「メール」パネルで、「ブラックリストにあるサーバからのメールを拒否」を選択し、その横にある「編集」ボタンをクリックして、DNSBL サーバのアドレスを入力します。
