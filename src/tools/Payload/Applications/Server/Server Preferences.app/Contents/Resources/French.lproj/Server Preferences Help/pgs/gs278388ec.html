Configuration d’un serveur à distance

Si vous avez récemment installé un nouveau serveur ou un ordinateur fonctionnant avec Mac OS X Server, vous pouvez le configurer via le réseau grâce à Admin Serveur et Assistant du serveur à partir d’un ordinateur administrateur. Il n’est pas nécessaire que le serveur que vous configurez soit équipé d’un écran. Pour obtenir des informations sur les ordinateurs administrateurs, consultez Préparation d’un ordinateur administrateur. Pour obtenir des informations sur la configuration de plusieurs serveurs à distance, ouvrez Admin Serveur et allez ensuite dans le menu Aide.

Pour configurer un serveur distant :

  1. Préparez la configuration en remplissant une copie imprimée de la feuille d’opérations Installation et configuration.

    La feuille d’opérations Installation et configuration se trouve dans le dossier Documentation sur le disque d’installation de Mac OS X Server.

  2. Si les services DHCP ou DNS vous sont fournis par votre fournisseur d’accès à Internet, un routeur Internet ou par d’autres serveurs de votre réseau, assurez-vous qu’ils sont configurés pour votre nouveau serveur et qu’ils fonctionnent.

    Pour obtenir des informations sur la configuration du DHCP, du DNS et des autres services réseau, consultez le chapitre 2, « Prêt pour Mac OS X Server », dans le manuel Premiers contacts.

  3. Assurez-vous que votre serveur dispose d’une connexion active à un réseau Ethernet sécurisé.

  4. Si vous voulez configurer votre serveur en tant que passerelle Internet afin que d’autres ordinateurs de votre réseau puissent bénéficier de la connexion Internet du serveur, vérifiez ce qui suit :

    • Un seul port ou une seule interface Ethernet se connecte à votre modem DSL, à votre modem câble ou à une autre source Internet. L’interface Internet doit avoir une adresse IP publique (et pas une adresse IP privée, telle que 10.0.1.1 ou 192.168.1.1). L’adresse IP publique doit être statique (fixe) afin que les ordinateurs des utilisateurs puissent accéder au serveur via Internet pour, par exemple, utiliser le service VPN du serveur.

    • Un autre port Ethernet se connecte à votre réseau local.

    Au cours de la configuration, Assistant du serveur identifie automatiquement quel port est connectée à Internet.

  5. Si le serveur est éteint, allumez-le.

    Quand le serveur démarre, Assistant du serveur s’ouvre automatiquement afin de démarrer la configuration à distance.

  6. Sur un ordinateur administrateur, ouvrez Admin Serveur et sélectionnez Prêt pour configurer dans la liste à gauche.

    Admin Serveur est situé dans /Applications/Serveur/.

    Si Admin Serveur vous demande un mot de passe pour vous connecter à un serveur qui est déjà configuré sur votre réseau, vous pouvez cliquez sur Annuler. Il n’est pas nécessaire d’avoir un compte administrateur pour configurer Mac OS X Server à distance.

  7. Sélectionnez le serveur cible sur la droite et cliquez ensuite sur Configurer.

    Admin Serveur identifie les serveurs non configurés par nom, adresse IP et adresse MAC. Le nom est généré à partir du modèle de l’ordinateur et de l’adresse MAC.

    Si le serveur dont vous avez besoin n’est pas listé, vous pouvez cliquez sur le bouton Actualiser (flèche courbe) pour qu’Admin Serveurcherche de nouveau des serveurs prêts à être configurés sur votre réseau local.

    Si le serveur dont vous avez besoin est sur un réseau local différent, sélectionnez Serveur > « Configurer un serveur distant ».

  8. Une fois l’Assistant du serveur ouvert, effectuez l’une des opérations suivantes, selon l’état du serveur que vous voulez configurer :

    • Si le serveur est listé dans la catégorie Authentification requise, sélectionnez-le, cliquez sur Authentifier et tapez son mot de passe.

    • Si le serveur n’est pas répertorié, cliquez sur Ajouter, saisissez le nom DNS du serveur ou son adresse IP au format IPv4 (par exemple, 192.0.2.200) et tapez son mot de passe.

    Le mot de passe pour une nouvelle installation de Mac OS X Server correspond aux huit premiers caractères du numéro de série matériel intégré. Pour trouver le numéro de série, cherchez une étiquette collée sur le serveur. Respectez la casse du numéro de série quand vous le saisissez. Pour un Xserve avec processeur Intel dont la carte logique principale a été remplacée et qui n’a pas de numéro de série, saisissez « System S » (sans les guillemets) comme mot de passe. Pour un ordinateur qui n’a pas de numéro de série matériel intégré, utilisez 12345678 comme mot de passe.

    Le mot de passe pour un serveur mis à niveau correspond au mot de passe de l’utilisateur root du serveur.

  9. Cliquez sur Continuer et passez aux sous-fenêtres de l’Assistant du serveur, en suivant les instructions à l’écran et en saisissant les informations que vous avez enregistrées sur la feuille d’opérations Installation et configuration.

    Pour obtenir des informations sur les réglages dans une sous-fenêtre de l’Assistant du serveur, cliquez sur le bouton Aide de la sous-fenêtre.

  10. Une fois la configuration du serveur terminée, l’Assistant du serveur propose trois options :

    • Configurer un autre serveur : 
      Revenez à l’étape 1 et configurez un autre serveur pour la première fois. (Puisque l’Assistant du serveur est déjà ouvert, vous pouvez ignorer les étapes 6 et 7.)
    • Configurer : 
      Ouvrez Préférences du serveur sur l’ordinateur que vous avez utilisé pour configurer le serveur distant. Vous pouvez ensuite ouvrir une nouvelle connexion vers le serveur distant et ainsi configurer les utilisateurs et les groupes, personnaliser les services et les informations système, et contrôler l’activité du serveur.
    • Partage d’écran : 
      Démarrez une session partage d’écran avec le serveur que vous venez de configurer. Au moment de l’authentification, utilisez le nom et le mot de passe du compte administrateur que vous avez créé sur ce serveur.

    Une fois la configuration de votre nouveau serveur terminée, vous pouvez effectuer des étapes supplémentaires pour en améliorer la sécurité, l’accessibilité et la fonctionnalité générale. Pour obtenir des informations, consultez Après la configuration d’un serveur.
