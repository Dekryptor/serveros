Fourniture d’un service VPN via un routeur Internet

Si votre serveur fournit un service VPN via un borne d’accès AirPort Extreme, une Time Capsule ou tout autre routeur configuré pour le partage d’une connexion Internet et si les ordinateurs des utilisateurs ont besoin d’établir des connexions VPN via leurs propres bornes d’accès ou routeurs domestiques, votre serveur doit se trouver sur un sous-réseau différent de celui des ordinateurs domestiques des utilisateurs VPN. Pour éviter tout conflit, assurez vous que les trois premiers numéros de l’adresse IP de votre serveur (tels que 10.0.1 ou 192.168.1) ne sont pas identiques à ceux des adresses IP sur les réseaux domestiques des utilisateurs VPN :

En demandant aux utilisateurs de modifier leurs adresses réseau :

Vous pouvez demander aux utilisateurs VPN de modifier les adresses IP sur leurs réseaux domestiques ou sur d’autres réseaux locaux pour que les trois premiers numéros ne soient pas identiques à ceux des adresses IP sur votre réseau local.

Par exemple, si votre adresse IP locale commence par 192.168.1, demandez aux utilisateurs VPN d’utiliser des adresses IP commençant par 192.168.2 sur leurs réseaux domestiques. Les réseaux privés peuvent utiliser des adresses commençant de 192.168.0 à 192.168.254, de 10.0.0 à 10.254.254 ou de 172.16.0 à 172.31.254. Dans tous les cas, utilisez le masque de sous-réseau 255.255.255.0.

En modifiant les adresses de votre réseau local :

Au lieu de demander aux utilisateurs VPN de modifier les adresses de leur réseau domestique, vous pouvez modifier les adresses IP de tous les périphériques situés sur le réseau local de votre serveur, y compris celles de votre borne d’accès AirPort ou de tout autre routeur, de votre serveur et de tous les autres ordinateurs.

Modifiez vos adresses IP locales afin que votre sous-réseau IP soit différent des sous-réseaux par défaut les plus répandus sur les bornes d’accès et autres routeurs, soit 10.0.1, 192.168.0 et 192.168.1.

Vous n’avez qu’à choisir un numéro différent, compris entre 2 and 254 pour le troisième numéro de vos adresses IP locales. Par exemple, si les adresses IP de votre réseau local commencent par 192.168.1, modifiez-les de sorte à ce qu’elles commencent par 192.168.58 ou 192.168.177. Si vos adresses IP locales commencent par 10.0.1, modifiez-les de sorte à ce qu’elles commencent par 10.0.29 ou 10.0.103. Vous pouvez également utiliser les valeurs 172.16.0 à 172.31.255. Dans tous les cas, utilisez le masque de sous-réseau 255.255.255.0.

Assurez-vous de modifier les adresses IP que votre borne d’accès AirPort, tout autre routeur Internet ou serveur DHCP attribue aux ordinateurs de votre réseau. Pour procéder à ces modifications sur une borne d’accès AirPort, utilisez Utilitaire AirPort (qui se trouve dans /Applications/Utilitaires/). Pour obtenir des instructions, ouvrez Utilitaire AirPort, puis consultez le menu Aide. Pour obtenir des informations sur la configuration d’un autre type de routeur ou de passerelle Internet, consultez la documentation correspondante.

Pour obtenir des informations sur la modification de l’adresse IP de votre serveur, consultez Modification de l’adresse IP de votre serveur.
