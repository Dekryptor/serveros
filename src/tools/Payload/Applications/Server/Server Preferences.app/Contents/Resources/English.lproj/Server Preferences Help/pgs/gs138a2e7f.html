If a User Isn’t Invited to Join Your Server

When a Mac with Mac OS X v10.6 Snow Leopard joins your network, it may automatically display an invitation to get services from your server. The invitation may not appear for several reasons.

The invitation to join your server doesn’t appear automatically if:

  • The user’s Mac has a Mac OS X version earlier than 10.6 Snow Leopard. For information about setting up that Mac, see Setting Up Users’ Computers Manually.

  • The user previously declined an invitation to join your server.

  • The user was logged in on his or her Mac while you created the user’s account on the server. The user needs to log out and then log in again.

  • The user didn’t log in on his or her Mac using his or her usual account.

  • The user’s account on your server doesn’t match the account on the user’s Mac. To match, the two accounts must have the same name or short name. For example, if the account name and short name are Gita Kumar and gkumar on the server but are Gita and gita on the user’s Mac, the accounts don’t match.

    To fix this, change the name in the user account on the server to match the user account name on the user’s Mac. The user can also connect manually without matching names. For instructions, see Setting Up a Mac by Using Accounts Preferences.

  • The user’s Mac already has a directory services connection. The user can manually add a connection to your server. For instructions, see Setting Up a Mac by Using Accounts Preferences.

  • The user’s account on the server is imported from a directory server (not created on your server). The user receives an email invitation with a button for getting services from your server. For more information, see Setting Up Newly Imported Users’ Macs.

  • Your server doesn’t have at least one service turned on in Server Preferences. For more information, see Checking Service Status.

  • The user’s Mac or your server has a network connection problem.
