新しい Mac を設定する

ユーザがサーバにアカウントを持っている場合、そのユーザは、新しい Mac または Mac OS X v10.6 Snow Leopard を新規にインストールした Mac の初期設定中に、「Mac OS X Server に接続」パネルを使ってそのサーバを選択できます。

「Mac OS X Server に接続」パネルは、Mac がネットワーク上に Snow Leopard Server を実行するサーバを検出した場合のみ表示されます。ユーザは「サーバ環境設定」の「ユーザ」パネルでアカウントを持っている必要があります。また、サーバの「サーバ環境設定」には、少なくとも 1 つのサービスが有効でなければなりません。

ユーザがこのパネルでの入力を完了すると、次のようになります：

  • サーバ上のアカウントに基づいて、ローカルのユーザアカウントがユーザの Mac 上に作成されます。両方のアカウントで同じロングネーム、ユーザ名、パスワードが使用されます。

  • ホームフォルダが、ユーザのコンピュータ上で設定されます。

  • ユーザのコンピュータが自動的にサーバに接続され、サーバのサービスを使用できるように構成されます。ユーザがアクセスできるサービスについては、「ユーザの Mac を自動的に設定する」を参照してください。

いくつかの理由により、「Mac OS X Server に接続」パネルでの入力を完了できない場合があります。たとえば、次のように入力します：

  • ユーザがサーバの名前を知らないか、またはサーバ上にユーザアカウントがありません。このような場合、ユーザは「次の Mac OS X Server を使用」の選択を解除して、「Mac OS X Server に接続」パネルでの入力を省略できます。

  • ユーザの Mac が、初期設定中にネットワークに接続されていません。この場合、「Mac OS X Server に接続」パネルでなく「ローカルアカウントの作成」パネルが表示され、サーバアカウントに基づかない新規アカウントが作成されます。

ユーザが何らかの理由で「Mac OS X Server に接続」パネルでの入力を完了していない場合でも、ユーザは、初期設定を完了した後で Mac を構成すれば、サーバのサービスを使用できます。ユーザがこれを行う方法については、「お使いのネットワークに参加する Mac を設定する」を参照してください。
