Managing File Sharing Service

Use the File Sharing pane to turn file sharing service on or off, and to control access to the Groups, Public, and Users shared folders. You can also add or remove your own shared folders, also known as share points.

To configure the File Sharing pane:

  • Click the On/Off switch to turn file sharing service on or off. This control also indicates whether file sharing is on or off.

  • Click checkboxes in the Shared Folders list to enable or disable access to the Backups folder, Groups folder, Public folder, Users folder, and each shared folder you add.

  • Click the Add (+) button to add a new shared folder.

    Adding a Shared Folder

  • Click the Delete (–) button to make a shared folder unavailable to file sharing users.

    Removing a Shared Folder

  • Click Edit Permissions to choose whether guest access is on or off, and change users’ read/write access, for a shared folder.

    Controlling Access to a Shared Folder

The Backups shared folder contains backup data for users who have their Time Machine preferences set to use the server for backup storage. You can’t use the Edit Permissions button or Delete (–) button with the Backups shared folder.

The Groups shared folder contains a folder for each group that has a file sharing folder set up in the Groups pane. You can’t use the Delete (–) button to make the Groups folder unavailable to file sharing users.

The Public shared folder contains files and folders put in it by users with accounts listed in the Users pane of Server Preferences.

The Users shared folder contains home folders for user accounts listed in the Accounts pane of System Preferences on the server.
