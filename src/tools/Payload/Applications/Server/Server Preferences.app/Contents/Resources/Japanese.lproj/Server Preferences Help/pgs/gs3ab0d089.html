SSL 証明書を使う

サーバは、SSL 証明書を使用して自身を電子的に識別し、ユーザのコンピュータやローカルネットワークおよびインターネット上のほかのサーバと安全に通信を行うことができます。SSL 証明書により、アドレスブック、iCal、iChat、メール、および Web サービスのセキュリティが一層高まります。これらのサービスでは、ユーザのコンピュータのアプリケーションとやりとりするデータを、証明書を利用して安全に暗号化／複合化します。

SSL 証明書を使うには：

  1. 「サーバ環境設定」の「情報」パネルで、「SSL 証明書」の右側にある「編集」ボタンをクリックします。

  2. 「SSL 証明書を使用」を選択し、次に、使用可能な証明書をポップアップメニューの上部から選択します。

    ポップアップメニューに証明書が含まれていない場合は、自己署名証明書を作成できます。操作方法については、「自己署名証明書を作成する」を参照してください。

    以前に生成された SSL 証明書を使用したい場合は、それを読み込むことができます。操作方法については、「証明書を読み込む」を参照してください。

サーバを設定したときにサーバ用に作成された自己署名証明書、または自分で作成した自己署名証明書を使うことはできますが、ユーザのアプリケーションではそれらは自動的には信頼されず、ユーザがその証明書を信頼するかどうかを確認するメッセージが表示されます。 署名入り証明書を使えば、そのようなメッセージを介して手動で証明書を受け取ることの不安や煩雑さがなくなり、ユーザは安心できます。また、自己署名証明書を使えば中間者攻撃によるなりすましが可能になりますが、署名入り証明書ではそれは不可能なので、ユーザは自分がアクセスしているサービスを信頼できます。「署名入り証明書を入手する」および「自己署名証明書を置き換える」を参照してください。
