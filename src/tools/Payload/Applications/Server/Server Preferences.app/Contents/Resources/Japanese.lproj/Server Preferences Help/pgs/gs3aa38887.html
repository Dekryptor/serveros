Web サービスを設定する

「サーバ環境設定」の「Web」パネルを使って、サーバの Web ホームページを変更したり、標準 Web サービスの有効／無効を個別に切り替えたりできます。

Web サービスを構成するには：

  1. 「サーバ環境設定」の「Web」パネルで「Web サービス」をクリックします。

  2. ポップアップメニューを使って、サーバの Web サイトのホームページを選択します。

    • Web サイト開発ソフトウェアで（自分または自分以外の人が）作成したデフォルトの Mac OS X Server Web サイトまたはカスタムの HTML Web サイトを使用するには、「サーバのホームページ」を選択します。

    • Wiki を選択して、それをお使いのサーバのメイン Web サイトにします。

      ポップアップメニューが淡色表示されている場合は、Wiki サービスが無効になっています。

      ポップアップメニューに Wiki が含まれていない場合は、当該ユーザまたはサーバ上のユーザアカウントを持っているだれかが Wiki を作成する必要があります。

    ポップアップメニューの代わりに「Wiki を作成」ボタンが表示されている場合は、Wiki に使用するサイトが「サーバ管理」を使って削除されています。 ボタンをクリックして Wiki を使用可能にしてください。

    ホームページおよび Web サービス設定の代わりに「カスタムサイトを表示」ボタンが表示されている場合は、カスタム Web サイトが、標準の Web サービス（Wiki、Web カレンダー、ブログ、および Web メール）と同一の DNS 名およびアクセスポート番号を使用して作成されています。通常は、Web サービスには、サーバの DNS 名（「サーバ環境設定」の「情報」パネルに示されます）と、ポート 80 を使用します。標準の Web サービスを使用可能にするには、ボタンをクリックしてカスタム Web サイトを削除します。

  3. 利用するサービスを選択して、利用しないサービスの選択を解除します。

    • Wiki： 
      サーバのユーザ全員が、Web ブラウザを使って自分の「マイページ」の Wiki ポータルを閲覧したり、Wiki を作成したり、アクセス権のある Wiki を使用したりできるようにします。Wiki オーナーは、自分の Wiki の「設定」セクションでアクセス制限を設定できます。
    • カレンダー： 
      各ユーザの「マイページ」ポータルにカレンダーを追加し、Wiki の作成者が各 Wiki に Web カレンダーを含めることができるようにします。
    • ブログ： 
      すべてのサーバユーザが Web ブラウザを使用してブログを作成できます。
    • Web メール： 
      すべてのサーバユーザが Web ブラウザを使って自分のメールにアクセスできるようにします。

    サービスが有効になっている場合は、その隣の矢印のリンクをクリックすると、そのサービスの Web サイトが開きます。

  4. Web サービスが無効になっている場合は、「入」／「切」スイッチをクリックして有効にします。

    Web サービスが無効になっている場合は、Wiki、Web カレンダー、ブログ、および Web メールは使用できません。

ユーザはサーバのホームページ（http://myserver.example.com ）にアクセスできます（斜体の部分をサーバの DNS 名に置き換えてください）。ホームページでユーザがリンクをクリックすると、「マイページ」、Wiki、ブログ、カレンダー、および Web メールが表示されます。サーバのホームページがプライベート Wiki である場合は、Wiki の閲覧権限があるユーザの名前とパスワードを使用してログインする必要があります。
