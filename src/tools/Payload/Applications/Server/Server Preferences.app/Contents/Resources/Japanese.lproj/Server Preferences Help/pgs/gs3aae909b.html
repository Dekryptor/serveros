署名入り証明書を入手する

自己署名証明書を使って、既知の認証局から署名入り証明書を入手できます。自己署名証明書は通常、自動的には信頼されませんが、署名入り証明書は信頼されます。SSL 証明書を使用できるサーバでは、ユーザのアプリケーションおよびそのほかのサーバと安全に通信するために、信頼された証明書が必要です。

有効な署名入り証明書を入手するには、自己署名証明書を使って証明書署名要求（CSR）ファイルを生成し、それを認証局に送信します。認証局によって申請が承認されると、署名入り証明書が作成、返送されます。

署名入り証明書を入手するには：

  1. 「サーバ環境設定」の「情報」パネルで、「SSL 証明書」の右側にある「編集」ボタンをクリックします。

  2. ポップアップメニューで、使用したい自己署名証明書を選択します。

    「SSL 証明書を使用」が選択されていないと、ポップアップメニューは表示されません。

  3. ポップアップメニューで「証明書署名」＞「証明書サイン要求を生成」と選択し、「保存」をクリックして、証明書署名要求（CSR）ファイルの場所を選択します。

  4. CSR を認証局へ送ります。

    通常は、CSR ファイルを「テキストエディット」で開き、その内容全体を認証局の Web サイトにあるフォームにコピー＆ペーストします。利用する認証局に連絡し、CSR の送信方法を決定します。認証局については、Web を検索してください。

認証局から自分の SSL 証明書を受領したら、それを自分の自己署名証明書と置き換えることができます。 操作方法については、「自己署名証明書を置き換える」を参照してください。
