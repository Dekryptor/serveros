新しいグループを作成する

サーバのユーザに専用の共有グループフォルダが必要となったとき、または共有フォルダおよび共有ファイルへのアクセスを制御する必要が生じたときはいつでも、新しいグループを作成できます。

新しいグループを作成するには：

  1. 「サーバ環境設定」の「グループ」パネルで追加（＋）ボタンをクリックします。

  2. グループの名前を入力し、必要に応じてユーザ名を変更します。

    グループ名は 255 文字（2 バイト文字は 85 文字まで）以内で入力します。空白を含めることができます。

    アカウントの作成後にユーザ名を変更することはできません。生成されたユーザ名を使用したくない場合は、別のユーザ名を入力します。

    ユーザ名は通常、8 文字以下ですが、1 バイト文字を 255 文字まで使用できます。a ～ z、A ～ Z、0 ～ 9、.（ピリオド）、_（アンダースコア）、または -（ハイフン）だけを使ってください。

  3. グループに割り当てるサービスを選択して、除外するサービスの選択を解除します。

    • ファイル共有フォルダ： 
      グループにファイル共有フォルダを提供します。メンバーは、グループの共有フォルダからファイルを取得したり、ファイルをこのフォルダに置いたりできます。このフォルダはグループのユーザ名を元に名前が付けられ、サーバの起動ディスクの「/グループ/」フォルダに置かれます。「ファイル共有フォルダ」の隣にあるリンクの矢印をクリックすると、そのグループの共有フォルダが Finder で開きます。
    • iChat 自動メンバーリスト： 
      すべてのグループメンバーの Jabber ID（スクリーン名）を、自動的に各メンバーの Jabber メンバーリストに追加します。グループに外部メンバーが存在する場合、そのメンバーは iChat 自動メンバーリストには含まれません。
    • グループ Wiki を作成： 
      お使いの Web ブラウザが開き、このグループ名を使って Wiki を作成するプロセスが開始します。Wiki を作成する際、視覚テーマを選択し、アクセス権を設定します。Wiki には、デフォルトで Web カレンダー、ブログ、およびメーリングリストが含まれています。作成した Wiki を後からカスタマイズできます。詳細については、任意の Wiki ページの「ヘルプ」リンクをクリックしてください。

    このグループ名を使って Wiki が作成された場合、「グループ Wiki を作成」ボタンは、次回「サーバ環境設定」の「グループ」パネルを表示したときに Wiki へのリンクに置き換わります。

  4. 「メンバー」をクリックしてグループにユーザを追加します。「外部メンバー」をクリックして、組織のディレクトリサーバからユーザとグループを追加します。操作方法については、「グループのメンバーを追加する／取り除く」および「グループの外部メンバーを追加する／取り除く」を参照してください。

    「外部メンバー」タブが表示されない場合は、サーバがディレクトリサーバに接続されていません。「サーバをディレクトリサーバに接続する」を参照してください。

共有フォルダまたはそのグループに使用権限のあるプライベート Wiki にアクセスするために、グループメンバーは自分のユーザアカウント名とパスワードを使用して認証される必要があります。グループサービスが使用可能かどうかは、ファイル共有サービス、iChat サービス、および Web サービスが有効かどうかによります。
