Si des utilisateurs ne peuvent pas accéder à un service via Internet

Si des utilisateurs peuvent accéder à un service sur le réseau local et non sur Internet, essayez les solutions suivantes :

  • Si la sous-fenêtre Sécurité de Préférences du serveur contient le message « Exposer ces services sur __ », assurez-vous que le service souhaité est répertorié.

  • Si la sous-fenêtre Sécurité de Préférences du serveur contient le message « Le gestionnaire AirPort est désactivé », activez-le, puis vérifiez que le service souhaité est répertorié.

  • Si votre serveur se connecte à Internet via une borne d’accès AirPort ou via un autre routeur Internet, il peut être nécessaire de configurer le routeur pour que les services de votre serveur soient disponibles sur Internet. Si vous utilisez une borne d’accès AirPort Extreme (802.11n) ou Time Capsule, Mac OS X Server peut le configurer automatiquement. Pour obtenir des informations sur la configuration d’autres modèles de bornes d’accès AirPort ou d’un routeur Internet, consultez le chapitre 2, « Préparation pour Mac OS X Server », du manuel Premiers contacts.
