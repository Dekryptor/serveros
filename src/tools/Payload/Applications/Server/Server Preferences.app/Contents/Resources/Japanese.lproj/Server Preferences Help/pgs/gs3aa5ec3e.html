VPN サービスについて

VPN（Virtual Private Network）サービスを使用すると、ユーザは、自宅またはほかのリモートの場所からインターネット経由でネットワークに接続できます。 ユーザは、セキュリティ保護された VPN 接続を確立して、ファイル共有、アドレスブック、メール、iChat、iCal、Web などのサービスにアクセスします。

VPN サービスは、機密性、認証、通信の整合性を保証するために、共有シークレットを含む L2TP プロトコルを使用します。

セキュリティ保護された共有シークレットは、サーバの設定時に自動的に生成されます。共有シークレットは、VPN 接続のためのクライアント・コンピュータ・ユーザの認証には使用されません。その代わりに、サーバは共有シークレットを持つクライアントコンピュータを信頼でき、クライアントコンピュータはシークレットを持つサーバを信頼できます。

サーバとクライアントコンピュータの両方が共有シークレットを持っている必要があります。Mac OS X v10.6 Snow Leopard を実行するコンピュータでは、自動的に共有シークレットが取得されてサーバの VPN サービスに接続するように設定できます。「ユーザの Mac を自動的に設定する」を参照してください。

ほかの Mac および Windows コンピュータを異なる方法で構成して、VPN サービスに接続させることができます。「Mac ユーザの VPN 接続を設定する」および「ユーザの VPN 接続を手動で設定する」を参照してください。

VPN サービスとお使いのサーバのファイアウォールは、どちらもローカルネットワークの外部からのサービスへのアクセスを許可できます。異なる点は、VPN サービスにはアクセスの認証が必要ですが、ファイアウォールを介したアクセス許可には認証が不要ということです。 VPN サービスが有効な場合は、お使いのファイアウォールを介してインターネットに公開されている一部のサービスが不要となることがあります。たとえば、Web サービスのみがインターネットに公開されるようにファイアウォールを設定して、お使いの Wiki およびカスタム Web サイト（どのような認証およびアクセス制限を設定するかによります）を一般ユーザが閲覧できるようにすることが可能です。サーバのユーザは、VPN 接続を介してほかのサービス（ファイル共有、アドレスブック、iCal、iChat、メール）にアクセスできます。

インターネット上の VPN サービスへのアクセスを許可したい場合で、ケーブル・ルーター、DSL ルーター、またはそのほかのネットワークルーターを持っている場合：

  • お使いのルーターでは VPN サービス用にポートフォワード（ポートマッピング）を設定する必要があります。ポートフォワードの詳細については、「お使いになる前に」マニュアルを参照してください。

  • お使いのルーターおよび VPN ユーザのルーターは、競合する IP アドレスを割り当てないように設定する必要があります。詳細については、「インターネットルーター経由で VPN サービスを提供する」を参照してください。

ローカルネットワークの外部の VPN サービスへのアクセスを許可したい場合で、お使いのローカルネットワークに独立したファイアウォールデバイスがある場合は、ファイアウォールの管理者に問い合わせて、VPN サービスで使用するポートおよびプロトコルのファイアウォールを解放してもらってください。ポートのリストについては、「お使いになる前に」マニュアルの付録を参照してください。
