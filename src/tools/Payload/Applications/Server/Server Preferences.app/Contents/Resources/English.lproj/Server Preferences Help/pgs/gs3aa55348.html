Managing Web Services

Use the Web pane of Server Preferences to turn the Web Server on or off, change the location of your server’s web homepage, publish custom websites, or turn standard web services—wikis, web calendars, blogs, and webmail—on or off.

To configure the Web pane:

Change the desired settings, using the following information as a guide:

  • On/Off: 
    Indicates the status of the Web Server, and clicking the On/Off switch turns the Web Server on or off.
  • Web Services: 
    Lets you choose a homepage for your server and turn wikis, web calendars, blogs, and webmail services on or off individually.
  • Custom Sites: 
    Lets you add, remove, or edit custom websites that you have created (or someone has created for you) using website development software.
  • Home page: 
    Choose Server Home Page to use the default Mac OS X Server website. Choose a wiki to make it your server’s website.
  • Wikis: 
    Makes all wikis available. If “Wikis” is selected, clicking the link arrow next to “Wikis” opens a list of wikis.
  • Calendar: 
    Adds a calendar to each user’s My Page portal, and allows wiki creators to include a web calendar in each wiki. If “Calendar” is selected, clicking the link arrow next to it requests a user name and password, and then opens that user’s web calendar.
  • Blogs: 
    Lets all server users create blogs using a web browser. If “Blogs” is selected, clicking the link arrow next to it opens a list of blogs.
  • Webmail: 
    Lets all server users access their mail using a web browser. If “Webmail” is selected, clicking the link arrow next to it opens the webmail website.
  • Create the Wiki: 
    This button appears if the site used for wikis has been deleted using Server Admin. Click the button if you want to make wikis available.
  • Reveal Custom Site: 
    This button appears if a custom website has been created using the same DNS name and communications port number as the wiki. Usually the wiki uses the server’s DNS name (shown in the Information pane of Server Preferences), and it uses port 80. Click the button if you want to make wikis available.
