Protection du compte administrateur système (root)

Le mot de passe d’administrateur saisi au cours de la configuration est également utilisé pour le compte utilisateur de l’administrateur système du serveur. Son nom abrégé est root. Vous pouvez utiliser le compte administrateur système (root) pour déplacer ou supprimer tout fichier du système, y compris les fichiers système auxquels ne peuvent pas accéder un compte administrateur du serveur ou tout autre compte utilisateur. Vous n’avez pas besoin des privilèges d’un utilisateur root pour administrer votre serveur.

Important :   protéger le mot de passe de l’utilisateur root est très important. Il doit donc être différent des mots de passe de tous les autres comptes. Après avoir configuré le serveur, vous devez modifier le mot de passe du compte utilisateur root.

Pour modifier le mot de passe de l’utilisateur root, ouvrez la sous-fenêtre Comptes dans les Préférences Système, sélectionnez Options d’ouverture de session, cliquez sur le bouton Modifier, puis sur Ouvrir l’utilitaire d’annuaire et choisissez Modifier > Changer de mot de passe root. Si le bouton Se connecter apparaît à la place du bouton Modifier dans les Options d’ouverture de session, cliquez dessus, puis sur Ouvrir l’utilitaire d’annuaire et choisissez Modifier > Changer de mot de passe root.

Vous pouvez également modifier le mot de passe de l’utilisateur root en ouvrant Terminal (dans /Applications/Utilities/) et saisissez sudo passwd root.
