アカウント環境設定を使用して Mac を設定する

Snow Leopard を実行する Mac がサーバに未接続で、サーバのサービスを利用するための案内が自動的に表示されない場合は、Mac ユーザは手動でそのサーバに接続できます。ユーザの Mac のローカルアカウントにサーバ上のユーザアカウントと一致する名前またはユーザ名がない場合、またはサーバが Mac とは異なるローカルネットワーク（IP サブネット）上に存在する場合、ユーザは手動でサーバに接続する必要があります。前回のサーバの使用案内を辞退した場合、または Mac とサーバの接続を削除したため再接続が必要な場合にも、ユーザが手動で接続しなければならないことがあります。

Mac をサーバに接続するには：

  1. Mac で「システム環境設定」の「アカウント」パネルを開き、「ログインオプション」をクリックします。

  2. 「接続」ボタンまたは「編集」ボタンをクリックして、追加（＋）ボタンをクリックします。

    「接続」ボタンは、Mac がディレクトリサーバに未接続の場合に表示されます。

    「編集」ボタンは、Mac がすでにディレクトリサーバに接続されている場合に表示されます。

  3. ポップアップメニューからサーバを選択し、そのサーバの DNS 名または IP アドレスを入力します。

  4. サービスの設定を促すダイアログが表示された場合は、ユーザのアプリケーションを設定してサーバのサービスを使用するかどうかを選択します。

    • サーバのサービスを使用するようアプリケーションを設定したい場合は、「サービスを設定」をクリックします。サーバでユーザアカウントの名前とパスワードを入力します。

      ユーザアカウントの変更およびユーザがアクセスできるサービスについては、「ユーザの Mac を自動的に設定する」を参照してください。

    • Mac を Open Directory ドメインに接続するだけでよい場合は、「接続のみ」をクリックします。ユーザのアプリケーションは設定されず、ユーザのローカルアカウントはユーザのサーバアカウントに関連付けられません。

    これらを選択するためのダイアログは、サーバで「サーバ環境設定」のサービスがいずれも有効になっていない場合、または手順 3 で指定したサーバで Mac OS X Server v10.6 以降が稼動していない場合は、表示されません。
