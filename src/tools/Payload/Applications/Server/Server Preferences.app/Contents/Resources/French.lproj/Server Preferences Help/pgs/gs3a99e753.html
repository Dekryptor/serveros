À propos des comptes d’administrateur

Vous devez disposer d’un compte administrateur sur votre serveur pour pouvoir créer d’autres comptes utilisateur ou des groupes, pour modifier les réglages du serveur et pour exécuter d’autres tâches à l’aide de Préférences du serveur. Le compte administrateur vous permet également de modifier les préférences verrouillées dans les Préférences Système, d’installer des logiciels sur le serveur et d’exécuter des tâches que les utilisateurs standards ne sont pas autorisés à exécuter.

Une fois la première configuration terminée, votre serveur dispose de deux comptes administrateur : un compte administrateur principal et un compte administrateur d’annuaire.

Compte administrateur principal

Le serveur a toujours un compte administrateur principal, dont vous avez saisi le nom et le mot de passe au moment de la configuration du serveur. Le compte administrateur principal est stocké sur le serveur avec les comptes utilisateur créés à l’aide de la sous-fenêtre Comptes dans les Préférences Système. Vous pouvez utiliser ce compte administrateur sur le serveur ; il peut également vous permettre de gérer votre serveur via le réseau depuis un autre Mac.

Compte administrateur d’annuaire

Si votre serveur héberge des utilisateurs et des groupes dans son propre annuaire cela signifie qu’il a également un compte administrateur d’annuaire. Ce compte utilise le mot de passe que vous avez saisi pour l’administrateur principal lors de la configuration, mais le nom correspondant est Administrateur de l’annuaire et le nom abrégé est diradmin. Lors de la configuration, vous pouvez saisir un nom et un nom abrégé différents grâce à l’option Configurer manuellement.

Le compte administrateur d’annuaire se trouve dans l’annuaire de votre serveur, de même que les comptes utilisateur standard que vous créez à partir de la sous-fenêtre Utilisateurs dans Préférences du serveur. Cependant, le compte administrateur d’annuaire n’apparaît pas dans la sous-fenêtre Utilisateurs des Préférences du serveur. Si une anomalie rend le compte administrateur principal inutilisable, vous pouvez utiliser le compte administrateur d’annuaire directement sur le serveur pour gérer le serveur via le réseau à partir d’un ordinateur administrateur.

Administrateurs sur un serveur mis à niveau

Si vous avez mis à niveau ou fait migrer votre serveur à partir d’une configuration de groupe de travail ou standard de Mac OS X Server 10.5, vous disposez de plusieurs comptes administrateur. Votre compte administrateur principal se trouve dans le répertoire de votre serveur. C’est en fait un compte administrateur d’annuaire dont le nom et le nom abrégé sont spécifiés au moment de la configuration du serveur Leopard. Vous disposez également d’un compte administrateur stocké sur votre serveur, ayant pour nom Administrateur local et nom abrégé localadmin. Pour en savoir plus sur ces comptes, consultez le manuel Premiers contacts pour Mac OS X Server 10.5. Vous le trouverez sur le site web des manuels Apple à l’adresse support.apple.com/manuals/ (en anglais).

Comptes administrateur supplémentaires

Lorsque vous créez un compte utilisateur, vous devez spécifier s’il s’agit d’un utilisateur administrateur ou standard. Vous pouvez également transformer un compte utilisateur importé en administrateur de serveur. Si vous ne souhaitez pas qu’un utilisateur soit autorisé à utiliser Préférences du serveur ou à installer des logiciels sur le serveur, ne lui donnez pas d’autorisation administrateur.

Sécurité du compte administrateur

Pour préserver la sécurité de votre serveur :

  • Ne confiez votre nom d’administrateur et votre mot de passe à personne.

  • Fermez la session lorsque vous quittez votre serveur ou mettez en place un économiseur d’écran verrouillé à l’aide de la sous-fenêtre Économiseur d’écran et de la sous-fenêtre Sécurité dans les Préférences Système. Si vous vous absentez alors que votre session est ouverte et que l’écran est déverrouillé, n’importe qui peut utiliser votre serveur et modifier des éléments à l’aide de vos autorisations d’administrateur.

  • Ne configurez jamais le serveur pour qu’il démarre automatiquement une session administrateur. Si vous le faites, toute personne qui redémarre le serveur peut ouvrir une session en tant qu’administrateur.

  • Pour renforcer la sécurité, ouvrez régulièrement une session sur le serveur via un compte utilisateur standard. Utilisez votre nom et votre mot de passe administrateur pour ouvrir Préférences du serveur ou toute autre application exigeant des autorisations administrateur.
