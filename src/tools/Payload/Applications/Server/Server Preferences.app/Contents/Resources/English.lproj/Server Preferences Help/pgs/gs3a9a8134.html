Managing Users

In the Users pane, you set up accounts for people who use the services that this server provides, and you control which services they can access. You can update their contact information and change their group memberships. You can also manage email welcome messages that go to new users.

Options
Add (+) button

Lets you create a new user account or import an existing user account from your organization’s directory server. (You have the choice of importing only if your server is connected to another directory server in your organization.) For instructions, see Adding a User Account.

Delete (–) button

Lets you delete the selected user account. For instructions, see Deleting a User Account.

Action (gear) button

Lets you manage automatic user import from groups or manage the email messages that are sent to welcome new users, invite newly imported users to use the server, and invite new external members of a group to use its services. For instructions, see Customizing Welcome and Invitation Email and Importing Groups of Users Automatically.

Account

Lets you change a user’s name, password, picture, or administrator privilege. For instructions, see Changing a User’s Account Settings.

Contact Info

Lets you change a user’s first and last names, address, email and chat addresses, website address, and blog address. For instructions, see Changing a User’s Contact Information.

Services

Lets you control a user’s access to individual services. For instructions, see Controlling a User’s Access to Services.

Groups

Lets you add a user to a group or remove a user from a group. For instructions, see Changing a User’s Group Membership.

Set Up

If you see a Set Up button instead of the options above, your server doesn’t have its own directory domain for users and groups. Clicking this button sets one up. For instructions, see Setting Up Users and Groups Management.

If you have imported users, some of their settings are dimmed because they are stored on the directory server that your server is connected to.
