ユーザのバックアップの保管場所を管理する

バックアップの保管場所を必要とする Mac OS X v10.6 Snow Leopard または Mac OS X v10.5 Leopard ユーザは、Snow Leopard Server の「Time Machine」を使って、安全かつ手軽にバックアップ作業を行うことができます。Snow Leopard ユーザが「システム環境設定」の「Time Machine」パネルでバックアップディスクをまだ選択していない場合は、サーバを保存場所にするかどうかを確認するメッセージが自動的に表示されます。そのほかのユーザは、「システム環境設定」の「Time Machine」パネルを開き、バックアップの保管に使用するディスクを変更する必要があります。

「サーバ環境設定」の「Time Machine」パネルで、サーバのディスク領域をユーザのコンピュータのバックアップ保存先として利用できるように設定したり、ユーザのバックアップの保管に使用するディスクを変更したりします。

「サーバ環境設定」の「Time Machine」パネルを構成するには：

次の情報を参考にして、必要に応じて設定を変更します：

  • 「入」／「切」： 
    「/Shared Items/Backups/」にある「Backups」共有フォルダを、このパネルで指定されたディスク上の、ユーザのバックアップ保管場所として使用できるようにします。
  • ディスクを選択： 
    クリックして、現在サーバに接続されているディスクのリストから保管場所を選びます。選択したディスク上に「Backups」共有フォルダが自動的に作成されます。
  • バックアップディスクを選択： 
    このボタンは、ユーザのバックアップの保管用ディスクが選択されていない場合に表示されます。このボタンをクリックして「Backups」共有フォルダを使用可能にし、そのフォルダを格納するディスクを選択します。この操作は、「Time Machine」パネルを有効にすることと同じです。
  • ファイル共有を開く： 
    このボタンはファイル共有サービスが無効のときに表示されます。ボタンをクリックして「ファイル共有」パネルを開きます。ユーザの Mac が「Backups」共有フォルダにアクセスできるようにするには、ファイル共有サービスが有効になっている必要があります。

バックアップディスク情報の下にあるテキスト内の矢印をクリックすると、「ユーザ」パネルでサーバの Time Machine バックアップの保管場所への各ユーザのアクセスを制御したり、Finder に「Backups」共有フォルダを表示したりできます。

バックアップディスクを変更すると、バックアップの保管場所にサーバを使用するように設定されていたユーザの「Time Machine」環境設定も、新しい場所の「Backups」共有フォルダを使用するように自動的に変更されます。ただし、古い「Backups」フォルダ内のユーザのバックアップデータは新しい「Backups」フォルダに自動的にはコピーされません。別のバックアップディスクを選択した後、次回のバックアップは完全バックアップになるため時間がかかることをユーザに通知することをお勧めします。
