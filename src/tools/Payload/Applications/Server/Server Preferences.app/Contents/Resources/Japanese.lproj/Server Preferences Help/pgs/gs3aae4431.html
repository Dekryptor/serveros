インターネットルーター経由で VPN サービスを提供する

サーバが VPN サービスを、AirMac Extreme ベースステーション、Time Capsule、またはインターネット接続を共有するよう設定されたネットワークルーター経由で提供しており、ユーザのコンピュータが自分のベースステーションまたはホームルーター経由で VPN 接続を行う必要がある場合は、サーバが VPN ユーザのホームコンピュータとは別の IP サブネット上にある必要があります。この競合を避けるには、10.0.1 または 192.168.1 など、サーバの IP アドレスの先頭の 3 組の数字が VPN ユーザのホームネットワークの IP アドレスのものとは異なっている必要があります。

ユーザにネットワークアドレスの変更を依頼する：

ホームネットワークまたはほかのローカルネットワークの IP アドレスの先頭の 3 組の数字がローカルネットワークの IP アドレスと重複しないように、VPN ユーザに IP アドレスを変更するよう依頼することができます。

たとえば、自分のローカル IP アドレスが 192.168.1 始まる場合、VPN ユーザには 192.168.2 で始まる IP アドレスをホームネットワークで使用するように依頼します。プライベートネットワークでは、192.168.0 ～ 192.168.254、10.0.0 ～ 10.254.254、172.16.0 ～ 172.31.254 の範囲のアドレスを使用できます。これらのすべての場合に、255.255.255.0 というサブネットマスクを使用します。

自分のローカルネットワークアドレスを変更する：

VPN ユーザにホームネットワークのアドレスの変更を依頼する代わりに、サーバのローカルネットワーク上にあるすべての装置の IP アドレスを変更することができます。これらの装置には、AirMac ベースステーションまたはそのほかのルーター、サーバ、あるいはそのほかのコンピュータが含まれます。

自分の IP サブネットがベースステーションおよびその他のルーターの代表的なデフォルトアドレス（10.0.1、192.168.0、192.168.1）と重複しないように、ローカル IP アドレスを変更します。

ローカル IP アドレスの 3 組目の数字に、2 ～ 254 の範囲でデフォルトとは異なる値を選択します。たとえば、ローカル IP アドレスが 192.168.1 で始まる場合は、192.168.58 や 192.168.177 に変更します。ローカル IP アドレスが 10.0.1 で始まる場合は、10.0.29 や 10.0.103 に変更します。また、172.16.0 ～ 172.31.255 の数字も使用できます。 これらのすべての場合に、255.255.255.0 というサブネットマスクを使用します。

お使いの AirMac ベースステーション、そのほかのインターネットルーター、または DHCP サーバがネットワーク上のコンピュータに割り当てている IP アドレスを必ず変更してください。この変更は、AirMac ベースステーションで「AirMac ユーティリティ」（「/アプリケーション/ユーティリティ/」にあります）を使用して行います。 操作方法については、「AirMac ユーティリティ」を開いて「ヘルプ」メニューを参照してください。ほかの種類のインターネットルーターまたはゲートウェイの構成については、それぞれのマニュアルを参照してください。

サーバの IP アドレスの変更については、「サーバの IP アドレスを変更する」を参照してください。
