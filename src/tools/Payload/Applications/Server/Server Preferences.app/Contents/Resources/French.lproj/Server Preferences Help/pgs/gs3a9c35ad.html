Si des utilisateurs ne peuvent pas configurer leur ordinateur pour utiliser le serveur

Si vous modifiez le nom DNS de votre serveur, les utilisateurs qui obtenaient des services via l’ancien nom DNS du serveur doivent reconfigurer leur ordinateur pour qu’il utilise le nouveau nom DNS :

  • Les utilisateurs Mac OS X 10.6 doivent supprimer votre serveur de la liste des serveurs d’annuaire en ouvrant la sous-fenêtre Comptes des Préférences Système, en cliquant sur Options d’ouverture de session, puis sur Édition, en sélectionnant votre serveur dans la liste et en cliquant sur le bouton Supprimer (–). Ils doivent ensuite remettre votre serveur dans la liste en cliquant sur le bouton Ajouter (+). Pour en savoir plus, consultez Configuration d’un Mac à l’aide des préférences Comptes.

    Dans de rares cas, un utilisateur Snow Leopard ne réussit pas à se connecter à votre serveur si son ordinateur comporte une connexion à un serveur d’annuaire supplémentaire. De tels utilisateurs doivent supprimer tous les serveurs d’annuaire, comme décrit ci-dessus ; normalement, ils doivent ensuite pouvoir ajouter une connexion à votre serveur.

  • Les utilisateurs d’une version antérieure de Mac OS X ou qui utilisent Windows doivent mettre à jour manuellement les abonnements au calendrier partagé, les adresses iChat, les adresses électroniques, les signets web et l’adresse VPN du serveur. Pour en savoir plus, consultez Configuration manuelle des ordinateurs des utilisateurs.
