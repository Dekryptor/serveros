Connexion de votre serveur à un serveur d’annuaire

Votre serveur peut avoir ses propres utilisateurs et groupes. Vous pouvez également le configurer pour importer des comptes utilisateur à partir d’un serveur Open Directory ou Active Directory de votre organisation. Pour pouvoir importer des comptes utilisateur, votre serveur doit être connecté au serveur d’annuaire. Les comptes utilisateur importés ont accès aux mêmes services que les comptes utilisateur créés sur votre serveur.

Si vous n’importez pas certains comptes utilisateur depuis le serveur de répertoire connecté, vous pouvez en faire des membres externes de groupes. Vous pouvez également leur donner accès aux wikis privés de votre serveur. Pour en savoir plus, consultez Importation d’utilisateurs, Ajout ou suppression de membres externes d’un groupe et Configuration des services Web.

Pour se connecter à un serveur d’annuaire :

  1. Ouvrez la sous-fenêtre Comptes dans les Préférences Système sur votre serveur.

    Ouvrez les préférences Comptes

  2. Cliquez sur Option d’ouverture de session et ensuite sur Modifier.

    Si vous voyez le bouton Se connecter au lieu du bouton Modifier, vous devez configurer votre serveur pour qu’il dispose de ses propres utilisateurs et groupes avant de pouvoir le configurer pour importer des utilisateurs à partir d’un serveur d’annuaire existant. Pour obtenir des instructions, consultez Configuration de la gestion des utilisateurs et des groupes.

  3. Cliquez sur le bouton Ajouter (+), puis choisissez le serveur de répertoire dans le menu local ou saisissez le nom DNS ou l’adresse IP du serveur de répertoire.

  4. Si la zone de dialogue se développe pour afficher les champs « Identifiant de l’ordinateur client », Nom d’utilisateur et Mot de passe, saisissez le nom et le mot de passe d’un compte utilisateur sur le serveur d’annuaire.

    • Pour un serveur Open Directory, vous pouvez saisir le nom et le mot de passe d’un compte utilisateur standard. Vous n’avez pas besoin d’utiliser un compte administrateur d’annuaire. Si la zone de dialogue indique que vous devez laisser les champs nom et mot de passe vides, vous pouvez vous connecter sans authentification, même si c’est moins sûr.

    • Pour un serveur Active Directory, vous pouvez saisir le nom et le mot de passe d’un compte administrateur Active Directory ou d’un compte utilisateur standard qui a le privilège « Ajouter des stations de travail au domaine ».
