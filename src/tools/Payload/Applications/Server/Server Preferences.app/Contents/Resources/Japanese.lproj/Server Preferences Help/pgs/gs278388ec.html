サーバをリモートで設定する

新しいサーバ、または Mac OS X Server が新規にインストールされたコンピュータがある場合は、管理者用コンピュータの「サーバ管理」および「サーバアシスタント」を使って、ネットワーク上でそれらを設定できます。設定するサーバにディスプレイは必要ありません。管理用コンピュータについては、「管理用コンピュータを準備する」を参照してください。複数のサーバのリモート設定については、「サーバ管理」を開いて「ヘルプ」メニューを参照してください。

リモートサーバを設定するには：

  1. 「インストールと設定のワークシート」をプリントして記入し、設定作業に備えます。

    「インストールと設定のワークシート」は、Mac OS X Server Install Disc の「Documentation」フォルダ内にあります。）

  2. ご利用の ISP が DHCP または DNS サービスを提供している場合、あるいはネットワーク上にインターネットルーターまたは別のサーバが存在する場合は、それらが新しいサーバ用に設定され、稼動していることを確認してください。

    DHCP、DNS、およびそのほかのネットワークサービスの望ましい設定方法については、「お使いになる前に」マニュアルの第 2 章「Mac OS X Server の準備をする」を参照してください。

  3. お使いのサーバが、安全な Ethernet ネットワークに接続しており、そのネットワークを利用できることを確認します。

  4. サーバをインターネットゲートウェイとして設定し、ネットワーク上のほかのコンピュータがそのサーバのインターネット接続を共有できるようにする場合は、次のことを確認してください：

    • 1 つの Ethernet ポートまたはインターフェイスが、お使いの DSL モデム、ケーブルモデム、またはそのほかのインターネットソースに接続していること。インターネットのインターフェイスは必ず、（10.0.1.1 や 192.168.1.1 などのプライベート IP アドレスではなく）パブリック IP アドレスを 1 つ必要とします。ユーザのコンピュータが（たとえばサーバの VPN サービスを利用するために）インターネット経由でサーバにアクセスするためには、パブリック IP アドレスは静的（固定）でなければなりません。

    • そのほかの Ethernet ポートがローカルネットワークに接続していること。

    設定中に、インターネットに接続しているポートが「サーバアシスタント」によって自動的に識別されます。

  5. サーバが起動していない場合は起動します。

    サーバが起動すると、「サーバアシスタント」が自動的に開き、リモート設定の開始を待機する状態になります。

  6. 管理者用コンピュータで「サーバ管理」を開き、左側のリストで「設定の準備ができました」を選択します。

    「サーバ管理」は「/アプリケーション/サーバ/」にあります。

    「サーバ管理」で、設定済みのサーバに接続するためのパスワードを要求された場合は、「キャンセル」をクリックします。Mac OS X Server をリモートで設定するのに、管理者アカウントは必要ありません。

  7. 右側にあるターゲットサーバを選択して、「設定」をクリックします。

    「サーバ管理」では、未設定のサーバが、名前、IP アドレス、および MAC アドレスによって識別されます。名前は、コンピュータのモデルおよび MAC アドレスを基に生成されます。

    目的のサーバがインストールされない場合は、「リフレッシュ」ボタン（曲がった矢印）をクリックして、ローカルネットワーク上で設定の準備ができているサーバを、「サーバ管理」でもう一度検索することができます。

    目的のサーバが別のローカルネットワーク上にある場合は、「サーバ」＞「リモートサーバを設定」と選択します。

  8. 「サーバアシスタント」が開いたら、設定したいサーバの状況に応じて、以下のいずれかの操作を行います：

    • リストでそのサーバが「認証が必要です」となっている場合は、それを選択して「認証」をクリックし、そのサーバのパスワードを入力します。

    • 目的のサーバがリストに表示されない場合は、サーバの DNS 名またはその IP アドレスを IPv4 形式で入力します（例：192.0.2.200）。

    Mac OS X Server の新規インストールのパスワードは、サーバに組み込まれているハードウェアのシリアル番号の、最初の 8 桁です。シリアル番号は、サーバに貼付されているラベルを参照してください。シリアル番号は、大文字と小文字の違いをラベルの通りに入力します。Intel ベースの Xserve で、メインのロジックボードが交換されているためハードウェアのシリアル番号がないものについては、パスワードとして「System S」と入力します（カギカッコは入力しません）。組み込みのハードウェアのシリアル番号がないコンピュータについては、「12345678」をパスワードとして使用します。

    アップグレードしたサーバのパスワードは、そのサーバのルートユーザのパスワードになります。

  9. 「続ける」をクリックし、画面上の指示に従って「インストールと設定のワークシート」に書き留めた情報を入力しながら、「サーバアシスタント」パネルの操作を進めます。

    「サーバアシスタント」パネルの設定については、パネル内の「ヘルプ」ボタンをクリックしてください。

  10. サーバの設定が完了すると、次の 3 つのオプションが提示されます：

    • 別のサーバを設定： 
      手順 1 に戻り、別のサーバを新規に設定します。（「サーバアシスタント」がすでに開いているので、手順 6 および 7 は省略できます。）
    • 構成： 
      リモートサーバの設定に使用したコンピュータで「サーバ環境設定」を開きます。次に、リモートサーバへの新しい接続を開いて、ユーザとグループの設定、サービスとシステム情報のカスタマイズ、またはサーバの利用状況の監視を行います。
    • 画面を共有： 
      設定したサーバで、画面共有のセッションを開始します。認証を要求されたら、そのサーバで作成した管理者アカウントの名前とパスワードを使用します。

    サーバの設定が完了したら、いくつかの追加手順を踏んで、新しいサーバのセキュリティ、アクセシビリティ、および全体的な利便性を拡張することができます。操作方法については、「サーバの設定を完了したら」を参照してください。
