Agent

Les agents Xgrid exécutent les sous­tâches de calcul d'une tâche. Sous Mac OS X Server, l'agent est désactivé par défaut. Lorsqu'un agent est activé au démarrage, il s'enregistre auprès d'un contrôleur. (Un agent ne peut être connecté qu'à un seul contrôleur à la fois.) Le contrôleur envoie alors à l'agent les instructions et les données nécessaires pour les tâches du contrôleur. À la réception des instructions du contrôleur, l'agent exécute les sous-tâches qui lui sont affectées et renvoie le résultat au contrôleur.

Par défaut, l'agent cherche à s'associer au premier contrôleur disponible sur le réseau local. Sinon, vous pouvez spécifier que l'agent doit s'associer à un contrôleur spécifique.

Vous pouvez également spécifier si l'agent est toujours disponible ou s'il n'est disponible que lorsque l'ordinateur est inactif. Un ordinateur est considéré comme inactif lorsqu'il n'existe aucune entrée de la souris ou du clavier ou qu'il ignore l'activité de l'unité centrale ou du réseau. Si un utilisateur retourne à un ordinateur qui exécute une sous-tâche d'une grille, l'ordinateur continue à exécuter la sous-tâche jusqu'à ce que celle-ci soit terminée.

Par défaut, l'agent de Mac OS X Server est dédié et l'agent d'un ordinateur Mac OS X (pas un serveur) est configuré pour accepter des tâches uniquement lorsque l''ordinateur n'a reçu aucune entrée de l'utilisateur pendant 15 minutes.
